The linear core of attention, reassociated

A matrix multiplied by a column of numbers produces another column: this is applying a linear map, a function that respects addition and scaling. Multiplying two matrices composes the two linear maps they stand for, and — exactly like composing functions — matrix multiplication is associative: (AB)C(AB)C and A(BC)A(BC) are the same matrix, whichever way you bracket them. This post is about one place where that freedom to rebracket is worth real arithmetic: the linear core of an attention layer, the central operation in the neural networks behind current language models.

Attention is a way of computing, for each item in a sequence, a weighted combination of “value” vectors, where the weights come from comparing that item’s “query” against every item’s “key.” Stack the queries, keys, and values as the rows of three matrices QQ, KK, VV, each of shape n×dn \times d — nn items in the sequence, dd numbers per item, with dd typically far smaller than nn. The linear core of the layer is the matrix product

Attention(Q,K,V)=(Q⋅K𝖳)⋅V.\text{Attention}(Q,K,V) = (Q\cdot K^\mathsf{T})\cdot V.

The inner factor QK𝖳Q K^\mathsf{T} is n×nn \times n: it scores every pair of items. Then those scores mix the rows of VV. Read left to right, that is the direct description of what the layer computes — and it is also the expensive way to compute it.

The same layer, bracketed two ways

QK𝖳Q K^\mathsf{T} is an n×nn \times n matrix — one entry for every pair of items in the sequence. But the final result is only n×dn \times d. Associativity lets us avoid ever building the large square in the middle:

(QK𝖳)V=Q(K𝖳V),(Q K^\mathsf{T})\,V \;=\; Q\,(K^\mathsf{T} V),

and the right-hand factor K𝖳VK^\mathsf{T} V is only d×dd \times d. From Impl/Attention.v:

  (* Left-to-right: materializes the n×n score matrix Q·Kᵀ.           *)
  Definition attn_naive : Matrix n d F := mat_mul (mat_mul Q (mat_tr K)) V.

  (* Right-to-left: materializes only the d×d matrix Kᵀ·V.            *)
  Definition attn_fast : Matrix n d F := mat_mul Q (mat_mul (mat_tr K) V).

mat_tr is the transpose, so mat_tr K is K𝖳K^\mathsf{T}. The naive form builds the n×nn \times n score matrix; the fast form builds instead the inner product K𝖳VK^\mathsf{T} V. Nothing of size n×nn \times n is ever formed. That these two produce the same matrix is a single application of associativity:

  Theorem attn_assoc : attn_naive == attn_fast.
  Proof.
    exact (mat_mul_assoc n d n d Q (mat_tr K) V).
  Qed.

The proof names no tactics at all — mat_mul_assoc applied to QQ, K𝖳K^\mathsf{T}, and VV, with the four dimension arguments n d n d, is the theorem. The == is the pointwise equality on matrices: the two sides agree entry by entry.

What the reassociation costs, counted

Reassociating is only worth doing if the second order is genuinely cheaper, so Impl/Attention.v counts the scalar multiplications each order performs. An m×nm \times n by n×pn \times p product has mpmp entries, each a sum of nn products, so it costs mnpmnp multiplications:

Definition mulcost (m n p : nat) : nat := (m * n * p)%nat.

Definition cost_naive (n d : nat) : nat :=
  (mulcost n d n + mulcost n n d)%nat.

Definition cost_fast (n d : nat) : nat :=
  (mulcost d n d + mulcost n d d)%nat.

Adding the two products in each order:

costnaive=ndn+nnd=2n2d,costfast=dnd+ndd=2nd2,\mathrm{cost}_{\mathrm{naive}} = n d n + n n d = 2n^2 d, \qquad \mathrm{cost}_{\mathrm{fast}} = d n d + n d d = 2n d^2,

so when dd is smaller than nn, the ratio is n/dn/d. Both of these are exact counts about the algorithm the definitions denote, not estimates. At the scale of a realistic layer — n=1024n = 1024 items, head dimension d=64d = 64 — the counts are checked as literal numbers. Because the naive count runs into hundreds of millions, well past what Coq’s unary natural numbers can hold in memory, the concrete checks pass through the binary numbers N:

Example cost_naive_1024_64 : N.of_nat (cost_naive 1024 64) = 134217728%N.
Proof.
  rewrite (cost_naive_toN 1024 64).
  reflexivity.
Qed.

Example cost_fast_1024_64 : N.of_nat (cost_fast 1024 64) = 8388608%N.
Proof.
  rewrite (cost_fast_toN 1024 64).
  reflexivity.
Qed.

134217728 multiplications for the naive order against 8388608 for the reassociated one. The transfer lemma cost_naive_toN rewrites the count into a product of binary numbers where the arithmetic is instant, and reflexivity confirms the value. The ratio between the two is checked directly, in Examples/AttentionLayer.v:

Example naive_over_fast_is_16 : (134217728 = 16 * 8388608)%N.
Proof. reflexivity. Qed.

A factor of sixteen — exactly n/d=1024/64n/d = 1024/64. At a size small enough for unary arithmetic to handle on its own, the same counts are checked without the detour through N:

Example cost_naive_16_4 : cost_naive 16 4 = 2048%nat.
Proof. reflexivity. Qed.

Example cost_fast_16_4 : cost_fast 16 4 = 512%nat.
Proof. reflexivity. Qed.

2048 against 512 at n=16n = 16, d=4d = 4 — a factor of four, again n/dn/d.

One rewrite makes the two implementations equal

The point of proving attn_assoc is that a “simple” implementation, written to read like the definition, and an “efficient” one, written to be cheap, can be shown to be the same layer — not by inspection, but by a setoid rewrite that the checker verifies. Examples/AttentionLayer.v writes both:

  Definition attn_simple : Matrix n d F :=
    mat_mul (mat_mul Q (mat_tr K)) V.

  Definition attn_efficient : Matrix n d F :=
    mat_mul Q (mat_mul (mat_tr K) V).

mat_mul respects the matrix equality == — replacing a factor by an equal one gives an equal product — so the associativity law is usable as a rewrite rule on a matrix expression. One rewrite turns the simple form into the efficient one:

  Theorem attn_equal : attn_simple == attn_efficient.
  Proof.
    unfold attn_simple, attn_efficient.
    rewrite mat_mul_assoc.
    reflexivity.
  Qed.

Walking it: unfold replaces the two names by their definitions, so the goal becomes mat_mul (mat_mul Q (mat_tr K)) V == mat_mul Q (mat_mul (mat_tr K) V). rewrite mat_mul_assoc applies associativity left to right, rebracketing the left side to match the right. reflexivity closes what is now an identity. The simple and the efficient implementations are provably the same function.

A run over the rationals

None of this is abstract: with concrete small matrices the layer computes a concrete matrix, and both orders can be checked entry by entry to agree with it. Impl/Attention.v fixes n=3n = 3 items and d=2d = 2, with rational entries:

Definition Qm3 : Matrix 3 2 Q :=
  mat32 (1#1)%Q (2#1)%Q (0#1)%Q (1#1)%Q (1#1)%Q ((-1)#1)%Q.

Definition Km3 : Matrix 3 2 Q :=
  mat32 (1#1)%Q (0#1)%Q (2#1)%Q (1#1)%Q (0#1)%Q (3#1)%Q.

Definition Vm3 : Matrix 3 2 Q :=
  mat32 (1#1)%Q (1#1)%Q (2#1)%Q (0#1)%Q (0#1)%Q (1#1)%Q.

By hand the score matrix and the layer output are

QK𝖳=(14601311−3),(QK𝖳)V=(97233−2).Q K^\mathsf{T} = \begin{pmatrix} 1 & 4 & 6\\ 0 & 1 & 3\\ 1 & 1 & -3 \end{pmatrix}, \qquad (Q K^\mathsf{T})\,V = \begin{pmatrix} 9 & 7\\ 2 & 3\\ 3 & -2 \end{pmatrix}.

The naive order is checked against those six output entries by computation:

Example attn_naive_Q_compute :
     attn_naive 3 2 Qm3 Km3 Vm3 Fin.F1 Fin.F1 == (9#1)%Q
  /\ attn_naive 3 2 Qm3 Km3 Vm3 Fin.F1 (Fin.FS Fin.F1) == (7#1)%Q
  /\ attn_naive 3 2 Qm3 Km3 Vm3 (Fin.FS Fin.F1) Fin.F1 == (2#1)%Q
  /\ attn_naive 3 2 Qm3 Km3 Vm3 (Fin.FS Fin.F1) (Fin.FS Fin.F1) == (3#1)%Q
  /\ attn_naive 3 2 Qm3 Km3 Vm3 (Fin.FS (Fin.FS Fin.F1)) Fin.F1 == (3#1)%Q
  /\ attn_naive 3 2 Qm3 Km3 Vm3 (Fin.FS (Fin.FS Fin.F1)) (Fin.FS Fin.F1)
       == ((-2)#1)%Q.
Proof.
  repeat split; reflexivity.
Qed.

Each Fin.F1 / Fin.FS ... picks a row and a column; the six conjuncts are the six entries of the output matrix above. repeat split breaks the conjunction apart and reflexivity evaluates each entry — the layer really computes these rationals. The reassociated order, attn_fast_Q_compute, is checked against the same six literals independently, with no appeal to attn_assoc: two different bracketings, computed separately, land on the same matrix.

What this is not

The scope should be stated precisely. The layer here is linear attention. A real transformer computes softmax(QK𝖳/d)⋅V\mathrm{softmax}(Q K^\mathsf{T}/\sqrt{d})\cdot V, with a softmax sitting between the score matrix and VV. The softmax is not linear, and with it in place the reassociation above is invalid — you cannot slide V inside it. Dropping the softmax, or replacing it with a linear kernel, is exactly what the “linear attention” family of models does, and it is that linear core, and only it, that this reassociation applies to. Within that core, the saving is real, machine-checked, and comes from one fact: matrix multiplication is associative.
