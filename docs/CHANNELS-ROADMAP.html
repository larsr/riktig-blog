Channels roadmap — faulty transports and the protocols that fix them

Piece-by-piece order for the transport leg of the channel-stack vision: model lossy / reordering / duplicating channels without probability, and prove that retransmission protocols (alternating-bit, sliding window — the essence of TCP, QUIC and their relatives) upgrade a bad channel into a good one. Written for the telecom track; the grammar/codec leg (message formats) is the sibling; this leg is about delivery.

The modeling decision: possibilistic, not probabilistic

A faulty channel is a nondeterministic relation between the sent trace and the delivered trace — no probabilities. Fault classes are closure properties:

Channel class Relation on (sent, delivered)
Perfect delivered = sent
Lossy delivered is a subsequence of sent
Lossy + reordering delivered is a permutation of a subsequence
+ duplicating delivered is a finite multiset of copies of sent items (order free)

These form a lattice under relation inclusion; “better channel” = smaller relation = refinement — the same notion used everywhere else in this project. (The table’s last two rows are aspiration, not yet implementation — see the T1 scope note below.)

A protocol is a channel transformer. Sender and receiver are state machines wrapped around a channel; the composite is again a channel (app-messages in, app-messages out). The theorem shape:

reliable? (sender ∘ C_lossy,dup,reorder ∘ receiver) ⊆ C_perfect-prefix

i.e. the composite relation lands in a stronger class. Reliability is a property of the composite, not the wire.

Safety/liveness split (this is how probability is avoided): - Safety — the application never sees corrupted, duplicated, or out-of-order data — is proved against the fully adversarial channel, unconditionally. - Liveness — everything sent is eventually delivered — is provably impossible against an adversary that drops everything, so it is proved under a fairness hypothesis: a message offered infinitely often is eventually delivered. (Bounded-loss variants — “at most k consecutive losses” — give finitary versions.) Probability, added later if ever, only quantifies fairness; the correctness logic stays. - Collapse theorem per protocol: over the perfect channel, the composite is observationally the identity channel — the retransmission machinery is provably invisible when unneeded. This is the user’s “analyze the no-loss case” made into a theorem.

Stages

T1 — Trace relations. Subsequence, permutation-of-subsequence, sub-multiset-with-duplication as inductive relations on lists; the lattice lemmas (each class contains the previous). Finite traces first; coinductive streams only when liveness demands them. Scope note (2026-07-13): what Channel/Trace.v actually builds is perfect/lossy/dup/lossy_dup/reorder. Reordering is kept as a separate class, incomparable with lossy_dup — Channel/Trace.v proves reorder_not_lossy_dup (a swap has no lossy_dup derivation) — so “each class contains the previous” does not hold as a single chain; the proved inclusions are perfect ⊆ lossy ⊆ lossy_dup, perfect ⊆ dup ⊆ lossy_dup, and perfect ⊆ reorder. The combined permutation-of-subsequence and sub-multiset-with-duplication classes (the table’s last two rows) are deferred; no protocol proof has needed them yet.

T2 — Machines and composition. Protocol endpoints as Mealy machines (state × input → state × output — Channel/Mealy.v’s m_step : S -> I -> S * O produces a single output per step; the output-list generalization was never needed, and Channel/ABP.v’s header documents why it bypasses the Mealy machinery entirely). Composition of sender-machine, channel relation, receiver-machine into a composite trace relation. (The interleaving semantics connects to the Sched/ theory and Interleave.v material.)

T3 — Alternating-bit protocol, safety. The “hello world”: one-bit sequence numbers + retransmit + ack over a lossy/duplicating channel. Theorem: app-delivered is always a prefix of app-sent (no dup, no reorder, no corruption to the app). Unconditional. Scope note (2026-07-13), matching Channel/ABP.v’s own header: the proved theorem is abp_safety : subseq (deliver b0 wire) ms — a subsequence, strictly weaker than the prefix promised above (gaps are allowed: if a whole burst is lost, later messages still get delivered). And no ack/return channel is modeled at all — the sender is a blind burst sender, emitting k copies of each packet unconditionally, with nothing coming back to react to. Closing either gap needs the explicit ack-channel model that T5’s note also asks for.

T4 — Fairness and liveness. Fair-channel hypothesis; ABP liveness: everything sent is eventually delivered. The collapse theorem for ABP over the perfect channel. Done for the finite, non-streaming setting (fair_wire: at least one copy per burst survives). Genuinely open, not yet named as its own stage: true fairness over unbounded/streaming retransmission (“offered infinitely often, eventually delivered”) needs the sender/channel/receiver run modeled coinductively — Channel/Trace.v’s header already earmarked streams for exactly this trigger. T5 will hit the same finite-vs-streaming choice again; whoever revisits either file should decide once, for both.

T5 — Sliding window. Go-Back-N, then selective repeat: sequence numbers beyond one bit, windows, cumulative vs selective acks — the mechanism core of TCP; QUIC’s per-stream reliability is the same theorem instantiated per stream. Safety first, liveness under fairness, collapse over perfect channel. Channel/GoBackN.v’s safety theorem is proved with two scope narrowings worth remembering, both flagged in the file but not here until now: (1) sequence numbers are unbounded nat, not the fixed-width mod-2W wraparound counters a real header uses — the wraparound interaction is unproved; (2) what’s modeled is closer to selective repeat (each message independently retransmitted up to k times) than literal reactive Go-Back-N, which retransmits its whole outstanding window on a detected gap — faithfully modeling that reaction needs an explicit ack/NACK channel or the same coinductive loop T4’s open item wants. A future pass should either rename the file to match what’s proved, or close the gap with the ack-channel model.

T6 — Meet the other legs. Packet formats for these protocols come from the grammar leg (single source of truth; headers are length-prefixed binary — the restricted Take/Repeat parser layer). Dogfood target: minikernel’s hand-rolled tcp.c. Long-range: the same machinery hosts RDCP/QUIC-style protocol experiments — the point of the framework is that a new retransmission scheme only owes the two theorems (safety unconditional, liveness under fairness) and inherits everything else.
