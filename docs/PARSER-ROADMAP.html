Parser roadmap — the categorical toolkit meets ~/enter-docker/parser

Source of truth for what already exists: ~/enter-docker/parser (not ~/coq-abnf, which is an empty scaffold — checked 2026-07-17). That project has a working monadic parser combinator library (ParserCombinators.v), an ABNF meta-parser (ABNFAst.v, ABNFParser.v), a grammar-driven parser GENERATOR (ABNFInterp.v’s parse_with_grammar/print_with_grammar, turning any parsed ABNF grammar into a runnable parser+printer for it), a whitespace-preserving pretty-printer (ABNFPretty.v), and — the crown jewel — ABNFGrammar.v’s self-hosting test: parse RFC 5234’s own grammar text with the static hand-written parser, feed the resulting AST to the generic interpreter* to build a dynamic parser, run that dynamic parser on the same text again, and check the pretty-printed result is letter-for-letter identical to the original — bootstrapping trust in the generator from the static parser. IMPLEMENTATION_NOTES.md there also documents real, hard-won lessons (PEG ordered-choice pitfalls, input-length-as-structural-bound termination, nested-inductive guard-checker traps) that the port below must not silently relearn.

Known limitation (Lars): ABNF cannot express length-prefixed binary fields (<length n><n bytes>) — ABNF is context-free, this needs a value threaded from an earlier parse into a later one. Out of scope for the port itself; noted for wave 3+.

The idea in one sentence

A parser IS a monad (S -> M (T * S) for M := option, or more generally any monad M), programs are Kleisli arrows in it, and “insert a channel between two composed programs” is exactly “insert an isomorphism in the Kleisli category” — the VSIso pattern (post #6/#9) transported from Vect to Kleisli(M).
