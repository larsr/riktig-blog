Linear algebra roadmap — coordinate-free first, matrices as representations

Principle: operations are defined as (multi)linear maps, never by components. Coordinates enter only through Basis (a linear iso onto Fin n -> F), and every component-level formula is a computation lemma obtained by transporting along that iso. Matrix multiplication is composition, transported. House style throughout: operational notations from Ops, laws as == equations, proofs as rewrite chains.

Wave L7 — decompositions (abstract first, matrices only as computation)

Principle for this wave specifically: a matrix is a basis-specific representation of a linear map (post #10), so factorization theorems belong on LinearMap/Hom first — a fact true regardless of basis — with a Matrix version only where it buys something concrete (extraction, an efficient algorithm). Ordered by dependency, not by importance.

Layer 1 — structural theorems on LinearMap, no Matrix import. - [~] Rank–nullity: dim V == dim (ker f) + dim (im f). Half done — Theory/Subspace.v builds the general reusable piece (a predicate P closed under +/scale/0 makes {v : V | P v} a VectorSpace, full Monoid->VectorSpace chain, Pointwise.v-style), and Theory/KerIm.v gives Ker f/Im f as instances of it, both resolving VectorSpace F (Ker f)/(Im f) by pure search. What’s missing, precisely named: subspace_has_basis_statement — every subspace of an n-dim space has a basis of some dimension k <= n (Steinitz exchange) — genuine dimension theory, not in the tree anywhere; without it there’s no way to even STATE dim (ker f) as a number. rank_nullity_statement is written as the target Prop (unproved) at the bottom of KerIm.v. - [ ] First isomorphism theorem: V / ker f ≅ im f. Needs quotient vector spaces, a construction not yet built (though Sub/ SubspacePred from Theory/Subspace.v may make the quotient construction — V modulo a subspace — a nearby, related build). - [ ] Direct-sum / invariant-subspace decomposition: V ≅ U ⊕ W with f block-diagonal — the abstract root of every block-matrix trick. - [ ] Idempotent ↔︎ splitting correspondence: f∘f == f iff V = ker f ⊕ im f. Half-built already — InnerGeometry.v’s proj/proj_orthogonal is an orthogonal projection; generalize the idempotent side. - [ ] Rank factorization: every rank-r map factors as (surjective onto an r-dim space) then (injective from it) — the abstract fact behind every low-rank matrix trick (relevant to the project’s stated interest in flash-attention/KV-cache reshaping). - [ ] Spectral theorem: self-adjoint f (Theory/Adjoint.v’s self_adjoint exists) has an orthonormal eigenbasis. The deep one — PDP⁻¹ and SVD are both computational shadows of this. Needs eigenvalues to exist: either an algebraically closed field or root-finding, which belongs in the analysis roadmap’s interval-refinement track, not here.
