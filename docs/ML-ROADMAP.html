Machine learning roadmap — transformers, flows, diffusion

Added 2026-07-08 at the user’s request: the concrete target that motivates finishing the analysis track. Two distinct kinds of goal live here, and they need different tools:

  1. Formalize the computation transformers/flows/diffusion models actually do — differentiable maps, composed, with the derivative flowing through by the chain rule already built (Analysis stage 6½). This is the “get the math right” half.
  2. Formalize the optimizations real implementations use — flash attention, KV-cache, and their kin all compute the same mathematical value as a naive reference implementation, but via a different, faster computation path. This is exactly the project’s refinement pattern (spec vs. Impl, proved equivalent) applied to performance instead of representation — the “get the equivalence proof right” half, and arguably the more novel one: nobody proves flash attention correct against a naive reference this way today.

Staged honestly by difficulty. The first three stages are cheap — they reuse machinery that already exists. The last two are genuinely hard new foundations (ODEs, stochastic processes) and are flagged as such rather than glossed over.

Stage A — Transcendental functions (blocking almost everything below)

Nothing in the tree defines exp, log, or any transcendental function — only field operations, order, and limits exist. Softmax, Gaussian densities, and the score function in diffusion models all need exp at minimum.

Stage B — Concrete differentiable layers (already queued, Analysis stage 6½)

  • [~] Softmax, matmul, ReLU, layernorm as is_deriv witnesses — softmax specifically needs Stage A’s exp. Matmul: done. Impl/MatrixNorm.v builds the entrywise L1 norm on Matrix m n F (no square root available in this tree, so not Frobenius — mat_norm A := Σ|A i j|); Theory/MatMulBilinear.v packages mat_mul as a BilinearMap, proves the SHARP submultiplicativity bound ‖mat_mul A B‖ <= ‖A‖ * ‖B‖ (C := 1, a Fubini-swap-and- factor argument), and gets mat_mul_is_deriv as one application of Theory/BilinearDeriv.v’s bilinear_is_deriv — every “multiply input by weights” transformer layer’s differentiability is now this theorem applied to its own weight matrix. Softmax, ReLU, layernorm still untouched.

Stage C — The optimization-equivalence pattern (the novel part)

This is the payoff stage: prove that a fast implementation computes the same value as a naive spec.

Stage F — ODEs, flows, and MeanFlow (genuinely hard, distant)

Flagged honestly as the least mature part of this roadmap. MeanFlow and continuous normalizing flows both need a vector field (a map assigning a tangent vector to each point) and its flow — the solution φ_t(x) of dφ/dt = V(φ_t(x)), φ_0 = id. Existence and uniqueness of that solution (Picard–Lindelöf) is real analysis depth this project hasn’t built any foundation for yet (no ODE theory exists anywhere in the tree).

Summary: what’s cheap vs. what’s hard

Cheap (reuses existing machinery, start anytime): Stage A’s exp interface, Stage B’s concrete layers, Stage D’s diffeomorphism definition, Stage E’s Riesz-gradient connection.

Medium (real proof work, but the pattern is known): Stage C’s flash-attention/KV-cache equivalence proofs, Stage D’s composability (blocked on finishing detM_mul), Stage E’s discrete-time denoising.

Hard (new foundations, no shortcuts): Stage F’s ODE existence theory; SDEs are explicitly not on this roadmap at all.
