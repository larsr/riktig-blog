Structure graph — the typeclass-inheritance hierarchy

The structure graph (SVG) — every node is a structural Class from Interface/*.v (Monoid, Group, AbGroup, Ring, Field, VectorSpace, InnerProduct, PosInner, NormedSpace, GeometricAlgebra, the Category/Functor family, …), clickable through to its coqdoc definition. An arrow points from a structure to the one it embeds (Group -> Monoid, AbGroup -> Group, Ring -> AbGroup, …) — the same reading as “Group embeds Monoid.”

Operational typeclasses (HasAdd, HasNeg, HasScale, EqRel, …) are deliberately absent: they carry one symbol each, not a structure, and including them would turn the picture into the same wiring diagram as the source instead of the hierarchy on top of it. The one bundle- style exception is ScaleLaws (the four scale laws), a property mixin in the same family as Assoc/Idl in Props.v, not a structure — excluded for the same reason.

Generated by blog/structure-graph.py (make structure-graph), from a scan of Interface/*.v’s Class declarations — never hand-maintained, so it can’t drift from the code.
