Bilinear maps, and why they’re automatically differentiable

A linear map between vector spaces over a field FF is a function that respects addition and scaling: f(u+u′)=f(u)+f(u′)f(u + u') = f(u) + f(u') and f(a⋅u)=a⋅f(u)f(a \cdot u) = a \cdot f(u), writing a⋅ua \cdot u for the scaling action of a scalar on a vector (the code writes it *:). Everything a linear map does is fixed by what it does to a basis, and it carries the zero vector to the zero vector. As always in this development, == is a setoid equality — a chosen equivalence relation every operation must respect — not equality of raw Coq terms.

This post is about the next step up: maps of two vector arguments that are linear in each one separately. They behave very differently from linear maps in one respect that turns out to matter for calculus, and the consequence is that establishing bilinearity is already enough to make such a map differentiable, with no further analysis to check by hand.

A bilinear map is linear in each slot separately

Record BilinearMap (F U V W : Type) {FF : Field F}
    {VU : VectorSpace F U} {VV : VectorSpace F V}
    {VW : VectorSpace F W} := {
  bm_fun :> U -> V -> W;
  #[global] bm_proper :: Proper (equ ==> equ ==> equ) bm_fun;
  bm_add_l : forall (u u' : U) (v : V),
    bm_fun (u + u') v == bm_fun u v + bm_fun u' v;
  bm_scale_l : forall (a : F) (u : U) (v : V),
    bm_fun (a *: u) v == a *: bm_fun u v;
  bm_add_r : forall (u : U) (v v' : V),
    bm_fun u (v + v') == bm_fun u v + bm_fun u v';
  bm_scale_r : forall (a : F) (u : U) (v : V),
    bm_fun u (a *: v) == a *: bm_fun u v
}.

The record is data — a choice of map bb together with proofs that it obeys four laws. Fix the second argument and it is linear in the first (bm_add_l, bm_scale_l); fix the first and it is linear in the second (bm_add_r, bm_scale_r). The extra bm_proper field says bb respects the setoid equality in both arguments.

Here is the one distinction to get right, because it is what makes bilinearity genuinely different from linearity. A bilinear map is not a linear map on the product space U×VU \times V. If it were, scaling the pair (u,v)(u,v) by aa would scale the output once, by aa. But scaling the pair scales both coordinates, and each scaling pulls out its own factor of aa:

b(a⋅u,a⋅v)=a⋅(a⋅b(u,v))=a2⋅b(u,v).b(a \cdot u,\; a \cdot v) = a \cdot \big(a \cdot b(u,v)\big) = a^2 \cdot b(u,v).

Scaling the input pair by aa scales the output by a2a^2, not by aa. Linear in each slot; quadratic on the pair. That single fact — that the output picks up the scalar twice — is exactly what will produce the quadratic remainder that makes the map differentiable.

One structure builds another: fixing a slot gives a linear map

Before the calculus, one clean algebraic consequence. If you hold the first argument of a bilinear map fixed at some uu, what remains is a function of one vector — and it is a genuine LinearMap, because bm_add_r and bm_scale_r are precisely the two linear-map laws for that function. The file packages exactly this:

  Definition bm_appl (u : U) : LinearMap F V W :=
    @Build_LinearMap F V W FF VV VW (fun v : V => b u v)
      (bm_appl_proper u)
      (bm_add_r b u)
      (fun (a : F) (v : V) => bm_scale_r b a u v).

Read the three proof arguments: the additivity law of the new linear map is bm_add_r b u — the right-slot additivity of bb with the first argument frozen at uu — and its scaling law is bm_scale_r b a u v, the right-slot scaling law, likewise frozen. Fixing a slot of a bilinear map hands you a linear map for free, with no new proof obligations: the laws you need are the laws you already have.

This immediately buys a fact that would otherwise take its own argument. A linear map sends the zero vector to the zero vector; so a bilinear map, fed the zero vector in either slot, returns zero.

The claim: bu0=0b\,u\,0 = 0 for every uu.

The blackboard argument is one sentence: v↦buvv \mapsto b\,u\,v is a linear map (that is bm_appl u), and every linear map kills zero (that is lm_zero), so bu0=0b\,u\,0 = 0. The Coq is that sentence, verbatim:

  Theorem bm_zero_r : forall u : U, b u 0 == 0.
  Proof.
    intros u.
    exact (lm_zero (bm_appl u)).
  Qed.

lm_zero (bm_appl u) is the general “linear maps send 00 to 00” lemma instantiated at the linear map you get by freezing the first slot at uu. Nothing about bilinearity had to be re-proved: the linear-map machinery applies the moment bm_appl exhibits the fixed-slot map as linear.

Bilinearity forces differentiability

Now the reason bilinear maps are worth isolating. Recall what it means for a function ff to be differentiable at a point aa with derivative a bounded linear map LL: the error left after the linear approximation, f(a+h)−f(a)−Lhf(a+h) - f(a) - L\,h, must shrink faster than ‖h‖\lVert h\rVert itself. Precisely, for every ε>0\varepsilon > 0 there is a δ>0\delta > 0 such that ‖f(a+h)−f(a)−Lh‖≤ε‖h‖\lVert f(a+h) - f(a) - L\,h\rVert \le \varepsilon\,\lVert h\rVert whenever ‖h‖<δ\lVert h\rVert < \delta. The derivative is the best linear map through the point; differentiability is the promise that the leftover is negligible next to hh.

Take a bilinear map B:U×V→WB : U \times V \to W, regarded as a function of the single pair argument p=(u,v)p = (u,v), and a base point (u0,v0)(u_0, v_0). The claimed derivative is the map sending a displacement (du,dv)(du, dv) to Bduv0+Bu0dvB\,du\,v_0 + B\,u_0\,dv — vary the first slot alone, plus vary the second slot alone:

  Definition bilinear_deriv_fun (p : U * V) : W :=
    B (fst p) v0 + B u0 (snd p).

This is a LinearMap F (U * V) W (its additivity and scaling laws follow from BB’s slot-wise laws — the file proves bilinear_deriv_add and bilinear_deriv_scale for exactly this). The question is whether it is the derivative: is the leftover quadratically small?

The remainder is a single cross term

The blackboard argument expands B(u0+du,v0+dv)B(u_0 + du,\; v_0 + dv) by bilinearity — right slot first, then each half in the left slot:

B(u0+du,v0+dv)=Bu0v0+Bduv0+Bu0dv+Bdudv.B(u_0 + du,\; v_0 + dv) = B\,u_0\,v_0 \;+\; B\,du\,v_0 \;+\; B\,u_0\,dv \;+\; B\,du\,dv.

Four terms. The first is the value Bu0v0=B(u0,v0)B\,u_0\,v_0 = B(u_0,v_0). The middle two are precisely the claimed linear approximation Bduv0+Bu0dvB\,du\,v_0 + B\,u_0\,dv. Subtract off the value and the linear part, and exactly one term survives — the cross term BdudvB\,du\,dv, the one where both displacements act. That is the quadratic term the “scales twice” observation predicts:

  Lemma bilinear_remainder_eq : forall (du : U) (dv : V),
    B (u0 + du) (v0 + dv) + - (B u0 v0 + (B du v0 + B u0 dv)) == B du dv.
  Proof.
    intros du dv.
    rewrite (bm_add_r B (u0 + du) v0 dv), (bm_add_l B u0 du v0),
      (bm_add_l B u0 du dv).
    exact (four_term_cancel (B u0 v0) (B du v0) (B u0 dv) (B du dv)).
  Qed.

Reading it: bm_add_r B (u0+du) v0 dv splits the right slot, turning B(u0+du)(v0+dv)B(u_0+du)(v_0+dv) into B(u0+du)v0+B(u0+du)dvB(u_0+du)\,v_0 + B(u_0+du)\,dv. The two bm_add_l rewrites then split the left slot of each piece, producing all four terms. four_term_cancel is a pure abelian-group rearrangement: (a+b+c+d)−(a+(b+c))=d(a + b + c + d) - (a + (b + c)) = d. What is left is BdudvB\,du\,dv, and nothing else.

The cross term is quadratically small

So differentiability comes down to: is ‖Bdudv‖\lVert B\,du\,dv\rVert small compared to ‖(du,dv)‖\lVert (du,dv)\rVert? On the product space we measure a pair by the sum norm, ‖(du,dv)‖=‖du‖+‖dv‖\lVert (du,dv)\rVert = \lVert du\rVert + \lVert dv\rVert; call this ‖h‖\lVert h\rVert. Assume BB is bounded — there is a constant C≥0C \ge 0 with ‖Buv‖≤C‖u‖‖v‖\lVert B\,u\,v\rVert \le C\,\lVert u\rVert\, \lVert v\rVert always (a hypothesis, exactly as for linear maps, not a field of the record). Since ‖du‖\lVert du\rVert and ‖dv‖\lVert dv\rVert are each at most their sum ‖h‖\lVert h\rVert,

‖Bdudv‖≤C‖du‖‖dv‖≤C‖h‖‖h‖.\lVert B\,du\,dv\rVert \le C\,\lVert du\rVert\,\lVert dv\rVert \le C\,\lVert h\rVert\,\lVert h\rVert.

This is quadratic in ‖h‖\lVert h\rVert, and that is why it is negligible: for a target ε\varepsilon, choosing the neighbourhood radius δ=ε⋅(1+C)−1\delta = \varepsilon \cdot (1 + C)^{-1} forces C‖h‖≤εC\,\lVert h\rVert \le \varepsilon whenever ‖h‖<δ\lVert h\rVert < \delta, which turns the quadratic bound C‖h‖⋅‖h‖C\,\lVert h\rVert \cdot \lVert h\rVert into the required ε⋅‖h‖\varepsilon \cdot \lVert h\rVert. The file proves exactly this padding estimate:

  Lemma bilinear_remainder_bound : forall C : F, 0 <= C ->
    (forall (u : U) (v : V), ‖ B u v ‖ <= C * ‖ u ‖ * ‖ v ‖) ->
    forall eps : F, 0 < eps ->
    forall (du : U) (dv : V),
      ‖ du ‖ + ‖ dv ‖ < eps * inv (1 + C) ->
      ‖ B du dv ‖ <= eps * (‖ du ‖ + ‖ dv ‖).

The proof is a chain of monotonicity steps: bound ‖Bdudv‖\lVert B\,du\,dv\rVert by C‖du‖‖dv‖C\,\lVert du\rVert\,\lVert dv\rVert (the boundedness hypothesis), raise each factor ‖du‖,‖dv‖\lVert du\rVert, \lVert dv\rVert to the sum ‖du‖+‖dv‖\lVert du\rVert + \lVert dv\rVert, and finally use ‖h‖<ε(1+C)−1\lVert h\rVert < \varepsilon\,(1+C)^{-1} to collapse one surviving factor of C‖h‖C\,\lVert h\rVert to ε\varepsilon. This is the one place where bilinearity demands genuine analysis rather than algebra — precisely because the remainder is the product of two small quantities, not the sum of two small quantities.

The theorem

Both halves assemble into the theorem. Differentiability in this development is a pair: the derivative is a bounded linear map, and the ε\varepsilon/δ\delta remainder estimate holds. The proof supplies one half from bilinear_deriv_bounded and the other from the two lemmas above:

  Theorem bilinear_is_deriv : bm_bounded B ->
    is_deriv (fun p : U * V => B (fst p) (snd p)) (u0, v0) bilinear_deriv_lm.
  Proof.
    intros Hbb.
    split.
    - exact (bilinear_deriv_bounded Hbb).
    - destruct Hbb as [C [HC HCb]].
      intros eps Heps.
      exists (eps * inv (1 + C)).
      split.
      + apply mul_pos.
        * exact Heps.
        * apply pos_inv. apply one_plus_nonneg_pos. exact HC.
      + intros [du dv] Hh.
        simpl.
        unfold lm_fun, bilinear_deriv_lm, bilinear_deriv_fun; simpl.
        calc (‖ B (u0 + du) (v0 + dv) + - (B u0 v0 + (B du v0 + B u0 dv)) ‖)
          == (‖ B du dv ‖)
          == (eps * ‖ (du, dv) ‖).
        * apply eq_le. apply norm_proper. exact (bilinear_remainder_eq du dv).
        * unfold vnorm, Prod_HasNorm, prod_norm; simpl.
          exact (bilinear_remainder_bound C HC HCb eps Heps du dv Hh).
  Qed.

Reading it: split breaks differentiability into its two obligations. The first, bilinear_deriv_bounded Hbb, discharges “the derivative is a bounded linear map.” For the second, unpacking the boundedness witness gives the constant CC; then for a given eps the proof offers the radius δ=ε(1+C)−1\delta = \varepsilon\,(1+C)^{-1} with exists (eps * inv (1 + C)), shows it is positive (mul_pos — a positive scalar times a positive inverse), and takes an arbitrary displacement (du, dv) inside that radius. The closing calc chain is the two-step estimate laid out above: the norm of the leftover equals the norm of the cross term BdudvB\,du\,dv (that is bilinear_remainder_eq, applied under norm_proper since equal vectors have equal norms), and that cross term is bounded by ε‖(du,dv)‖\varepsilon\,\lVert (du,dv)\rVert (that is bilinear_remainder_bound). The unfolding of prod_norm is where the sum norm ‖(du,dv)‖=‖du‖+‖dv‖\lVert (du,dv)\rVert = \lVert du\rVert + \lVert dv\rVert meets the shape the bound was proved in.

So bilinearity is not merely compatible with differentiability — it forces it. Once a map is shown linear in each slot and bounded, its derivative is written down explicitly (vary one slot at a time, add), and the error is a single cross term whose norm is a product of two small things, hence quadratically small. No limits are computed by hand; the one genuine analysis step is the padding estimate, and it is the same estimate for every bilinear map at once.
