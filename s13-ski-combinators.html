Combinators: eliminating variables with SKI

A lambda term is built from three things: a variable, an abstraction λ. body that binds a variable, and an application f a. We write variables as de Bruijn indices — a variable is a number counting how many binders you cross on the way out to the λ that introduced it, so λ.λ. 1 is “the function of two arguments returning the first.” Reduction is the single rule that applying a λ to an argument substitutes the argument for the bound variable, and two terms count as equal when reduction can bring them to a common form.

That one rule — substitution — is where all the fuss lives. You cannot just textually drop the argument in: a free variable in the argument can be accidentally captured by a binder it lands under, so real substitution has to renumber indices as it goes (the lifting bookkeeping). Every implementation of the lambda calculus pays this tax on every step.

Why remove variables at all

Here is the idea worth the trouble. Suppose we could rewrite every lambda term using only a small, fixed set of closed building blocks — constants that bind nothing and contain no free variables — combined by application alone. Then there are no bound variables anywhere, and so there is nothing to capture, nothing to renumber, nothing to lift. A term with no binders is trivially safe to copy, to reorder, to compile: substitution degenerates into “plug a subterm into a hole.” That is combinatory logic, and the classic building blocks are three combinators called S, K, and I.

Their reduction rules are just three rewrites:

Ix→xKxy→xSfgx→fx(gx)I\,x \;\to\; x \qquad K\,x\,y \;\to\; x \qquad S\,f\,g\,x \;\to\; f\,x\,(g\,x)

I is the identity; K throws its second argument away; S applies f and g to a shared x and combines the results. Remarkably, that is enough to express every lambda term. As a first taste, S K K behaves exactly like the identity I:

SKKx→Kx(Kx){S rule}→x{K rule}S\,K\,K\,x \;\to\; K\,x\,(K\,x) \quad\{S\text{ rule}\}\quad \to\; x \quad\{K\text{ rule}\}

The S rule shares x into both branches, then the outer K keeps the left branch and drops the right. In the Coq development, where the combinators are named CS, CK, CI (bare S and I would clash with the successor of a natural number and with the proof of True), this is checked by running a fuelled normalizer on an opaque argument CVar 0:

(* SKK = I, the classic warm-up. *)
Example nf_SKK : nf 10 (CS @@ CK @@ CK @@ CVar 0) = CVar 0.
Proof. reflexivity. Qed.

The @@ is application, and CVar 0 is an inert constant with no reduction rule of its own — it stands in for an arbitrary argument so the check cannot cheat by simplifying it.

The combinators, as a datatype

The whole syntax of combinatory logic is five constructors: the three combinators, the inert variable used for testing, and application.

Inductive comb : Type :=
  | CS : comb
  | CK : comb
  | CI : comb
  | CVar : nat -> comb
  | CApp : comb -> comb -> comb.

Notation "f @@ x" := (CApp f x) (at level 40, left associativity).

One-step reduction is the three contraction rules above, plus two congruence rules that let a step happen inside a left or right subterm:

Inductive cstep : comb -> comb -> Prop :=
  | cstep_S : forall f g x,
      cstep (CS @@ f @@ g @@ x) (f @@ x @@ (g @@ x))
  | cstep_K : forall x y,
      cstep (CK @@ x @@ y) x
  | cstep_I : forall x,
      cstep (CI @@ x) x
  | cstep_app1 : forall f f' a,
      cstep f f' ->
      cstep (f @@ a) (f' @@ a)
  | cstep_app2 : forall f a a',
      cstep a a' ->
      cstep (f @@ a) (f @@ a').

Write cstar for the reflexive-transitive closure of cstep: cstar t u means t reduces to u in zero or more steps.

Bracket abstraction: putting a variable back

To translate a lambda term into combinators we need to reverse a binder. Given a combinator term t that mentions the variable CVar 0, we want a closed combinator [0] t such that applying it to any argument u reproduces t with u in place of CVar 0. That operation is bracket abstraction, babs. The rules are the textbook ones, with no cleverness:

[x]x=I[x]y=Ky[x]c=Kc[x](MN)=S([x]M)([x]N)[x]\,x = I \qquad [x]\,y = K\,y \qquad [x]\,c = K\,c \qquad [x]\,(M\,N) = S\,([x]M)\,([x]N)

The variable being abstracted becomes I; anything not containing it (another variable, or a bare combinator c) is guarded by K so the argument is discarded; an application is split by S so the argument reaches both sides. In de Bruijn form, “the variable being abstracted” is index 0, and any other variable S n shifts down to n:

Fixpoint babs (t : comb) : comb :=
  match t with
  | CVar O => CI
  | CVar (S n) => CK @@ CVar n
  | CApp f a => CS @@ babs f @@ babs a
  | c => CK @@ c
  end.

The meaning we want babs to have is captured by a substitution csub u t: replace CVar 0 by u and shift every other variable down by one.

Fixpoint csub (u : comb) (t : comb) : comb :=
  match t with
  | CVar O => u
  | CVar (S n) => CVar n
  | CApp f a => csub u f @@ csub u a
  | c => c
  end.

The correctness of bracket abstraction

Here is the result that makes bracket abstraction trustworthy: for every term t and every argument u, applying the abstracted term to u reduces to the substitution.

([0]t)u→*t[u/0]([0]\,t)\;u \;\longrightarrow^{*}\; t[\,u/0\,]

Verbatim from the source:

Lemma babs_correct : forall t u, cstar (babs t @@ u) (csub u t).
Proof.
  induction t as [ | | | n | f IHf a IHa]; intros u; simpl.
  - eapply cstar_step. { apply cstep_K. } apply cstar_refl.
  - eapply cstar_step. { apply cstep_K. } apply cstar_refl.
  - eapply cstar_step. { apply cstep_K. } apply cstar_refl.
  - destruct n as [| n]; simpl.
    + eapply cstar_step. { apply cstep_I. } apply cstar_refl.
    + eapply cstar_step. { apply cstep_K. } apply cstar_refl.
  - eapply cstar_step. { apply cstep_S. }
    eapply cstar_trans. { apply cstar_app1. apply IHf. }
    apply cstar_app2. apply IHa.
Qed.

The proof is a structural induction on t, one branch per shape the term can have.

  • The first three branches are t = CS, t = CK, t = CI. For each, babs t = CK @@ t, so (CK @@ t) @@ u takes one K step to t, and csub u t = t because a bare combinator has no variables to substitute. Each branch is a single cstep_K followed by cstar_refl.
  • The CVar n branch splits on whether n is zero. If n = 0, then babs (CVar 0) = CI, so CI @@ u takes one I step to u, and csub u (CVar 0) = u — the two sides meet. If n = S n', then babs (CVar (S n')) = CK @@ CVar n', one K step to CVar n', matching csub u (CVar (S n')) = CVar n'.
  • The application branch t = f a is the only one that recurses. Here babs (f a) = CS @@ babs f @@ babs a, so cstep_S turns (CS @@ babs f @@ babs a) @@ u into (babs f @@ u) @@ (babs a @@ u). The two induction hypotheses IHf and IHa reduce the two halves to csub u f and csub u a; cstar_app1 applies the first inside the left of the application, cstar_app2 the second inside the right, and cstar_trans chains them. The result is csub u f @@ csub u a, which is exactly csub u (f a).

No side condition is needed: combinatory reduction places no restriction on when a redex may fire, so the per-redex law holds for every argument u.

Compiling a lambda term to combinators

With babs in hand, translating a lambda term is three lines: a variable becomes the corresponding combinator variable, an application becomes an application, and an abstraction becomes a bracket abstraction of its translated body.

Fixpoint translate (t : tm) : comb :=
  match t with
  | Var n => CVar n
  | Abs b => babs (translate b)
  | App f a => translate f @@ translate a
  end.

Take the smallest interesting closed term, k = λ.λ. 1 (the function of two arguments that returns the first). Working the translation by hand, outermost binder last:

λ.λ.1_↦[0]([0]1_){each binder becomes a bracket}=[0](K0_){inner bracket: guard and shift}=S(KK)I{outer bracket splits the application}\lambda.\lambda.\,\underline{1} \;\mapsto\; [0]\,\big([0]\,\underline{1}\big) \quad\{\text{each binder becomes a bracket}\}\quad =\; [0]\,(K\,\underline{0}) \quad\{\text{inner bracket: guard and shift}\}\quad =\; S\,(K\,K)\,I \quad\{\text{outer bracket splits the application}\}

Read left to right: the inner [0] turns CVar 1 into CK @@ CVar 0 (a non-zero variable, guarded by K and shifted down to CVar 0); the outer [0] sees an application CK @@ CVar 0, so it splits with S, sending babs CK = CK @@ CK to the left and babs (CVar 0) = CI to the right. The Coq development confirms the computed result exactly:

Example translate_k : translate k = CS @@ (CK @@ CK) @@ CI.
Proof. reflexivity. Qed.

And the translated term genuinely behaves like k: handed two opaque arguments, it returns the first, checked by running the normalizer.

Example translate_k_fst :
  cstar (translate k @@ CVar 0 @@ CVar 1) (CVar 0).
Proof. apply (nf_cstar 20). reflexivity. Qed.

The simplest case is simpler still — the identity λ.0 compiles to CI outright:

Example translate_id : translate idt = CI.
Proof. reflexivity. Qed.

babs_correct is the local guarantee behind these runs: it says a single translated redex steps to the right combinator-level substitution. The source stops short of the full simulation theorem (that every lambda reduction is mirrored by a combinator reduction), and says so plainly rather than leaving an Admitted behind — the general statement needs substitution-commutation lemmas over de Bruijn indices, and weak combinatory reduction does not mirror beta reduction underneath a binder. What is proved is the per-redex law, and it is enough to compile and run closed terms.
