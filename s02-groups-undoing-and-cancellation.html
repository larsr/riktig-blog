Groups: undoing, and the cancellation trick

A monoid is a set with an associative combining operation + and a do-nothing element 0 — for example the integers under addition, or lists under concatenation. What a monoid does not promise is that you can undo a combination. Adding 3 to a number can be reversed by adding -3; concatenating a list cannot be reversed at all. A group is a monoid where every element has an undo.

The group axioms

To “undo” a means to have an element that cancels it back to the do-nothing element 0. The library names the undo operation - (unary negation) and asks for two laws: negating and then adding gives 0, in either order.

(−a)+a=0,a+(−a)=0.(-a) + a \;=\; 0, \qquad a + (-a) \;=\; 0.

Here is the whole structure, from Interface/Group.v:

Class Group G := {
  #[global] grp_monoid     :: Monoid G;
  #[global] grp_neg        :: HasNeg G;
  grp_invl       : forall a : G, (- a) + a == 0;
  grp_invr       : forall a : G, a + (- a) == 0;
  #[global] grp_neg_proper :: Proper (equ ==> equ) (@neg G _)
}.

A group is a monoid (grp_monoid) with an added negation operation (grp_neg, from the HasNeg vocabulary class), plus the two undo laws grp_invl and grp_invr, and a proof that - respects the equality (grp_neg_proper). Because grp_monoid is marked with ::, everything proved about monoids — associativity, the identity laws, id_unique from the previous post — is available for a group with no restatement. The two undo laws are registered under the names invl and invr, which is how the proofs below refer to them.

Cancellation on the left

The first thing undo buys you is cancellation: if a + b == a + c, then b == c. In ordinary arithmetic this is obvious, but notice the proof uses nothing except the group laws — so it holds in every group, including ones where the elements are not numbers at all (matrices, symmetries, functions under composition). The idea is to add -a on the left of both sides and let the laws collapse it:

b=0+b=(−a+a)+b=−a+(a+b)=−a+(a+c)=(−a+a)+c=0+c=c.b \;=\; 0 + b \;=\; (-a + a) + b \;=\; -a + (a + b) \;=\; -a + (a + c) \;=\; (-a + a) + c \;=\; 0 + c \;=\; c.

Here is the proof verbatim from Theory/Group.v:

Theorem cancel_l : forall a b c : G, a + b == a + c -> b == c.
Proof.
  intros a b c Heq.
  calc b
    == (0 + b)          { idl  }
    == ((- a + a) + b)  { invl }
    == (- a + (a + b))  { assoc }
    == (- a + (a + c))  { Heq  }
    == ((- a + a) + c)  { assoc }
    == (0 + c)          { invl }
    == c                { idl  }.
Qed.

The calc block is the blackboard chain above, written out and machine-checked: it reads b == (0 + b) == ... == c, and each { ... } names the single law that turns one line into the next. Walking it step by step, each step matching one equality on the blackboard:

  • == (0 + b) { idl } — insert a 0 on the left, by the left-identity law idl read backwards.
  • == ((- a + a) + b) { invl } — expand that 0 into -a + a, the left-undo law invl.
  • == (- a + (a + b)) { assoc } — reassociate, so a + b appears as a block. This is associativity, assoc.
  • == (- a + (a + c)) { Heq } — replace a + b by a + c, the hypothesis Heq we were given.
  • == ((- a + a) + c) { assoc } — reassociate the other way, so -a + a is a block again.
  • == (0 + c) { invl } — collapse -a + a back to 0.
  • == c { idl } — drop the 0, and the goal b == c is reached.

The whole proof is the blackboard calculation, one law per line, with no appeal to what the elements actually are. Cancelling on the right is the mirror image — from b + a == c + a conclude b == c — and its proof is the same moves reflected, adding -a on the right and using invr/idr where the left version used invl/idl:

b=b+0=b+(a+−a)=(b+a)+−a=(c+a)+−a=c+(a+−a)=c+0=c.b \;=\; b + 0 \;=\; b + (a + -a) \;=\; (b + a) + -a \;=\; (c + a) + -a \;=\; c + (a + -a) \;=\; c + 0 \;=\; c.

Theorem cancel_r : forall a b c : G, b + a == c + a -> b == c.
Proof.
  intros a b c Heq.
  calc b
    == (b + 0)          { idr  }
    == (b + (a + - a))  { invr }
    == ((b + a) + - a)  { assoc }
    == ((c + a) + - a)  { Heq  }
    == (c + (a + - a))  { assoc }
    == (c + 0)          { invr }
    == c                { idr  }.
Qed.

Double negation

A short one that reuses cancellation. Negating twice returns the original element: - (- g) == g. The reason is that -g has exactly one undo, and both - - g and g qualify, since each adds to -g to give 0. To turn “both undo -g” into “they are equal”, cancel the shared -g:

(−g)+(−−g)=0=(−g)+g,(-g) + (--g) \;=\; 0 \;=\; (-g) + g,

and cancelling -g from the left leaves --g == g. In Coq, from Theory/Group.v:

Theorem neg_involutive : forall g : G, - (- g) == g.
Proof.
  intros g.
  apply (cancel_l (- g)).
  calc ((- g) + - - g)
    == (0 : G)      { (invr (- g)) }
    == ((- g) + g)  { (invl g)     }.
Qed.
  • apply (cancel_l (- g)) uses the theorem just proved: to show - - g == g, it is enough to show (- g) + - - g == (- g) + g. That is the new goal.
  • == (0 : G) { (invr (- g)) } — the left side (- g) + - - g is -g plus its own negation, which is 0 by the right-undo law invr at the element -g.
  • == ((- g) + g) { (invl g) } — and 0 is (- g) + g by the left-undo law invl at g. The two sides meet at 0, so they are equal and cancel_l finishes the job.

Any additive map preserves zero and negation, for free

The last section of Theory/Group.v proves something once that would otherwise be re-proved for every operation in the library. Take any map f between two groups that preserves addition — f (a + b) == f a + f b. Such a map automatically preserves 0 and negation as well, with no extra hypotheses. The setup fixes f and its two properties:

  Context (f : G -> K).
  Context (fp : Proper (equ ==> equ) f).
  Context (fadd : forall a b : G, f (a + b) == f a + f b).

fp says f respects equality; fadd is the addition-preserving law. That f 0 == 0 then follows. The argument: f 0 added to itself equals f 0, and the only element that is unchanged by adding f 0 is 0, so cancel one copy of f 0:

f0+f0=f(0+0)=f0=f0+0.f\,0 + f\,0 \;=\; f(0 + 0) \;=\; f\,0 \;=\; f\,0 + 0.

  Theorem hom_zero : f 0 == 0.
  Proof.
    apply (cancel_l (f 0)).
    calc (f 0 + f 0)
      == (f (0 + 0)) { (fadd 0 0) }
      == (f 0)       { idl }
      == (f 0 + 0)   { idr }.
  Qed.

Notice this proof reuses cancel_l, proved two sections up: apply (cancel_l (f 0)) reduces the goal f 0 == 0 to f 0 + f 0 == f 0 + 0. Then the chain: f 0 + f 0 == f (0 + 0) by fadd (backwards); f (0 + 0) == f 0 because 0 + 0 == 0 by the left-identity law idl inside f; and f 0 == f 0 + 0 by the right-identity law idr. Cancelling the shared f 0 leaves f 0 == 0. The companion lemma hom_neg, with statement forall a : G, f (- a) == - f a, preserves negation, and it in turn uses hom_zero.

Seeing it run on the integers

The point of proving hom_zero abstractly is that every additive map inherits it. Multiplication by a fixed integer is such a map: 3 * (a + b) == 3 * a + 3 * b is the distributive law. So multiply-by-3 on the integers preserves 0 and negation for free — no separate proof, just an instantiation of hom_zero and hom_neg:

Definition triple (x : Z) : Z := (3 * x)%Z.

Lemma triple_proper : Proper (equ ==> equ) triple.
Proof. intros x y Hxy. unfold triple. rewrite Hxy. reflexivity. Qed.

Lemma triple_add : forall a b : Z, triple (a + b) == triple a + triple b.
Proof. intros a b. unfold triple. exact (Z.mul_add_distr_l 3 a b). Qed.

(* hom_zero, instantiated: 3 * 0 == 0, with no separate proof *)
Definition triple_zero : triple 0 == 0 := hom_zero triple triple_proper triple_add.

triple_zero is hom_zero applied to triple — the fact that 3 * 0 == 0 obtained purely from the abstract theorem. And because the integers compute, it is not only provable but calculable:

Example triple_zero_compute : triple 0 = 0%Z.
Proof. reflexivity. Qed.

Example triple_neg_compute : triple (- 5) = ((- (triple 5)))%Z.
Proof. reflexivity. Qed.

The second reflects hom_neg: tripling -5 gives the negation of tripling 5, both sides reducing to -15. The general theorem and the concrete computation agree, as they must.
