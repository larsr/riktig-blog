New vector spaces from old: pointwise, products, and spaces of maps

A vector space over a field FF is a set whose elements you can add (with a zero and negatives, addition commutative and associative) and scale by an element of FF, subject to four compatibility laws: scaling distributes over vector sums and over scalar sums, two scalings compose by multiplying the scalars, and scaling by 11 changes nothing. That is the whole definition — no basis, no coordinates. Throughout, == is a setoid equality: a chosen equivalence relation that every operation must respect, not Coq’s built-in equality of terms.

Here is the observation this whole post turns on. Once you have one vector space, a whole family of others arrives with essentially no new work, because each one borrows its operations and its laws from a space you already have. Functions into a vector space, pairs of vector spaces, the trivial one-point space, a field regarded as a space over itself, and even the linear maps between two spaces — every one of them is a vector space, and in each case the proof is the same short move: do the operation at each point (or each component), and every law holds because it already holds underneath.

Functions into a space, pointwise

Fix any type X and any vector space V. The functions X -> V form a vector space. There is nothing to invent: the operations act at each argument, and equality is agreement at every argument.

#[global] Instance PW_EqRel {X M : Type} `{EqRel M} : EqRel (X -> M) :=
  fun f g => forall x, f x == g x.

#[global] Instance PW_Zero {X M : Type} `{Zero M} : Zero (X -> M) :=
  fun _ => 0.

#[global] Instance PW_Add {X M : Type} `{HasAdd M} : HasAdd (X -> M) :=
  fun f g x => f x + g x.

#[global] Instance PW_Neg {X M : Type} `{HasNeg M} : HasNeg (X -> M) :=
  fun f x => - f x.

#[global] Instance PW_Scale {X F V : Type} `{HasScale F V} : HasScale F (X -> V) :=
  fun a f x => a *: f x.

The zero function returns V’s zero everywhere; the sum f + g is the function x↦f(x)+g(x)x \mapsto f(x) + g(x); scaling a *: f is x↦a⋅f(x)x \mapsto a \cdot f(x), writing ⋅\cdot for the scaling *:. Now take one of the four scaling laws and watch it fall out point by point. Scaling distributes over a sum of functions:

(a⋅(u+v))(x)=a⋅(u(x)+v(x)){the pointwise definitions of + and ⋅}=a⋅u(x)+a⋅v(x){the same law one level down: distributivity in V, at x}=(a⋅u+a⋅v)(x){the pointwise definitions again.} \begin{aligned} (a \cdot (u + v))(x) &= a \cdot \big(u(x) + v(x)\big) && \{\text{the pointwise definitions of } + \text{ and } \cdot\}\\ &= a \cdot u(x) + a \cdot v(x) && \{\text{the same law one level down: distributivity in } V \text{, at } x\}\\ &= (a \cdot u + a \cdot v)(x) && \{\text{the pointwise definitions again.}\} \end{aligned}

In Coq the whole thing is: fix a point, and appeal to the V-level law there.

Lemma pw_scale_add : forall (a : F) (u v : X -> V),
  a *: (u + v) == a *: u + a *: v.
Proof. intros a u v x. apply sc_scale_add. Qed.

intros a u v x fixes the scalar, the two functions, and — crucially — a single argument x, because equality of functions here is equality at every x. Once x is fixed both sides are ordinary elements of V, and sc_scale_add is exactly V’s law a⋅(u+v)=a⋅u+a⋅va \cdot (u + v) = a \cdot u + a \cdot v. One line, because the work was already done in V.

The other laws are identical in shape (apply sc_add_scale, apply sc_scale_scale, apply sc_scale_1, each after intros ... x), so the full vector-space structure assembles with no genuinely new argument anywhere:

#[global] Instance PW_VectorSpace : VectorSpace F (X -> V) :=
  @Build_VectorSpace F (X -> V) _ PW_AbGroup PW_Scale PW_ScaleLaws.

Because the construction is itself generic, it iterates: functions into functions into V are a vector space too, found by instance search with no extra code, which the file records as a check —

Example pw_vs_nested (F V : Type) (HF : Field F) (HV : VectorSpace F V) :
  VectorSpace F (nat -> bool -> V) := _.

The := _ means Coq is asked to find the structure on nat -> bool -> V; it succeeds by peeling one arrow, then the next, then landing on V.

The same move, four more times

The remaining constructions differ only in what “at each point” means.

Products. For vector spaces A and B, the pairs A * B form a vector space with everything acting componentwise — (a1,b1)+(a2,b2)=(a1+a2,b1+b2)(a_1, b_1) + (a_2, b_2) = (a_1 + a_2,\ b_1 + b_2) and a⋅(u,v)=(a⋅u,a⋅v)a \cdot (u, v) = (a \cdot u,\ a \cdot v) — and equality the conjunction of the two componentwise equalities. Each law splits into its first and second component and is discharged by the corresponding law in A and in B (intros [u1 u2] ...; split; apply ...).

The one-point space. unit, with its single inhabitant tt, is a vector space over any field. The cleanest equality is the relation that holds of every pair — so every law is literally a proof of True:

Lemma unit_scale_add : forall (a : F) (u v : unit),
  a *: (u + v) == a *: u + a *: v.
Proof. intros a u v. exact I. Qed.

This is the zero-dimensional space, the neutral element for products.

A field over itself. A field F is a vector space over F where scaling is just multiplication. There is no separate structure to check: the four scaling laws are the ring laws F already satisfies, handed over verbatim.

#[global] Instance Field_Scale : HasScale F F := mult.

#[global] Instance Field_ScaleLaws : ScaleLaws F F :=
  @Build_ScaleLaws F F _ _ Field_Scale
    field_sc_proper ring_distl ring_distr ring_mul_assoc ring_mul_1l.

Read the second definition slot by slot: distributivity of scaling over a vector sum is ring_distl, scaling by 11 is ring_mul_1l, and so on. No new proofs, just a re-labelling of laws F already had.

Linear maps. The most striking case: the linear maps from U to V — the functions respecting ==, sums, and scaling — are themselves a vector space, Hom U V. Add two maps pointwise, scale a map pointwise, and each result is again linear because the laws hold in V at every point; the vector-space laws for Hom U V then reduce, once more, to the laws in V:

Lemma hom_scale_add : forall (a : F) (f g : Hom),
  a *: (f + g) == a *: f + a *: g.
Proof.
  intros a f g u.
  apply sc_scale_add.
Qed.

The same two-line rhythm — introduce a point u, appeal to the V-level law — now applied to a space whose elements are transformations. A space of maps is a space in its own right.

When are two of these “the same” space?

These constructions produce many spaces that are visibly different as data yet ought to count as the same space. A * unit carries a useless second component; A * B and B * A differ only in the order of a pair; and — the case worth the name — a function returning a record, X -> A * B, holds exactly the same information as a pair of functions, (X -> A) * (X -> B).

The right notion of sameness for vector spaces is an isomorphism: a linear map one way, a linear map back, and a proof that each undoes the other. In the code that is a four-field record:

Record VSIso (F U V : Type) {FF : Field F}
    {VU : VectorSpace F U} {VV : VectorSpace F V} := {
  vsi_to : LinearMap F U V;
  vsi_from : LinearMap F V U;
  vsi_fromto : forall u : U, vsi_from (vsi_to u) == u;
  vsi_tofrom : forall v : V, vsi_to (vsi_from v) == v
}.

The two LinearMap fields are the maps; the last two fields are the round-trip laws — from(to(u))=u\mathrm{from}(\mathrm{to}(u)) = u and to(from(v))=v\mathrm{to}(\mathrm{from}(v)) = v: go over and back and you are where you started, in either direction. Anything true of one carrier transports to the other by rewriting along these two laws.

The main isomorphism is the record/column one,

X→A×B≅(X→A)×(X→B).X \to A \times B \;\;\cong\;\; (X \to A) \times (X \to B).

Splitting f into its two columns goes forward, zipping a pair of functions back into one goes backward, and both round-trips hold at every point:

Lemma soa_fromto : forall f : X -> A * B,
  (fun x : X => (fst (f x), snd (f x))) == f.
Proof.
  intros f x.
  split.
  - reflexivity.
  - reflexivity.
Qed.

Splitting f into its first and second components and re-pairing them at each x rebuilds f on the nose — the split breaks the pair equality into its two halves, each an identity. The full isomorphism bundles that with the other round-trip and the two linear maps:

Definition vsiso_soa :
  @VSIso F (X -> A * B) ((X -> A) * (X -> B)) FF VXAB VCols :=
  @Build_VSIso F (X -> A * B) ((X -> A) * (X -> B)) FF VXAB VCols
    soa_split soa_merge soa_fromto soa_tofrom.

Take X to be Fin n, the type of positions 0, …, n-1. Then X -> A * B is an array of structs — n slots, each a record — and (X -> A) * (X -> B) is a struct of arrays — one array of the A fields beside one array of the B fields. vsiso_soa is a machine-checked certificate that these two memory layouts hold the same vector, so a program may switch between them freely. The same file proves the small structural isomorphisms too — dropping a unit factor, swapping a pair, reassociating a nested product — and shows them composing, so a regrouping like A×(B×𝟏)≅B×AA \times (B \times \mathbf{1}) \cong B \times A is assembled from those pieces rather than proved by hand.
