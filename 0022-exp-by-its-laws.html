riktig #22 — exp, by its laws

Twenty-second post documenting the riktig project. A short one: the first transcendental function enters the tree, and it enters as a spec, not a formula.

What we are doing, and why. Verified software in Coq, built for modifiability. Post #21 proved backprop and forward-mode AD compute the same Jacobian — the first payoff of the ML roadmap (post #20-and- a-half’s docs/ML-ROADMAP.md). Today is the roadmap’s Stage A: exp, the function nearly everything downstream needs — softmax, Gaussian densities, diffusion’s score function — and, until now, a function that simply didn’t exist anywhere in this codebase.

What was missing

Search the tree before today and you find field operations, order, limits — and no exp, no log, nothing transcendental. That’s not an oversight so much as an honest gap: every course builds exp from a power series or a limit, both of which need real analysis machinery (convergence, completeness) this project has been careful to introduce only as needed. But softmax needs exp today, and the project’s own rule for exactly this situation already exists — write the spec first, defer the construction:

Class Exp (F : Type) `{OrderedField F} := {
  exp_fn        : F -> F;
  #[global] exp_fn_proper :: Proper (equ ==> equ) exp_fn;

  exp_zero : exp_fn 0 == 1;
  exp_add  : forall a b : F, exp_fn (a + b) == exp_fn a * exp_fn b;
  exp_pos  : forall a : F, 0 <= exp_fn a;
  exp_mono : forall a b : F, a < b -> exp_fn a < exp_fn b
}.

Four laws — the Proper field is setoid plumbing, recording that exp_fn respects == so it can be rewritten under, not a law about exp itself. exp is a monotone homomorphism from the additive group (F,+,0)(F,+,0) to the multiplicative monoid (F,×,1)(F,\times,1), landing in the nonnegative numbers. Nothing here says how to compute it — a power-series or limit-based construction is separate, later work, the same relationship Interface/Reals.v has to its eventual Cauchy-sequence build. Anything satisfying these four laws is legally “an exp” for every theorem downstream, whether it’s ever been evaluated.

What four laws buy for free

Small, cheap corollaries, all proved generically — no construction, no concrete F, just algebra:

Lemma exp_neg : forall a : F, exp_fn (- a) == inv (exp_fn a).
Lemma exp_nonzero : forall a : F, exp_fn a =/= 0.
Lemma exp_strict_pos : forall a : F, 0 < exp_fn a.
Lemma exp_injective : forall a b : F, exp_fn a == exp_fn b -> a == b.

The nonvanishing fact is the nicest of these, because it comes free from a law that looks unrelated: exp a * exp (-a) == exp 0 == 1 exhibits exp a as invertible directly, so if it were 0 the field would be forced to satisfy 0 == 1 — impossible in any field worth the name. No case split, no separate axiom; the homomorphism law alone makes exp a a unit.

exp_injective is the more interesting one to sit with, because it’s half of the fact that makes log possible at all: exp : F -> F is a bijection onto the positive elements. Injectivity — different inputs give different outputs — falls out of strict monotonicity with no extra assumptions. Surjectivity — every positive element is some exp a — is the harder half, and it needs completeness (the least- upper-bound property Interface/Reals.v states but doesn’t yet build a witness for). So log waits, honestly, for that piece; today’s file gets you exactly as far as the algebra allows and stops there rather than assuming ahead.

The one thing deliberately left out

Every calculus class’s favorite fact about exp — that it’s its own derivative — is not in this file, and the reason is worth stating plainly rather than glossing over. To even write down “exp′ = exp” you need exp_fn : F -> F to be checked against Theory/Derivative.v’s is_deriv, which asks the codomain to carry a NormedSpace structure. Nobody has yet built the norm that turns a bare OrderedField into a normed space over itself (the obvious candidate is the field’s own abs, but that instance doesn’t exist in the tree today). So rather than force that construction into this file — or worse, quietly skip the fact and hope nobody notices — the file ends with a precise TODO naming exactly what’s missing and sketching the class that will carry it once it exists:

Class ExpDeriv (F : Type) `{OrderedField F}
    {NSF : NormedSpace F F} `{Exp F} := {
  exp_is_deriv : forall a : F,
    is_deriv exp_fn a (lm_scale (exp_fn a))
}.

kept separate from Exp itself deliberately: softmax never differentiates, so it shouldn’t have to drag a NormedSpace-over- itself instance along for the ride just to typecheck.

Where this sits on the roadmap

docs/ML-ROADMAP.md staged the path to transformers, flash attention, and diffusion by difficulty; Stage A was flagged as blocking almost everything below it, and cheap — reusing machinery that already exists rather than building new foundations. That held: four laws, a handful of one-paragraph corollaries, no new proof techniques. Softmax is next.

Next: Wave L5’s determinant multiplicativity — the one remaining hard blocker, now under a fresh attempt via a Leibniz permutation-sum reformulation rather than the row-swap route that stalled twice — or, if that lands first, Stage B’s concrete differentiable layers.
