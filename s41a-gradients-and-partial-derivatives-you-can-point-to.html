Partial derivatives, the gradient, and the Jacobian: the chain rule is matrix multiplication

A function of one variable changes at one rate: at a point aa, a single number f′(a)f'(a) says how fast the output moves when the input does. A function of several inputs and several outputs has no single such number. Move the first input and every output responds at its own rate; move the second input and all the outputs respond at different rates again. A map from nn inputs to mm outputs carries m×nm \times n rates of change at every point — one per input-output pair. Three questions come with that picture. What single object records all those rates at once, so that “the derivative at aa” still means one thing? What exactly is one of the individual rates — a partial derivative — and why is it well defined? And when two such functions are composed, why do their tables of rates combine by the strange-looking row-times-column rule of matrix multiplication, rather than entry by entry?

This post answers all three with one idea: the derivative at a point is a linear map, the partial derivatives are that map’s values on basis vectors, the Jacobian is that map’s matrix once bases are chosen — and composing linear maps is what matrix multiplication computes, so the chain rule for Jacobians is bookkeeping, not new analysis.

The derivative at a point is a linear map

Recalled briefly from earlier in this series: a normed space is a vector space over an ordered field FF together with a norm ‖⋅‖\lVert\cdot\rVert measuring the size of a vector, and a linear map LL is a function between vector spaces with L(u+v)=L(u)+L(v)L(u + v) = L(u) + L(v) and L(cu)=cL(u)L(c\,u) = c\,L(u). The derivative of ff at aa is the linear map LL that best approximates ff near aa: the error of the approximation f(a+h)≈f(a)+L(h)f(a + h) \approx f(a) + L(h) shrinks faster than hh itself,

‖f(a+h)−f(a)−L(h)‖≤ε‖h‖for every ε>0, once ‖h‖<δ(ε). \lVert f(a+h) - f(a) - L(h) \rVert \;\le\; \varepsilon\,\lVert h \rVert \qquad \text{for every } \varepsilon > 0, \text{ once } \lVert h \rVert < \delta(\varepsilon).

In Coq that is a definition, not a metaphor. Throughout, == is the equivalence relation each type carries (for the rationals it is “same value,” not syntactic equality), and lm_bounded L says there is a constant CC with ‖L(u)‖≤C‖u‖\lVert L(u)\rVert \le C\,\lVert u \rVert for all uu — the continuity condition a linear map needs in an infinite-dimensional space, harmless in a finite-dimensional one:

Definition is_deriv (f : U -> V) (a : U) (L : LinearMap F U V) : Prop :=
  lm_bounded L /\
  forall eps : F, 0 < eps ->
    exists delta : F, 0 < delta /\
      forall h : U, ‖ h ‖ < delta ->
        ‖ f (a + h) + - (f a + L h) ‖ <= eps * ‖ h ‖.

Read it back against the problem: f is the function of several variables, a the point, and L the one object that records all the rates of change at once — feed L a displacement h in any direction and it returns the first-order response of every output simultaneously. Nothing in the definition mentions coordinates, columns, or entries.

Two facts about this definition are proved where it lives and are used below. First, the derivative is unique: if is_deriv f a L1 and is_deriv f a L2 then L1 == L2 (theorem deriv_unique). Second, the derivative of a composite is the composite of the derivatives — the coordinate-free chain rule:

Theorem deriv_comp : forall (f : U -> V) (g : V -> W) (a : U)
    (Lf : LinearMap F U V) (Lg : LinearMap F V W),
  Proper (equ ==> equ) g ->
  is_deriv f a Lf -> is_deriv g (f a) Lg ->
  is_deriv (fun u : U => g (f u)) a (lm_comp Lg Lf).

Here lm_comp Lg Lf is the composition Lg∘LfL_g \circ L_f as a linear map, and Proper (equ ==> equ) g says g sends equivalent arguments to equivalent results — a side condition the proof needs in order to move g across an equivalence of arguments. The analytic content is the expected two-part estimate: the composite’s error splits into gg’s own error at the increment f(a+h)−f(a)f(a+h) - f(a), plus LgL_g applied to ff’s error, and both parts are of order ‖h‖\lVert h \rVert because each factor’s error is and each linear map is bounded. That proof was walked in the earlier derivative post; this post consumes it.

A partial derivative is the derivative applied to a basis vector

Coordinates enter through a basis. Recalled from the basis post: a Basis n F U packages nn basis vectors basis_vec B : Fin.t n -> U (where Fin.t n is the type of indices 0,…,n−10,\dots,n-1) together with a linear coordinate map basis_coords B : LinearMap F U (Fin.t n -> F), and two round-trip laws saying that building a vector from its coordinates and reading coordinates off a built vector both give back what they started from.

The classical ∂f/∂xi\partial f/\partial x_i at aa is the rate of change of ff along the ii-th coordinate direction — that is, the response of ff’s best linear approximation to the ii-th basis vector. So once the derivative-as-a-linear-map exists, the partial derivative is not a new limit to define; it is an application:

Definition partial_at {n : nat} (L : LinearMap F U V) (B : Basis n F U)
    (i : Fin.t n) : V :=
  L (basis_vec B i).

Each argument answers to the problem statement. L is the total derivative (the object recording all rates); B is the choice of directions worth naming — partial derivatives do not exist until someone picks coordinates, and the definition makes that dependence a visible argument instead of a convention; i says which direction; and the result is a vector of V: the first-order response of all the outputs to a unit step along direction ii. Note that ∂if(a)\partial_i f(a) is a value in the codomain, not a number — the numbers appear only after the codomain, too, gets coordinates.

For this to deserve the notation ∂if(a)\partial_i f(a) the answer must not depend on which derivative witness was used.

Statement. Any two derivatives of ff at aa have the same partial derivatives.

On the blackboard: derivatives are unique, so this is one application of uniqueness. If L1L_1 and L2L_2 both satisfy the error estimate at aa, take any uu and any ε>0\varepsilon > 0, and test both estimates (each run at ε/2\varepsilon/2) at the scaled displacement h=tuh = t\,u for a small t>0t > 0. The triangle inequality sandwiches the difference of the two linear maps between the two errors:

t‖L1(u)−L2(u)‖=‖L1(h)−L2(h)‖≤‖f(a+h)−f(a)−L2(h)‖+‖f(a+h)−f(a)−L1(h)‖≤εt‖u‖, t\,\lVert L_1(u) - L_2(u) \rVert \;=\; \lVert L_1(h) - L_2(h) \rVert \;\le\; \lVert f(a{+}h) - f(a) - L_2(h) \rVert + \lVert f(a{+}h) - f(a) - L_1(h) \rVert \;\le\; \varepsilon\, t\, \lVert u \rVert ,

and cancelling tt leaves ‖L1(u)−L2(u)‖≤ε‖u‖\lVert L_1(u) - L_2(u)\rVert \le \varepsilon\,\lVert u\rVert. A nonnegative quantity below ε‖u‖\varepsilon\,\lVert u\rVert for every positive ε\varepsilon is zero, so L1(u)=L2(u)L_1(u) = L_2(u) at every uu — in particular at each basis vector:

Lemma partial_at_welldef : forall (n : nat) (f : U -> V) (a : U)
    (L1 L2 : LinearMap F U V) (B : Basis n F U) (i : Fin.t n),
  is_deriv f a L1 -> is_deriv f a L2 ->
  partial_at L1 B i == partial_at L2 B i.
Proof.
  intros n f a L1 L2 B i HD1 HD2.
  exact (deriv_unique f a L1 L2 HD1 HD2 (basis_vec B i)).
Qed.

The walk is one step: deriv_unique delivers L1 == L2, which for linear maps means “equal at every argument,” and the proof term applies that pointwise fact at the argument basis_vec B i — exactly the blackboard’s “in particular at each basis vector.”

The Jacobian is the matrix of the derivative

The earlier representation post established that a matrix is not a definition of a linear map but a description of one: given a basis BU of the domain (size nn) and a basis BV of the codomain (size mm), the matrix of a linear map has, in column jj, the BV-coordinates of the map’s value on the jj-th domain basis vector —

Definition mat_of (f : LinearMap F U V) : Matrix m n F :=
  fun i j => basis_coords BV (f (basis_vec BU j)) i.

— and this reading-off is a bijection between linear maps and m×nm \times n matrices (a Matrix m n F is literally a function Fin.t m -> Fin.t n -> F, a rectangle of scalars). The Jacobian of ff at aa is nothing beyond this: the matrix, in the chosen bases, of the derivative at aa.

Definition jacobian {f : U -> V} {a : U} {L : LinearMap F U V}
    (Hd : is_deriv f a L) : Matrix m n F :=
  mat_of BU BV L.

The argument Hd deserves a pause: the matrix is computed from L alone, but the definition takes the evidence Hd : is_deriv f a L rather than a bare linear map, so that “the Jacobian of f at a” can only be formed for an L that actually is the derivative — and by deriv_unique, any two admissible Ls give the same matrix up to ==.

Statement. Column jj of the Jacobian consists of the codomain coordinates of the jj-th partial derivative.

On the blackboard this is the definition meeting the definition: entry (i,j)(i,j) of the matrix of LL is coordinate ii of L(bj)L(b_j), and L(bj)L(b_j) is the jj-th partial derivative, so

(Jf(a))ij=(coords(L(bj)))i=(coords(∂jf(a)))i. \big(J_f(a)\big)_{ij} \;=\; \Big( \mathrm{coords}\big( L(b_j) \big) \Big)_i \;=\; \Big( \mathrm{coords}\big( \partial_j f(a) \big) \Big)_i . 

Theorem jacobian_col : forall {f : U -> V} {a : U} {L : LinearMap F U V}
    (Hd : is_deriv f a L) (i : Fin.t m) (j : Fin.t n),
  jacobian Hd i j == basis_coords BV (partial_at L BU j) i.
Proof.
  intros f a L Hd i j.
  reflexivity.
Qed.

The proof is reflexivity and that is the point: both sides unfold to the same expression basis_coords BV (L (basis_vec BU j)) i, one through the definition of mat_of and one through the definition of partial_at. The calculus notation (“the Jacobian’s columns are the partial derivatives”) and the linear-algebra notation (“the matrix’s columns are the images of the basis vectors, in coordinates”) are the same sentence.

Two corollaries of the picture cost nothing. First, the gradient: when ff is scalar-valued — codomain FF itself, viewed as a one-dimensional space over itself — the Jacobian is a 1×n1 \times n matrix. Its single row, with entries ∂f/∂x1,…,∂f/∂xn\partial f/\partial x_1, \dots, \partial f/\partial x_n, is what calculus calls the gradient of ff at aa (often displayed as a column by transposing). No separate definition is needed: the gradient is the Jacobian at m=1m = 1. Second, a function that is already linear (and bounded) is its own derivative everywhere, so its Jacobian at every point is just its own matrix:

Example jacobian_of_linear : forall (g : LinearMap F U V)
    (Hb : lm_bounded g) (a : U),
  jacobian (deriv_lm g a Hb) == mat_of BU BV g.
Proof.
  intros g Hb a.
  reflexivity.
Qed.

Again reflexivity: jacobian discards the evidence and applies mat_of to the linear map inside, and deriv_lm g a Hb (the theorem that a bounded linear map is its own derivative at any point) carries g itself as that linear map.

The chain rule is matrix multiplication

Now the third question: why does composing functions multiply matrices?

Statement. The Jacobian of g∘fg \circ f at aa is the matrix product of the Jacobian of gg at f(a)f(a) and the Jacobian of ff at aa.

Jg∘f(a)=Jg(f(a))Jf(a). J_{g \circ f}(a) \;=\; J_g\big(f(a)\big)\; J_f(a).

The reasoning splits into one analysis fact and one algebra fact, both already on the table. Analysis: the derivative of the composite is the composition of the derivatives — that is deriv_comp above, and it is where all the ε\varepsilon–δ\delta work lives. Algebra: writing maps in bases turns composition into matrix product,

[[Lg∘Lf]]=[[Lg]][[Lf]],entrywise[[Lg∘Lf]]ij=∑k[[Lg]]ik[[Lf]]kj, \big[\!\big[\, L_g \circ L_f \,\big]\!\big] \;=\; \big[\!\big[\, L_g \,\big]\!\big]\, \big[\!\big[\, L_f \,\big]\!\big], \qquad\text{entrywise}\quad \big[\!\big[\, L_g \circ L_f \,\big]\!\big]_{ij} = \sum_{k} \big[\!\big[\, L_g \,\big]\!\big]_{ik}\, \big[\!\big[\, L_f \,\big]\!\big]_{kj},

where [[⋅]][\![\,\cdot\,]\!] is the matrix of a map in the chosen bases. That algebra fact is the representation theorem’s composition law, proved in the earlier matrices post; its Coq name and statement:

Theorem mat_of_comp : forall (g : LinearMap F V W) (f : LinearMap F U V),
  mat_of BU BW (lm_comp g f)
    == mat_mul (mat_of BV BW g) (mat_of BU BV f).
Proof.
  intros g f i j.
  exact (mat_of_coords BV BW g (f (basis_vec BU j)) i).
Qed.

One sentence of walk, since it is consumed twice below: entry (i,j)(i,j) of the left side is coordinate ii of g(f(bj))g(f(b_j)), and the computation rule mat_of_coords — coordinates of g(v)g(v) equal the matrix of gg applied to the coordinates of vv — read at v=f(bj)v = f(b_j) produces exactly the sum ∑k[[g]]ik[[f]]kj\sum_k [\![g]\!]_{ik}\, [\![f]\!]_{kj}, which is entry (i,j)(i,j) of the right side.

With both facts in place, the chain rule for Jacobians is their juxtaposition:

Jg∘f(a)=[[Lg∘Lf]]=[[Lg]][[Lf]]=Jg(f(a))Jf(a), J_{g\circ f}(a) \;=\; \big[\!\big[\, L_g \circ L_f \,\big]\!\big] \;=\; \big[\!\big[\, L_g \,\big]\!\big]\, \big[\!\big[\, L_f \,\big]\!\big] \;=\; J_g\big(f(a)\big)\, J_f(a),

the first equality by deriv_comp (plus uniqueness: Lg∘LfL_g \circ L_f is the composite’s derivative), the second by mat_of_comp, the third by the definition of the two factor Jacobians. In Coq the statement comes in two flavours. The engine version makes the derivative hypotheses explicit and speaks mat_of:

Theorem jacobian_comp : forall (f : U -> V) (g : V -> W) (a : U)
    (Lf : LinearMap F U V) (Lg : LinearMap F V W),
  is_deriv f a Lf -> is_deriv g (f a) Lg ->
  is_deriv (fun u : U => g (f u)) a (lm_comp Lg Lf) ->
  mat_of BU BW (lm_comp Lg Lf)
    == mat_mul (mat_of BV BW Lg) (mat_of BU BV Lf).
Proof.
  intros f g a Lf Lg Hf Hg Hgf.
  exact (mat_of_comp BU BV BW Lg Lf).
Qed.

And the readable version speaks jacobian directly, using deriv_comp to manufacture the composite’s derivative evidence rather than assuming it:

Theorem jacobian_chain_rule : forall (f : U -> V) (g : V -> W) (a : U)
    (Lf : LinearMap F U V) (Lg : LinearMap F V W)
    (Pg : Proper (equ ==> equ) g)
    (Hf : is_deriv f a Lf) (Hg : is_deriv g (f a) Lg),
  jacobian n p BU BW (deriv_comp f g a Lf Lg Pg Hf Hg)
    == mat_mul (jacobian m p BV BW Hg) (jacobian n m BU BV Hf).
Proof.
  intros f g a Lf Lg Pg Hf Hg.
  exact (mat_of_comp BU BV BW Lg Lf).
Qed.

Walking jacobian_chain_rule clause by clause: Hf and Hg are the two derivatives being consumed — ff differentiable at aa with derivative Lf, gg differentiable at f(a)f(a) with derivative Lg — and Pg is the same respectful-of-equivalence side condition deriv_comp always needs. The term deriv_comp f g a Lf Lg Pg Hf Hg in the statement is the composite’s derivative evidence, so the left side is the Jacobian of g∘fg \circ f at aa, taken between the outer bases BU and BW. The right side is the matrix product of the two factor Jacobians, with the middle basis BV appearing once in each factor — the shared index kk of the entrywise sum ranges over the middle space’s basis. The proof is a single exact: after jacobian unfolds to mat_of, both sides are precisely the two sides of mat_of_comp applied to Lg and Lf. The derivative hypotheses are never touched by the proof — their role was to pin down which linear maps get represented, and all remaining content is the algebra of representation. That is the answer to the third question: matrices multiply because composition of linear maps, pushed through coordinates, is the row-times-column sum — and the chain rule is composition.

Running it: partial derivatives with visible numbers

The definitions above compute. The concrete stage is the coordinate plane Fin.t 2 -> Q — pairs of rationals indexed by Fin.F1 (first) and Fin.FS Fin.F1 (second) — with its standard basis FinVec_Basis 2, whose basis vectors are the Kronecker tuples ei(j)=1e_i(j) = 1 if i=ji = j, else 00, and whose coordinate map is the identity. A matrix acts on coordinates through mat_vec_lm A, the linear map v↦Avv \mapsto A\,v. The examples below live at the end of the Jacobian source file and compile with the library; 2#1 is the rational 2/12/1, and vm_compute asks Coq to fully evaluate both sides before comparing.

Take the linear map f(x,y)=(2x+3y,5x+7y)f(x,y) = (2x + 3y,\; 5x + 7y) and, for composing, g(u,v)=(u+v,v)g(u,v) = (u + v,\; v):

Definition Af : Matrix 2 2 Q := matQ2 (2#1) (3#1) (5#1) (7#1).
Definition Bg : Matrix 2 2 Q := matQ2 (1#1) (1#1) (0#1) (1#1).

(matQ2 a11 a12 a21 a22 builds the evident Matrix 2 2 Q row by row.) Since ff is linear it is its own derivative everywhere, so its partial derivatives should be the columns of its matrix: ∂f/∂x=(2,5)\partial f/\partial x = (2, 5) and ∂f/∂y=(3,7)\partial f/\partial y = (3, 7). They are, by evaluation:

Example partial_x_of_f :
     partial_at (mat_vec_lm Af) (FinVec_Basis 2) Fin.F1 Fin.F1 == (2#1)%Q
  /\ partial_at (mat_vec_lm Af) (FinVec_Basis 2) Fin.F1 (Fin.FS Fin.F1)
       == (5#1)%Q.
Proof. split; vm_compute; reflexivity. Qed.

Example partial_y_of_f :
     partial_at (mat_vec_lm Af) (FinVec_Basis 2) (Fin.FS Fin.F1) Fin.F1
       == (3#1)%Q
  /\ partial_at (mat_vec_lm Af) (FinVec_Basis 2) (Fin.FS Fin.F1)
       (Fin.FS Fin.F1) == (7#1)%Q.
Proof. split; vm_compute; reflexivity. Qed.

Each conjunct is one number: partial_at applies the map to a basis vector, the sum ∑jAije1(j)\sum_j A_{ij}\, e_1(j) collapses to a column entry, and vm_compute reduces both sides to literal rationals that reflexivity compares. Assembling all four entries with mat_of against the standard bases reads the whole matrix back — the Jacobian matrix of the linear map ff is its own matrix, entry by entry:

Example jacobian_matrix_of_f :
     mat_of (FinVec_Basis 2) (FinVec_Basis 2) (mat_vec_lm Af) Fin.F1 Fin.F1
       == (2#1)%Q
  /\ mat_of (FinVec_Basis 2) (FinVec_Basis 2) (mat_vec_lm Af) Fin.F1
       (Fin.FS Fin.F1) == (3#1)%Q
  /\ mat_of (FinVec_Basis 2) (FinVec_Basis 2) (mat_vec_lm Af)
       (Fin.FS Fin.F1) Fin.F1 == (5#1)%Q
  /\ mat_of (FinVec_Basis 2) (FinVec_Basis 2) (mat_vec_lm Af)
       (Fin.FS Fin.F1) (Fin.FS Fin.F1) == (7#1)%Q.
Proof. repeat split; vm_compute; reflexivity. Qed.

For the chain rule with numbers: composing gives g(f(x,y))=(7x+10y,5x+7y)g(f(x,y)) = (7x + 10y,\; 5x + 7y), so the composite’s xx-partial should be (7,5)(7, 5) — and that should equal the first column of the matrix product BgAfB_g A_f. The general theorem instantiates in one line, and the numbers check by evaluation:

Example chain_rule_concrete :
  mat_of (FinVec_Basis 2) (FinVec_Basis 2)
      (lm_comp (mat_vec_lm Bg) (mat_vec_lm Af))
    == mat_mul (mat_of (FinVec_Basis 2) (FinVec_Basis 2) (mat_vec_lm Bg))
         (mat_of (FinVec_Basis 2) (FinVec_Basis 2) (mat_vec_lm Af)).
Proof.
  exact (mat_of_comp (FinVec_Basis 2) (FinVec_Basis 2) (FinVec_Basis 2)
           (mat_vec_lm Bg) (mat_vec_lm Af)).
Qed.

Example partial_x_of_g_after_f :
     partial_at (lm_comp (mat_vec_lm Bg) (mat_vec_lm Af)) (FinVec_Basis 2)
       Fin.F1 Fin.F1 == (7#1)%Q
  /\ partial_at (lm_comp (mat_vec_lm Bg) (mat_vec_lm Af)) (FinVec_Basis 2)
       Fin.F1 (Fin.FS Fin.F1) == (5#1)%Q
  /\ mat_mul Bg Af Fin.F1 Fin.F1 == (7#1)%Q
  /\ mat_mul Bg Af (Fin.FS Fin.F1) Fin.F1 == (5#1)%Q.
Proof. repeat split; vm_compute; reflexivity. Qed.

One boundary of the demonstration should be stated plainly. Everything above the norm — partial_at, mat_of, mat_mul, the whole basis-and-matrix layer — needs only a field, vector spaces, and bases, which is why these examples reduce to literal rationals. But is_deriv itself mentions a norm, and the library currently defines no norm instance on the carrier Fin.t n -> Q (its matrix-norm file records that gap explicitly), so the statements that mention is_deriv — jacobian, jacobian_chain_rule — are exercised here at the abstract level, as in jacobian_of_linear, rather than re-instantiated at Q. For the linear maps in this section that costs nothing: a linear map is its own derivative, so the columns computed by partial_at are its partial derivatives in exactly the sense the rest of the post proves.
