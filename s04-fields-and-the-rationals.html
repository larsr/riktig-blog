Fields, and why Q is more interesting than it looks

A ring is a set you can add, subtract, and multiply in: an abelian group under + (with negation - a and zero 0), together with an associative multiplication * that has a unit 1 and distributes over addition, a * (b + c) == a * b + a * c. What a ring does not promise is division. This post adds it — carefully, because division cannot be total — and then shows that the most instructive example, the rationals, forces a change in what the equality sign == even means.

As before, == is the structure’s own equivalence relation, and a =/= b abbreviates ~ (a == b). The calculational proofs read as chains: calc x == y { law } == z { law } means x equals y by law, which equals z by law, with each brace naming the one fact that justifies its step.

The field axioms

A field is a ring in which multiplication commutes and every nonzero element has a multiplicative inverse.

Class Field F := {
  #[global] field_ring       :: Ring F;
  field_mul_comm   : forall a b : F, a * b == b * a;
  #[global] field_inv        :: HasInv F;
  (* field_invl carries a nonzero side condition, so it does not fit
     Props.v's Invl (unconditional, like Group's negation) — it stays
     named directly, not bound to a generic property class. *)
  field_invl       : forall a : F, a =/= 0 -> inv a * a == 1;
  #[global] field_inv_proper :: Proper (equ ==> equ) (@inv F _);
  field_nontrivial : (1 : F) =/= 0
}.

field_mul_comm supplies exactly the commutativity a general ring withholds. field_inv provides an inverse function inv, and field_invl is its law. field_nontrivial rules out the one-element ring where 0 and 1 coincide (and where “every nonzero element is invertible” would be vacuous).

Why the inverse law has a side condition

Compare the inverse law of a field with the negation law of a group. In a group, - a is defined for every element and (- a) + a == 0 holds for every element, no exceptions. That unconditional shape is what lets the group’s negation register as a generic “left inverse” instance, usable by name anywhere.

A field’s inverse cannot be like that. inv is a total function on the type — Coq requires inv 0 to be some value — but the law inv a * a == 1 cannot hold at a = 0. In any nontrivial ring x * 0 == 0 for every x, and 0 =/= 1, so no x satisfies x * 0 == 1; zero simply has no inverse. The law must therefore carry the hypothesis a =/= 0, as field_invl does. That extra hypothesis makes it a genuinely different shape of statement from the group’s unconditional inverse — which is why, as the source comment records, field_invl stays a plainly named axiom rather than being bound to the generic Invl class the way group negation is. The asymmetry is not an accident of the encoding; it is the mathematical fact that subtraction is total and division is not.

Cancellation

The first thing an inverse buys is cancellation: if a is nonzero, a * b == a * c forces b == c. On the blackboard the trick is to multiply through by inv a and let it annihilate the a:

b=1⋅b=(a−1a)b=a−1(ab)=a−1(ac)=(a−1a)c=1⋅c=cb \;=\; 1\cdot b \;=\; (a^{-1}a)\,b \;=\; a^{-1}(ab) \;=\; a^{-1}(ac) \;=\; (a^{-1}a)\,c \;=\; 1\cdot c \;=\; c

Theorem mul_cancel_l : forall a b c : F,
  a =/= 0 -> a * b == a * c -> b == c.
Proof.
  intros a b c Ha Heq.
  calc b
    == (1 * b)             { idl }
    == ((inv a * a) * b)   { (field_invl a Ha) }
    == (inv a * (a * b))   { (assoc (inv a) a b) }
    == (inv a * (a * c))   { Heq }
    == ((inv a * a) * c)   { (assoc (inv a) a c) }
    == (1 * c)             { (field_invl a Ha) }
    == c                   { idl }.
Qed.

Walking the seven steps: idl is the left-identity law, and because the goal is about 1 * b it resolves to multiplication’s identity 1 * b == b (the same operation-chosen-by-the-goal mechanism the ring post used). field_invl a Ha rewrites 1 as inv a * a, the one place the nonzero hypothesis Ha is spent. assoc (inv a) a b reassociates (inv a * a) * b to inv a * (a * b), tucking the pair a * b together. That pair is exactly what the hypothesis Heq rewrites, turning a * b into a * c. Then assoc reassociates the other way, field_invl a Ha folds inv a * a back to 1, and idl strips it. The proof spends the nonzero hypothesis twice, once to introduce inv a * a and once to remove it, with Heq applied in the sheltered middle.

The rationals, where equality is not equality

Now the point of the title. The rationals Q are the field everyone learns first, and they are more subtle than they look, because a rational number has many names: 1/2 and 2/4 are the same number written two ways. A fraction is a pair (numerator, denominator), and two different pairs can denote one value.

The standard library represents a rational literally as such a pair, written n # d, and its equality Qeq is not structural: Qeq declares a # b and c # d equal exactly when a * d = c * b as integers. So 1 # 2 and 2 # 4 are different data — different pairs of integers — but Qeq-equal, because 1 * 4 = 2 * 2. The field of rationals is built on Q by taking == to be Qeq:

#[global] Instance Q_EqRel : EqRel Q := Qeq.

This one line is the design decision the whole post is about. A rational number is an equivalence class of fractions — many representatives, one meaning — and choosing Qeq as == makes the formalization say exactly that. Structural equality would be the wrong mathematics: it would call 1/2 and 2/4 different, and no amount of later proof could repair the mistake. The price of the right choice is that every operation must be shown to respect the equivalence — the Proper fields in the ring and field classes — so that replacing a fraction by an equal one never changes a result. Those obligations are discharged by the standard library’s own registered morphisms for Qplus, Qmult, and the rest, reused verbatim.

Two concrete facts from the file make this tangible. First, the generic ring theory lands on a rational with no extra work — mul_0_l, the 0 * a == 0 lemma proved once for every ring, holds of Q up to Qeq:

Example q_mul_0_l : (0 * 7 == 0)%Q.
Proof. exact (mul_0_l 7%Q). Qed.

Second, because == on Q is decidable, one can compute whether two fractions denote the same rational. Applying the decision procedure dec_eq to the two spellings 1 # 1 and 2 # 2 of the number one:

Example q_dec_eq_check : {(1#1)%Q == (2#2)%Q} + {(1#1)%Q =/= (2#2)%Q} :=
  dec_eq (1#1)%Q (2#2)%Q.

The type of this example is a constructive decision between “these two distinct pairs are equal” and “they are not”, and the two pairs 1 # 1 and 2 # 2 are equal under Qeq because 1 * 2 = 2 * 1. The equality that a beginner takes for granted when they cancel 2/2 to 1 is, here, a fact the machine settles by cross-multiplying — and it is the same equality mul_cancel_l and every other field theorem quietly respects.
