riktig #20 — A = QR, for real

Twentieth post documenting the riktig project; the linear-algebra course continues. A short one: one theorem, earned mostly by reuse.

What we are doing, and why. Verified software in Coq, built for modifiability. Post #16 built Gram-Schmidt: turn a family of vectors into an orthogonal one by subtracting, at each step, the projections onto everything already processed. Today that construction gets packaged the way every linear algebra course eventually packages it — as a matrix factorization, A=QRA = QR — and the proof that the packaging is correct turns out to need almost no new mathematics, because the mathematics was already done.

What the pieces mean

Take nn vectors, the columns of a matrix AA. QQ’s columns are those same vectors, orthogonalized by Gram-Schmidt. RR records, for each original column, how to rebuild it from QQ’s columns — and because Gram-Schmidt only ever subtracts earlier projections, RR is upper triangular: column jj of AA only ever needs QQ’s columns 11 through jj.

Theorem qr_factorization : forall (v : Fin.t n -> V),
  (forall k, norm2 (gs_fin v k) =/= 0) ->
  qr_A B v == mat_mul (qr_Q B v) (qr_R v).

Theorem qr_R_upper : forall i j,
  fin_to_nat j < fin_to_nat i -> qr_R v i j == 0.

qr_R_upper is a genuine theorem, not something true by definition — it says the entries above the diagonal are provably zero, which is the honest content of “upper triangular.”

The one surprise: R’s diagonal isn’t the norm

Every textbook picture has RR’s diagonal holding the lengths of the orthogonalized vectors, so that QQ ends up orthonormal. Here it doesn’t — and the reason is worth sitting with, because it’s not a bug, it’s what the construction is actually doing. Post #16’s Gram-Schmidt never normalizes; QQ’s columns are orthogonal, not unit-length (normalizing needs a square root, and this project doesn’t have one yet — it’s deliberately deferred to the interval-refinement track, where “square root” gets an honest constructive meaning). Since QQ’s jj-th column is the raw Gram-Schmidt output, reconstructing AA’s jj-th column from it needs coefficient exactly 11 on the matching column — not the norm, which would only be the right coefficient if QQ had already been normalized to length one. So:

Theorem qr_R_diag_nonzero : forall i, qr_R v i i =/= 0.

proved unconditionally — no hypothesis needed, because the diagonal is forced to be the multiplicative identity, and 1≠01 \ne 0 is just what it means to be a field. A stronger, cleaner fact than the textbook version, and it falls out for free from being honest about what “orthogonal, not orthonormal” QR actually looks like.

Why the proof was mostly reuse

The interesting engineering wasn’t the theorem — it was fitting post #16’s list-based Gram-Schmidt to matrices, which need columns indexed by Fin n, not an unindexed list. The tempting route — convert the indexed family to a list, run gs, convert back — turns out to lose exactly the information the orthogonality proof needs: list-level Gram-Schmidt’s own orthogonality theorem only concludes “orthogonal, or the same vector” (a disjunction), because a list forgets which position produced which output. Matrices need to know which column. So the fix re-expresses Gram-Schmidt directly at Fin-indexed families and proves, as a bridge lemma, that this re-expression computes the same thing post #16’s construction does — letting every hard-won property from that file (orthogonality, the projection-subtraction structure) transfer across untouched. The representation changed; the theorems didn’t need to be re-proved. That sentence could be this whole project’s motto.
