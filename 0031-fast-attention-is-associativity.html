riktig #31 — fast attention is one associativity away

Thirty-first post in the series documenting the riktig project.

What we are doing, and why. We are building software whose correctness is proved in Coq. Post #30 showed that a matrix applied to a coordinate vector is a linear operator, and that multiplying matrices is composing operators (mat_vec_mul) — so the order in which a chain of matrix products is evaluated is an algorithm choice that cannot change the value. Today that choice with money on it: a linear-attention layer computed left to right materializes an n×n matrix and costs 2n²d multiplications; computed right to left it materializes only d×d and costs 2nd². The two are equal by one application of associativity — and both the equality and the costs are machine-checked.

The layer, honestly framed

An attention layer in a transformer takes three matrices of shape n × d — queries Q, keys K, values V, where n is the number of tokens and d the head dimension, with d much smaller than n — and computes softmax(Q·Kᵀ/√d)·V. What is formalized here is linear attention: the same shape with the softmax dropped, (Q·Kᵀ)·V. That is not a simplification made quietly for the proof’s sake; it is the whole point. Softmax is not linear, sits between the two products, and makes the reassociation below invalid — which is precisely why the fast-attention line of work (linear attention and its kernel-approximation relatives) removes or linearly approximates it, buying the right to reassociate. Nothing about softmax is defined or proved in this file; the header of Impl/Attention.v says so in as many words.

Two evaluation orders, one value

(* Left-to-right: materializes the n×n score matrix Q·Kᵀ.           *)
Definition attn_naive : Matrix n d F := mat_mul (mat_mul Q (mat_tr K)) V.

(* Right-to-left: materializes only the d×d matrix Kᵀ·V.            *)
Definition attn_fast : Matrix n d F := mat_mul Q (mat_mul (mat_tr K) V).

Theorem attn_assoc : attn_naive == attn_fast.
Proof.
  exact (mat_mul_assoc n d n d Q (mat_tr K) V).
Qed.

That is the entire proof: associativity of matrix multiplication, proved once by a Fubini argument in Impl/Matrix.v, instantiated at the shapes (n×d)·(d×n)·(n×d). The naive order’s intermediate Q·Kᵀ has n² entries; the fast order’s Kᵀ·V has d² — and d ≤ n gives d² ≤ n² (intermediate_smaller). At n = 1024 tokens and d = 64 that is a million-entry score matrix against four thousand.

The costs, machine-checked

Counting is proof too. mulcost m n p := m*n*p counts the scalar multiplications the bigopF definition of an m×n by n×p product performs — m·p entries, each a sum of n products. (A statement about the algorithm the definition denotes; Coq’s own evaluator is not being benchmarked.) Then:

Definition cost_naive (n d : nat) : nat :=
  (mulcost n d n + mulcost n n d)%nat.    (* = 2·n²·d *)

Definition cost_fast (n d : nat) : nat :=
  (mulcost d n d + mulcost n d d)%nat.    (* = 2·n·d² *)

Lemma cost_fast_le : forall n d : nat,
  (d <= n)%nat -> (cost_fast n d <= cost_naive n d)%nat.

The inequality is nonlinear in n and d, so it is assembled from the monotonicity lemmas rather than handed to lia. At transformer scale the counts overflow what unary nat arithmetic can evaluate in memory, so the concrete corollaries go through two proved transfer lemmas into binary N — and there the numbers are checked by reflexivity:

Example cost_naive_1024_64 : N.of_nat (cost_naive 1024 64) = 134217728%N.
Example cost_fast_1024_64  : N.of_nat (cost_fast 1024 64)  = 8388608%N.

134 million multiplications against 8 million — a factor of exactly n/d = 16.

For a single vector: no product at all

Post #30’s mat_vec_mul, applied twice, turns the whole layer into a pipeline of three matrix-vector applications:

Theorem attn_pipeline : forall w : Fin.t d -> F,
  mat_vec attn_naive w == mat_vec Q (mat_vec (mat_tr K) (mat_vec V w)).

Feed a coordinate vector through V, then Kᵀ, then Q: no matrix product is formed anywhere — not even the d×d one.

Checked on actual numbers

The file closes with a concrete instance over Q (n = 3, d = 2, with small integer matrices). Both association orders are evaluated independently, entry by entry, against the hand-computed output

(QK𝖳)V=Q(K𝖳V)=(97233−2)(Q K^{\mathsf T})V \;=\; Q(K^{\mathsf T}V) \;=\; \begin{pmatrix} 9 & 7 \\ 2 & 3 \\ 3 & -2 \end{pmatrix}

each closed by reflexivity with no appeal to attn_assoc — the theorem is then applied to the same instance as a separate resolution check. The pipeline is likewise computed on a concrete vector, both ways, meeting at (23, 8, -1).

What was proved, plainly

  • attn_naive == attn_fast for every field, every n, d, and every Q, K, V (attn_assoc) — the reassociation at the heart of linear attention is exact, not approximate.
  • cost_fast n d <= cost_naive n d whenever d <= n (cost_fast_le), with the concrete 16× count at n = 1024, d = 64 verified by computation; the fast order’s intermediate is smaller (intermediate_smaller).
  • Applied to one vector, the layer needs no matrix product at all (attn_pipeline).
  • Not proved, honestly: anything about softmax (its presence breaks the reassociation — that is why fast attention drops it), anything about memory hierarchies or actual hardware, and the cost model describes the denoted algorithm, not Coq’s evaluation.
