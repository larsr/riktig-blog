Liveness for the alternating-bit protocol: one surviving copy per burst is enough

The alternating-bit protocol tags each message with one bit, flipping it each time it moves on to the next message, so that a receiver expecting a particular bit can tell a fresh message from a stale retransmission over a channel that both loses and duplicates packets. The safety theorem for that protocol says: whatever the receiver delivers is a subsequence of what the application sent — no message is corrupted, duplicated, or reordered on its way out. That theorem holds unconditionally, with no assumption about the channel’s behaviour at all.

And that is exactly its limit. A channel that quietly drops every single packet satisfies every hypothesis of the safety theorem: it delivers the empty list, and the empty list is a subsequence of anything. Safe, and useless. Safety promises nothing wrong comes out; it does not promise anything comes out. This post is about the missing half — a precise statement of what it takes for messages to actually arrive, and the theorem that follows from it.

You cannot prove arrival from nothing

There is no getting around this by being cleverer with the proof. The drops-everything channel is a real behaviour of the lossy-and-duplicating relation the protocol runs over, and against it no protocol delivers anything. So “everything sent is delivered” is simply false as a statement about all channels in that class. If we want a theorem that concludes delivery, we must add a hypothesis that excludes the adversary — a promise that the channel does not lose everything, that it is in some sense fair.

The fairness hypothesis, stated precisely

Fairness here is finitary and local: it asks that, burst by burst, at least one copy of each message’s burst survives the channel. It does not ask for a lossless channel — copies may still be lost and duplicated freely within a burst — only that the burst is not lost entirely. Here is the hypothesis, from Channel/Fairness.v, defined by recursion on the message list in lock-step with how the sender lays out its bursts:

Fixpoint fair_wire (b : bit) (ms : list M) (k : nat) (wire : list Packet) : Prop :=
  match ms with
  | [] => wire = []
  | m :: ms' =>
      exists w1 w2,
        wire = w1 ++ w2 /\
        lossy_dup (sender_burst b m k) w1 /\
        w1 <> [] /\
        fair_wire (negb b) ms' k w2
  end.

Read it against the sender’s own structure. For an empty message list, the wire is empty. For m :: ms', the wire splits as w1 ++ w2, where w1 is what the channel produced from m’s burst — related to that burst by the same lossy_dup relation as always, so arbitrary loss and duplication within the burst are still permitted — and w2 is what it produced from the rest of the stream, recursively fair, with the bit flipped (negb b). The one new clause, the whole of fairness, is w1 <> []: the fragment for m’s burst is nonempty. At least one copy of each message gets through.

Two things are worth noticing about this being an assumption, not a derived fact. First, it constrains the channel: it is a stronger condition than plain lossy_dup, picking out the fair channels from the adversarial ones. Second, it is genuinely satisfiable only when the sender actually sends copies — if the budget k = 0, every burst is empty, lossy_dup cannot conjure a packet from an empty input, so no nonempty w1 can be lossy_dup-related to an empty burst, and fair_wire becomes unsatisfiable for a nonempty message list. That is the right degenerate behaviour: with nothing sent, fairness has nothing to promise.

To check that this hypothesis is not an unrelated model bolted onto the side, one lemma confirms it refines the relation safety already reasons about — a fair wire is in particular a lossy_dup wire of the sender’s whole stream,

fairb(ms,k,w)⇒lossy_dup(sent(b,ms,k),w),\mathrm{fair}_b(ms, k, w) \;\Longrightarrow\; \mathrm{lossy\_dup}\bigl(\mathrm{sent}(b, ms, k),\; w\bigr),

proved by induction on ms, concatenating the per-burst lossy_dup fragments back together:

Lemma fair_wire_lossy_dup : forall (k : nat) (ms : list M) (b : bit) (wire : list Packet),
  fair_wire b ms k wire -> lossy_dup (sender_stream b ms k) wire.
Proof.
  intros k ms.
  induction ms as [| m ms' IH]; intros b wire H.
  - simpl in H. rewrite H. simpl. apply lossy_dup_nil.
  - simpl in H. destruct H as [w1 [w2 [Hwire [H1 [Hne H2]]]]].
    rewrite Hwire. simpl.
    apply lossy_dup_app.
    + exact H1.
    + apply (IH (negb b) w2 H2).
Qed.

So fair_wire says everything lossy_dup said, plus the per-burst survival clause. It is the same channel, with fairness added on top, not a different channel.

One surviving copy delivers exactly one message

The engine of the liveness proof is the burst-level counterpart of the safety proof’s crux lemma. There, a burst delivered at most one message. Here, given that the burst’s fragment is nonempty, it delivers exactly one — the message — and flips the expected bit. The reasoning is the same homogeneity argument: every surviving packet is the same record {| pkt_bit := b; pkt_msg := m |}, so if any survive, the first matches the expected bit b, delivers m, and flips the expectation away from all the rest. Writing De(w)D_e(w) for what the receiver delivers from ww starting at expectation ee and Ne(w)N_e(w) for its final expectation,

lossy_dup(burstb(m,k),w)∧w≠[]⇒Db(w)=[m]∧Nb(w)=¬b.\mathrm{lossy\_dup}\bigl(\mathrm{burst}_b(m,k),\, w\bigr) \;\wedge\; w \neq [\,] \;\Longrightarrow\; D_b(w) = [m] \;\wedge\; N_b(w) = \neg b.

Lemma burst_delivery_fair : forall (b : bit) (m : M) (k : nat) (w : list Packet),
  lossy_dup (sender_burst b m k) w -> w <> [] ->
  deliver b w = [m] /\ next_bit b w = negb b.
Proof.
  intros b m k w H Hne.
  assert (Hall : forall p, In p w -> p = {| pkt_bit := b; pkt_msg := m |}).
  { apply (lossy_dup_preserves_all_eq {| pkt_bit := b; pkt_msg := m |}
             (sender_burst b m k) w H).
    intros p Hp. apply (sender_burst_all_eq b m k p Hp). }
  destruct w as [| p w'].
  - destruct (Hne eq_refl).
  - assert (Hp : p = {| pkt_bit := b; pkt_msg := m |}).
    { apply Hall. simpl. left. reflexivity. }
    rewrite Hp. simpl.
    rewrite (eqb_bit_true b b eq_refl). simpl.
    assert (Hall' : forall q, In q w' -> q = {| pkt_bit := b; pkt_msg := m |}).
    { intros q Hq. apply Hall. simpl. right. exact Hq. }
    assert (Hneq : pkt_bit ({| pkt_bit := b; pkt_msg := m |}) <> negb b).
    { simpl. intros Heq. apply (negb_neq b). symmetry. exact Heq. }
    destruct (deliver_homog_neq {| pkt_bit := b; pkt_msg := m |} w' (negb b) Hall' Hneq)
      as [Hd Hn].
    rewrite Hd. rewrite Hn.
    split; reflexivity.
Qed.

Walking it:

  • Hall is homogeneity again: every packet in the received fragment w is the record {| pkt_bit := b; pkt_msg := m |}, from lossy_dup_preserves_all_eq fed sender_burst_all_eq.
  • destruct w. The empty case contradicts the nonemptiness hypothesis Hne directly (Hne eq_refl) — this is the one place the survival clause is consumed at the burst level.
  • In the p :: w' case, p is the record (from Hall), its bit is b, so eqb_bit_true fires the match: deliver emits pkt_msg = m and recurses with the flipped bit negb b.
  • The tail w' is still all copies of the same record (Hall'), whose bit b no longer matches the flipped expectation negb b (Hneq), so deliver_homog_neq says the tail delivers nothing and leaves the bit at negb b. Result: deliver b w = [m] and next_bit b w = negb b.

The liveness theorem, proved and walked

Now the induction over the message list, carrying the running bit exactly as the safety proof did. The claim is the strongest possible one: over a fair wire, the receiver delivers the message list exactly — not merely a subsequence of it, but all of it, in order:

fairb(ms,k,w)⇒Db(w)=ms.\mathrm{fair}_b(ms, k, w) \;\Longrightarrow\; D_b(w) = ms.

Lemma abp_liveness_gen : forall (k : nat) (ms : list M) (b : bit) (wire : list Packet),
  fair_wire b ms k wire -> deliver b wire = ms.
Proof.
  intros k ms.
  induction ms as [| m ms' IH]; intros b wire H.
  - simpl in H. rewrite H. reflexivity.
  - simpl in H. destruct H as [w1 [w2 [Hwire [H1 [Hne H2]]]]].
    rewrite Hwire.
    rewrite (deliver_app w1 w2 b).
    destruct (burst_delivery_fair b m k w1 H1 Hne) as [Hd Hn].
    rewrite Hd. rewrite Hn. simpl.
    rewrite (IH (negb b) w2 H2).
    reflexivity.
Qed.

Walking it:

  • Base case, ms = []. Fairness forces wire = [], and deliver b [] = [], which is ms.
  • Inductive case, ms = m :: ms'. Unfolding fair_wire gives the split wire = w1 ++ w2, with H1 : lossy_dup (sender_burst b m k) w1, the fairness clause Hne : w1 <> [], and H2 : fair_wire (negb b) ms' k w2.
  • rewrite (deliver_app w1 w2 b) splits the receiver at the burst boundary: deliver b wire = deliver b w1 ++ deliver (next_bit b w1) w2.
  • destruct (burst_delivery_fair b m k w1 H1 Hne) — this is where fairness is spent. The nonemptiness Hne is the third argument handed to burst_delivery_fair; it is precisely what turns “at most one message” into “exactly one.” The result is Hd : deliver b w1 = [m] and Hn : next_bit b w1 = negb b.
  • Rewriting with Hd and Hn reduces the goal to m :: deliver (negb b) w2 = m :: ms', and the induction hypothesis IH on the fair tail w2 (with the flipped bit negb b) gives deliver (negb b) w2 = ms'.

The theorem stated for the sender’s own starting bit is the immediate corollary:

Theorem abp_liveness : forall (k : nat) (b0 : bit) (ms : list M) (wire : list Packet),
  fair_wire b0 ms k wire -> deliver b0 wire = ms.
Proof.
  intros k b0 ms wire H.
  apply (abp_liveness_gen k ms b0 wire H).
Qed.

Contrast the two conclusions. Safety concluded subseq (deliver b0 wire) ms from a bare lossy_dup hypothesis. Liveness concludes the equality deliver b0 wire = ms — everything sent, delivered, in order — from the stronger fair_wire hypothesis. The gap between “a subsequence” and “the whole list” is exactly the gap between the two hypotheses, and it is entirely paid for by the one clause w1 <> [].

A sanity check: the perfect channel

One special case pins the whole thing down. Over a perfect channel — where received equals sent, no loss and no duplication at all — the receiver should deliver the message list on the nose. The perfect channel is the tightest relation:

Definition perfect s r : Prop := r = s.

and delivery is exact, provided the sender actually retransmits at least once:

k>0∧w=sent(b0,ms,k)⇒Db0(w)=ms.k > 0 \;\wedge\; w = \mathrm{sent}(b_0, ms, k) \;\Longrightarrow\; D_{b_0}(w) = ms.

Theorem abp_collapse_perfect : forall (k : nat) (b0 : bit) (ms : list M) (wire : list Packet),
  k > 0 -> perfect (sender_stream b0 ms k) wire -> deliver b0 wire = ms.
Proof.
  intros k b0 ms wire Hk Hp.
  unfold perfect in Hp.
  rewrite Hp.
  apply (abp_collapse_perfect_gen k ms b0 Hk).
Qed.

The k > 0 hypothesis earns its place. With k = 0 the sender emits an empty burst for every message, so sender_stream b0 ms 0 = [] and the perfect channel faithfully delivers nothing — which equals ms only when ms is itself empty. A single retransmission attempt is all it takes over a channel that loses nothing; the theorem asks for exactly that and no more. It follows the same discipline as the fairness hypothesis: state the minimal condition the conclusion truly needs, and no larger one.

What is genuinely left open is fairness in its full, infinite form — “a message offered infinitely often is eventually delivered” — which needs the sender, channel, and receiver modelled as infinite streams rather than finite lists. That is a different and larger construction than the finite setting here supports, and it is not attempted — the finitary per-burst statement above is what is actually proved.
