The integral as a specification

There are two ways to pin down a mathematical operation. One is to give a formula that computes it. The other is to list the laws it must obey and refuse to say anything more — no formula, no construction, just the properties any implementation must have. A function that raises e to a power, for instance, can be pinned down entirely by four laws (it sends 0 to 1, turns sums into products, and so on); everything you can prove about it then follows from those laws alone, and holds for every function that satisfies them, whether you built it from a power series or some other way. This post applies exactly that move to integration: an interface that says what an integral is, by its laws, with no Riemann sums and no Lebesgue measure anywhere in sight.

The laws an integral obeys

Write int f a b for the integral of f from a to b — on the blackboard, ∫abf\int_a^b f. The scalars a, b come from an ordered field F (think of the rationals Q), and the values f t live in a normed vector space V over F: a space where vectors can be added, scaled by a scalar (written c *: v), and measured by a norm (written ‖ v ‖). Here is the whole specification, from Interface/Integral.v:

Class Integral (F V : Type) `{OrderedField F}
    {VS : VectorSpace F V} {NS : NormedSpace F V} := {
  (* integrand, lower endpoint, upper endpoint *)
  int : (F -> V) -> F -> F -> V;
  #[global] int_proper ::
    Proper (pointwise_relation F equ ==> equ ==> equ ==> equ) int;
  int_linear_add : forall (f g : F -> V) (a b : F),
    int (fun t : F => f t + g t) a b == int f a b + int g a b;
  int_linear_scale : forall (c : F) (f : F -> V) (a b : F),
    int (fun t : F => c *: f t) a b == c *: int f a b;
  int_additivity : forall (f : F -> V) (a b c : F),
    int f a b + int f b c == int f a c;
  int_const : forall (v : V) (a b : F),
    int (fun _ : F => v) a b == (b + - a) *: v;
  int_bound : forall (f : F -> V) (a b M : F),
    (forall t : F, a <= t -> t <= b -> ‖ f t ‖ <= M) ->
    a <= b ->
    ‖ int f a b ‖ <= M * (b + - a)
}.

Read the fields one at a time. The first, int, is just the operation: a function of an integrand and two endpoints. The rest are the laws.

  • int_proper says the integral respects equality: replacing the integrand by a pointwise-equal one, or an endpoint by an equal one, gives an equal result. (== is the structure’s own equality, and equ is its name.)
  • int_linear_add and int_linear_scale are linearity: the integral of a sum is the sum of the integrals, and a scalar pulls straight out through the integral sign.
  • int_additivity is the gluing law, ∫ab+∫bc=∫ac\int_a^b + \int_b^c = \int_a^c. Note it is stated with no condition on the order of a, b, c — it holds unconditionally, and that single choice is what forces the integral to reverse sign when its endpoints are swapped, as we will see below.
  • int_const fixes the integral of a constant function: ∫abv=(b−a)v\int_a^b v = (b - a)\,v. Here b + - a is b minus a.
  • int_bound is the estimate everything downstream leans on: if ‖f‖ stays below M across [a, b] and a <= b, then the integral is no larger than M times the width of the interval.

That is the entire contract. Nothing here says how to compute an integral, and nothing here mentions completeness — the property that distinguishes the real numbers from the rationals. So anything we prove from these laws is true of every model at once: a Riemann-sum integral over Q, an interval-arithmetic integral, whatever comes later. The laws are the meaning; the construction is a separate concern.

The empty integral is zero

The first consequence: integrating from a point back to the same point gives nothing. On the blackboard, let I=∫aafI = \int_a^a f; the chain is

I+I=I{additivity at the endpoints (a,a,a)}=I+0{adding 0 changes nothing}I=0{cancel the shared I on the left}\begin{aligned} I + I &= I && \{\text{additivity at the endpoints } (a, a, a)\}\\ &= I + 0 && \{\text{adding } 0 \text{ changes nothing}\}\\ I &= 0 && \{\text{cancel the shared } I \text{ on the left}\} \end{aligned}

The cancellation in the last step is the group law: adding the same element on the left is reversible, so equal sums force equal summands. Here is the proof exactly as it stands in Theory/Integral.v:

Theorem int_empty : forall (f : F -> V) (a : F), int f a a == 0.
Proof.
  intros f a.
  apply (cancel_l (int f a a)).
  rewrite int_additivity, idr.
  reflexivity.
Qed.

Walking it:

  • intros f a names the integrand and the point.
  • apply (cancel_l (int f a a)) is the cancellation step. To prove I == 0 it is enough to prove I + I == I + 0: adding the same I on the left of both sides is reversible (that is the group cancellation law), so the two smaller sides being equal forces the original ones equal.
  • rewrite int_additivity collapses the left side. int f a a + int f a a matches additivity at endpoints (a, a, a), which rewrites it to a single int f a a.
  • rewrite idr handles the right side: int f a a + 0 is just int f a a by the right-identity law idr.
  • reflexivity closes it — both sides are now int f a a.

Swapping the endpoints negates the integral

The second consequence is the sign rule ∫abf=−∫baf\int_a^b f = -\int_b^a f, which we promised would fall out of unconditional additivity. On the blackboard: glue ∫ba\int_b^a and ∫ab\int_a^b end to end,

∫baf+∫abf=∫bbf{additivity at (b,a,b)}=0{the empty integral, just proved}\begin{aligned} \int_b^a f + \int_a^b f &= \int_b^b f && \{\text{additivity at } (b, a, b)\}\\ &= 0 && \{\text{the empty integral, just proved}\} \end{aligned}

So ∫abf\int_a^b f is the thing you add to ∫baf\int_b^a f to get zero — that is precisely what “the additive inverse of ∫baf\int_b^a f” means, and additive inverses are unique. The Coq, from the same file:

Theorem int_rev : forall (f : F -> V) (a b : F),
  int f a b == - int f b a.
Proof.
  intros f a b.
  apply neg_unique_l.
  rewrite int_additivity.
  apply int_empty.
Qed.

Walking it:

  • intros f a b names the integrand and both endpoints.
  • apply neg_unique_l uses the fact that an additive inverse is unique: to show int f a b is the negative of int f b a, it suffices to show the two of them sum to zero. The goal becomes int f b a + int f a b == 0.
  • rewrite int_additivity collapses that sum. int f b a + int f a b matches additivity at (b, a, b), rewriting to int f b b.
  • apply int_empty finishes: int f b b == 0, which is the previous theorem at the point b.

The integral of nothing is nothing

One more, the shortest of the three. Integrating the constantly-zero function over any range gives the zero vector. On the blackboard it is a two-step chain:

∫ab0=(b−a)⋅0{the constant law, at the value 0}=0{scaling the zero vector gives the zero vector}\begin{aligned} \int_a^b 0 &= (b - a) \cdot 0 && \{\text{the constant law, at the value } 0\}\\ &= 0 && \{\text{scaling the zero vector gives the zero vector}\} \end{aligned}

The Coq, again verbatim:

Theorem int_zero_fn : forall a b : F,
  int (fun _ : F => (0 : V)) a b == 0.
Proof.
  intros a b.
  rewrite (int_const 0 a b).
  apply scale_zero.
Qed.

rewrite (int_const 0 a b) is the first blackboard step, turning the integral into (b + - a) *: 0; apply scale_zero is the second, the vector-space fact that scaling the zero vector yields 0. It is the last of the three plain consequences the theory file records.

What is deliberately missing

The law you might expect at the top — the fundamental theorem, that ∫abg=G(b)−G(a)\int_a^b g = G(b) - G(a) when G is an antiderivative of g — is not in the class, and the interface file says so in as many words. It is not provable from the laws above (those pin the integral down on constant and piecewise data, but connecting it to derivatives needs either completeness or a concrete construction), and it is not yet a field of the class either, because it would drag in the derivative’s exact formulation before that has settled. So it waits — either as a theorem in a file that has both a derivative and a model, or as a law of a stronger class that extends this one. Leaving it out is not a gap; it is the same discipline — state only what the laws can support — that lets the three theorems above hold for every integral there will ever be, without any of them having been built yet.
