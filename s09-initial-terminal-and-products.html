Initial and terminal objects, products, and the universal-property style of proof

A category is a collection of objects together with arrows between them. For each pair of objects xx and yy there is a collection of arrows written x ~> y; arrows compose associatively (g ∘ f runs f then g), and every object carries an identity arrow that does nothing under composition. One design choice will matter throughout: arrows are compared not by Coq’s built-in definitional equality but by a per-pair equivalence relation written ==, a hom-setoid. Categories built out of syntax — lambda terms, for instance — need to treat two arrows as equal when they agree only after simplification, and definitional equality is too strict for that. (A functor, a map between categories that preserves identities and composition, was the subject of the previous post; we will not need it here.)

This post is about a different kind of definition and a different kind of proof. Most of the objects one meets are built by writing down their elements. The objects here are pinned down instead by a single sentence about arrows — “there is exactly one arrow with such and such a shape” — and that phrase, “exactly one,” is the whole subject. It is called a universal property, and proving things from one is a genuinely different activity from equational rewriting: you prove existence and uniqueness separately, and the uniqueness half is often proved by turning the universal property against itself. We will do that twice.

Two objects defined by counting arrows

A terminal object is an object into which every object has exactly one arrow. An initial object is the mirror image: every object has exactly one arrow out of it. In symbols, with 11 the terminal object and 00 the initial one: for every object xx there is exactly one arrow x→1x \to 1, and exactly one arrow 0→x0 \to x. Here are the two classes, verbatim from the source:

(* An initial object: every object has exactly one morphism out of it.
   The exact dual of Terminal, stated up to the hom-setoid. *)
Class Initial {O : Type} (C : Category O) := {
  init : O;
  emit : forall x : O, init ~> x;
  emit_unique : forall (x : O) (f : init ~> x), f == emit x
}.

(* A terminal object: every object has exactly one morphism into it.
   Uniqueness is stated up to the hom-setoid, not definitional equality. *)
Class Terminal {O : Type} (C : Category O) := {
  term : O;
  bang : forall x : O, x ~> term;
  bang_unique : forall (x : O) (f : x ~> term), f == bang x
}.

Read Terminal slowly, because the shape recurs everywhere below. It carries three things: a distinguished object term; a chosen arrow bang x : x ~> term for every object x (the exclamation-mark name is traditional — “the unique arrow,” written !); and the field that does the real work, bang_unique, which says that any arrow f : x ~> term you might have is equal to that chosen one. Existence is bang; uniqueness is bang_unique. Together they are “exactly one.” Initial is the same statement with every arrow reversed.

A concrete instance: Coq’s own types

None of this is abstract nonsense until you see it land on something familiar. Take the category whose objects are Coq types and whose arrows X ~> Y are ordinary functions X -> Y. In that category the one-element type unit is terminal: from any type there is exactly one function into unit, the one that ignores its input and returns tt. In symbols, writing ⋆\star for the single inhabitant: every f:X→1f : X \to 1 satisfies f(x)=⋆f(x) = \star at every xx, because there is nothing else f(x)f(x) could be. Here is that instance, and the proof that the arrow really is unique:

(* Tactic exception: uniqueness into unit is forced by destructing the
   unit result f x — unit's universal property is its case analysis. *)
Lemma unit_bang_unique (X : Type) (f : X -> unit) :
  fn_equ X unit f (fun _ : X => tt).
Proof.
  intros x. destruct (f x). reflexivity.
Qed.

#[global] Instance CoqTerminal : Terminal CoqCat :=
  @Build_Terminal Type CoqCat
    unit
    (fun X : Type => fun _ : X => tt)
    unit_bang_unique.

The proof is one line of content: given any f : X -> unit, at each argument x the value f x lives in unit, and unit has a single inhabitant, so case-analysis (destruct (f x)) leaves tt = tt. There is nowhere else for f x to go. Dually, the empty type Empty_set is initial — from it there is exactly one function to any type, because a function out of the empty type has no input to disagree about — and the source records that as CoqInitial in the same file.

The universal-property proof style, on the smallest possible example

Before the theorem, the intuition. When an object is defined by “there is exactly one arrow into it,” the uniqueness clause is a tool you can point at any arrow of the right shape and conclude it equals the canonical one. The trick that follows appears again and again: if two arrows both have the shape the universal property talks about, they are both equal to the canonical arrow, hence to each other. You never compute either arrow; you only note that they could not differ.

The smallest instance of the trick: any arrow from a terminal object to itself is the identity. On the blackboard it is two applications of one fact. Write ! for the unique arrow into term. Then any endomorphism f:term→termf : \text{term} \to \text{term} satisfies f=!f = {!}, and so does the identity, id=!\mathrm{id} = {!}; therefore

f=!{uniqueness, aimed at f}=id{uniqueness, aimed at id} f \;=\; {!} \quad\{\text{uniqueness, aimed at } f\}\quad =\; \mathrm{id} \quad\{\text{uniqueness, aimed at } \mathrm{id}\}

In Coq:

(* An endomorphism of a terminal object is a morphism into it, hence
   the bang — and so is cat_id, so uniqueness equates the two. *)
Lemma term_endo_id (T : Terminal C) :
  forall f : term (Terminal:=T) ~> term (Terminal:=T), f == cat_id.
Proof.
  intros f.
  rewrite (bang_unique _ f).
  symmetry.
  apply bang_unique.
Qed.

Step by step. intros f takes an arbitrary arrow f from term to itself. rewrite (bang_unique _ f) uses uniqueness on f: it is an arrow into term, so it equals the canonical bang term, and we replace f by bang term, leaving the goal bang term == cat_id. symmetry flips this to cat_id == bang term. Finally apply bang_unique uses uniqueness a second time, now on cat_id, which is also an arrow into term and hence also the canonical bang. Two uses of the same field, no calculation. The initial object obeys the mirror-image lemma init_endo_id, proved the same way with emit.

Uniqueness up to isomorphism

Now the theorem this style exists for. A universal property can never name a unique object — if T is terminal, anything isomorphic to T is terminal too, since you can transport the arrows across the isomorphism. What it can guarantee is that any two objects satisfying it are isomorphic, and canonically so. Recall that an isomorphism between objects x and y is a pair of arrows x ~> y and y ~> x whose two composites are the identities; the source packages this as a record Iso x y, and writes x ≅ y (“x is isomorphic to y”) for its mere existence.

The argument for two terminal objects T1T_1 and T2T_2: each has a unique arrow into the other, call them !12:T1→T2!_{12} : T_1 \to T_2 and !21:T2→T1!_{21} : T_2 \to T_1. Compose them one way and you get an endomorphism of T1T_1; compose the other way, an endomorphism of T2T_2. By the lemma we just proved, both composites are identities:

T1→!12T2→!21T1=idT1,T2→!21T1→!12T2=idT2. T_1 \xrightarrow{\;!_{12}\;} T_2 \xrightarrow{\;!_{21}\;} T_1 \;=\; \mathrm{id}_{T_1}, \qquad T_2 \xrightarrow{\;!_{21}\;} T_1 \xrightarrow{\;!_{12}\;} T_2 \;=\; \mathrm{id}_{T_2}.

That is exactly the data of an isomorphism, and the source builds it as a record literal — the two chosen arrows in the iso_to/iso_from fields, and term_endo_id discharging each round-trip:

(* Terminal objects are unique up to isomorphism: the two bangs
   compose to endomorphisms of terminal objects, which are identities. *)
Definition terminal_iso (T1 T2 : Terminal C) :
  Iso (term (Terminal:=T1)) (term (Terminal:=T2)) := {|
  iso_to := bang (Terminal:=T2) (term (Terminal:=T1));
  iso_from := bang (Terminal:=T1) (term (Terminal:=T2));
  iso_fromto := term_endo_id T1 _;
  iso_tofrom := term_endo_id T2 _
|}.

iso_to is the unique arrow from T1T_1 into T2T_2; iso_from the unique arrow from T2T_2 into T1T_1. The field iso_fromto must prove iso_from ∘ iso_to == cat_id — but that composite is an endomorphism of T1T_1, so term_endo_id T1 _ is exactly the proof, and iso_tofrom the same for T2T_2. Notice there is no Proof ... Qed here at all: the isomorphism is constructed, not searched for, because the two arrows it is made of were forced on us. Turning it into the relation ≅ is a one-liner:

Corollary terminal_isomorphic (T1 T2 : Terminal C) :
  term (Terminal:=T1) ≅ term (Terminal:=T2).
Proof. exact (inhabits (terminal_iso T1 T2)). Qed.

Initial objects satisfy the dual statement, initial_isomorphic, built identically from init_endo_id. This is why one is entitled to speak of the terminal object, or the empty type, or the one-element type: not because there is literally one, but because any two are the same in the only sense a category can see.

Products, by the same recipe

A binary product of two objects a and b is an object equipped with two projection arrows and one universal property tying them together. The intuition is the ordered pair: an element of a × b is determined by its two coordinates, and to build a map into a × b it is enough to say what its two coordinates are. Here is the class:

(* Binary products: a limiting cone for each pair of objects.
   Uniqueness is stated up to the hom-setoid, not definitional equality. *)
Class HasProducts {O : Type} (C : Category O) := {
  prod_obj : O -> O -> O;
  pi1 : forall {a b : O}, prod_obj a b ~> a;
  pi2 : forall {a b : O}, prod_obj a b ~> b;
  pair_mor : forall {x a b : O}, (x ~> a) -> (x ~> b) -> (x ~> prod_obj a b);
  #[global] pair_proper :: forall x a b : O,
      Proper (equ ==> equ ==> equ) (@pair_mor x a b);
  pair_pi1 : forall (x a b : O) (f : x ~> a) (g : x ~> b),
      pi1 ∘ pair_mor f g == f;
  pair_pi2 : forall (x a b : O) (f : x ~> a) (g : x ~> b),
      pi2 ∘ pair_mor f g == g;
  pair_unique : forall (x a b : O) (f : x ~> a) (g : x ~> b)
                       (h : x ~> prod_obj a b),
      pi1 ∘ h == f -> pi2 ∘ h == g -> h == pair_mor f g
}.

prod_obj a b is the product object; pi1 and pi2 are the two projections. pair_mor f g is the mediating arrow: given a pair of arrows f : x ~> a and g : x ~> b out of a common source, it builds the single arrow x ~> prod_obj a b whose coordinates are f and g. The laws pair_pi1 and pair_pi2 say projecting recovers the coordinates you put in — π1∘⟨f,g⟩=f\pi_1 \circ \langle f, g\rangle = f — and pair_unique is again the “exactly one” clause: any arrow h with those two projections is the pairing. In Coq’s own category the product is the pair type prod, with fst and snd as the two projections; the source instance CoqProducts discharges the two projection laws by reflexivity and the uniqueness law by case-analysis on the pair.

The clean small theorem here reads the uniqueness clause backwards. Any arrow h into a product equals the pairing of its own two projections — the eta law for products. On the blackboard: h’s coordinates are π1∘h\pi_1 \circ h and π2∘h\pi_2 \circ h, and the pairing of an arrow’s coordinates is the arrow itself,

⟨π1∘h,π2∘h⟩=h. \langle\, \pi_1 \circ h,\; \pi_2 \circ h \,\rangle \;=\; h. 

(* Every mediating morphism into a product is the pairing of its
   projections: the eta law, read off from uniqueness. *)
Theorem pair_eta : forall (x a b : O) (h : x ~> prod_obj a b),
    pair_mor (pi1 ∘ h) (pi2 ∘ h) == h.
Proof.
  intros x a b h.
  symmetry.
  apply pair_unique; reflexivity.
Qed.

intros x a b h fixes the arrow. symmetry turns the goal into h == pair_mor (pi1 ∘ h) (pi2 ∘ h), which is precisely the shape pair_unique concludes. apply pair_unique then asks for the two hypotheses pi1 ∘ h == pi1 ∘ h and pi2 ∘ h == pi2 ∘ h — literally reflexive, so the single reflexivity (applied to both goals by the ;) closes them. The whole proof is: h has some projections, and the pairing of an arrow’s own projections is that arrow. Once more we computed nothing; the universal property did the work.

That is the shape of every proof in this corner of the library. Existence is a chosen arrow; uniqueness is a field you aim at any arrow of the right shape; and identity, isomorphism, and the eta law all fall out of aiming it twice.
