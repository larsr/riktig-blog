riktig #29 — decidable if-then-else

Twenty-ninth post documenting the riktig project. One small typeclass, Decidable, lets recursive programs be written as plain lambda terms with their conditions in ordinary if-then-else form — and hands every proof about them the branch facts for free.

The idea

A condition in a program is a proposition; a condition a program can run is a decision procedure. One typeclass connects the two (Interface/Decidable.v):

Class Decidable (P : Prop) := dec : {P} + {~ P}.

A value of {P} + {~ P} is a yes/no answer carrying evidence, and Coq’s if already branches on it. So wherever an instance exists one writes

if dec (b = 0) then .. else ..

and gets both worlds at once. At the term level this behaves exactly like an ordinary if: it reduces, simpl sees through it, and closed computations go through by Logic.eq_refl. In a proof, destruct (dec (b = 0)) yields two goals, one holding b = 0 and the other b <> 0 — the branch condition is in the context, not lost in a boolean.

The vocabulary is a small three-file family. The signature above is the whole of Interface/Decidable.v. What follows generally is Theory/Decidable.v: decidability is closed under the connectives (and_dec, or_dec, not_dec, impl_dec), decidable equality transports to lists and pairs, and a dec conditional can be read through any boolean that reflects its proposition. The concrete carriers — nat with its equality and order comparisons, bool with the b = true test — are Impl/Decidable.v.

A whole program: Euclid’s algorithm

Here is gcd, complete, from Programs/Gcd.v. The recursion is not structural (a mod b is not a subterm of anything), so it goes through the standard library’s well-founded Fix, on the tupled argument, with the measure snd — the divisor shrinks at every call:

Definition gcd : nat -> nat -> nat.
 apply (uncurry ∘ (Fix (well_founded_ltof (nat * nat) snd) (fun _ => nat))).
  refine
  ( fun p gcd =>
      match p as x return p=x -> nat with
      | (a, b) =>
          fun e =>
          if dec (b = 0)
            then a
            else gcd (b, a mod b) _
      end eq_refl );
  (subst p; unfold ltof; simpl; abstract (now apply Nat.mod_upper_bound)).
 Defined.

The body is one plain term — a fun, a match, an if — that reads as the two-line mathematical definition it is. The single _ is the termination obligation, discharged by the one tactic line after the term. And this is where dec quietly earns its keep: the hole sits inside the else branch, so the context already contains b <> 0 — exactly what Nat.mod_upper_bound needs to show a mod b < b. The condition the program branched on is the fact the termination proof uses; nothing is stated twice.

Because a Fix-defined function does not unfold like a Fixpoint, each program derives its recursion equation once, via Fix_eq:

Lemma gcd_step: forall a b : nat,
    gcd a b = if dec (b = 0) then a else gcd b (a mod b).

From here on, all reasoning goes through gcd_step; the Fix machinery is never touched again. And the definitions compute — Coq accepts these by reduction alone:

Check Logic.eq_refl : gcd 48 18 = 6.
Check Logic.eq_refl : gcd 270 192 = 6.

The same shape everywhere

The other programs in theories/Programs/ are the same pattern with different measures. merge compares with the ordered-values class (if dec (a0 < b0), any DecTotalOrder carrier), well-founded on the sum of the two list lengths. Binary search shrinks a numeric range, quicksort recurses on partition sizes, merge sort on split’s output — none structural, all one dec-conditioned lambda term plus one derived _step equation.

For programs whose conditions come from a boolean function (merge sort’s Nat.leb), one general lemma in Theory/Decidable.v bridges the two styles:

Lemma dec_if_bool (P : Prop) {DP : Decidable P} (b : bool) (A : Type) (a1 a2 : A) :
  (P <-> b = true) ->
  (if dec P then a1 else a2) = (if b then a1 else a2).

so a clause equation stated with dec can be restated with <? or <=? in one line, and stdlib facts like Nat.ltb_lt plug in directly.

What is proved about them

The properties live in theories/Theory/, one file per program, reasoning only through the _step equations. For gcd (Theory/Gcd.v): gcd a b divides both arguments and every common divisor divides it — via a bridge lemma gcd_eq_stdlib : gcd a b = Nat.gcd a b identifying the program with the standard library’s function. Binary search returns an index holding the target and, on sorted ranges, cannot miss (Theory/BinarySearch.v); merge is a sorted permutation and commutative on sorted inputs (Theory/Merge.v); merge sort is correct (Theory/MergeSort.v). Print Assumptions on the gcd chain answers Closed under the global context — with the flat tupled form, even Fix_eq’s extensionality premise closes pointwise, so no axiom enters.
