Order and absolute value, axiomatically

A field is a set with addition, subtraction, multiplication, and division — every nonzero element has a multiplicative inverse — all obeying the familiar commutative and distributive laws. The rational numbers are the first example. But a field says nothing about whether one element sits below another, and nothing about how far an element is from zero: <= and abs are simply not part of its vocabulary. This post adds them, as two operations governed by laws rather than by any construction, and then derives from those laws a fact that looks too plain to need a proof at all — that 0 <= 1.

Order and absolute value as vocabulary

Two of the structures in this library keep what operations exist apart from how they behave. Order is opt-in vocabulary — most structures never mention it — so its two symbols live in their own file rather than in the shared symbol table. Here they are, from Interface/Order.v:

Class HasLe T := le : Rel T.
Infix "<=" := le (at level 70, no associativity).

Class HasAbs T := abs : T -> T.

HasLe says there is a binary relation written <=; HasAbs says there is a unary operation abs. Neither says anything yet about how they behave. A strict order is not a third operation — it is a definition that unfolds to a conjunction the order laws already speak about:

Definition lt {T : Type} `{EqRel T} `{HasLe T} (a b : T) : Prop :=
  a <= b /\ a =/= b.
Infix "<" := lt (at level 70, no associativity).

So a < b means exactly a <= b and a =/= b — below, and not equal.

The ordered field: the laws

The structure bundles the vocabulary with its laws. Here is the whole OrderedField class, from Interface/Order.v:

Class OrderedField F `{Field F} := {
  #[global] of_le  :: HasLe F;
  #[global] of_abs :: HasAbs F;
  le_refl       : forall a : F, a <= a;
  le_trans      : forall a b c : F, a <= b -> b <= c -> a <= c;
  le_antisym    : forall a b : F, a <= b -> b <= a -> a == b;
  le_total      : forall a b : F, a <= b \/ b <= a;
  (* Setoid equality is not decidable, so squeeze arguments ("x lies
     below every positive eps, hence x is zero") end in a double
     negation; this law discharges it.  Every intended model satisfies
     it: decidable orders (Q) and classical models trivially, and
     constructive Cauchy reals define <= negatively (a <= b := ~ b < a),
     so their <= is stable as well. *)
  le_stable     : forall a b : F, ~ ~ a <= b -> a <= b;
  #[global] le_proper :: Proper (equ ==> equ ==> iff) (@le F _);
  add_le_mono_l : forall a b c : F, a <= b -> c + a <= c + b;
  mul_nonneg    : forall a b : F, 0 <= a -> 0 <= b -> 0 <= a * b;
  #[global] abs_proper :: Proper (equ ==> equ) (@abs F _);
  abs_nonneg    : forall a : F, 0 <= abs a;
  abs_zero_iff  : forall a : F, abs a == 0 <-> a == 0;
  abs_mul       : forall a b : F, abs (a * b) == abs a * abs b;
  (* Constructively derivable from the other laws only via a case
     analysis on le_total plus several monotonicity lemmas (abs (- a)
     squares to abs a * abs a, then nonneg square roots are unique);
     stated as a law to keep proofs that use it one rewrite long. *)
  abs_neg       : forall a : F, abs (- a) == abs a;
  abs_triangle  : forall a b : F, abs (a + b) <= abs a + abs b;
  abs_nonneg_id : forall a : F, 0 <= a -> abs a == a
}.

The top two fields (:: marks a field visible to Coq’s instance search) supply the <= and abs operations to any field known to be ordered. The next block governs <=: it is reflexive, transitive, antisymmetric — a <= b and b <= a force a == b — and total, meaning any two elements compare one way or the other. le_stable says <= is not refuted by refuting its refutation; it stands in for decidability, which is deliberately not assumed. add_le_mono_l and mul_nonneg are the two ways order interacts with the arithmetic: adding the same thing to both sides preserves <=, and a product of nonnegatives is nonnegative.

The last block is the point of the title. abs is not defined as “a if a is nonnegative, else -a” — that would be a case split on the sign, and would need the order to be decidable. Instead abs is an operation pinned down entirely by five laws: it is nonnegative (abs_nonneg), it is zero exactly when its argument is (abs_zero_iff), it is multiplicative (abs_mul), it is unchanged by negation (abs_neg), and it obeys the triangle inequality (abs_triangle). One extra law, abs_nonneg_id, says abs leaves a nonnegative element alone. Everything else about absolute value is derived from these.

The absolute value of one is one

Here is a first consequence, small but not free. In English: the absolute value of 1 is 1.

The argument does not case-split on any sign. It observes that abs 1 is a nonzero solution of the equation x = x * x, and a nonzero solution of that equation must be 1. Why is abs 1 such a solution? Because absolute value is multiplicative:

|1|⋅|1|=|1⋅1|{multiplicativity}=|1|{1⋅1=1}=|1|⋅1{x=x⋅1}. \lvert 1\rvert \cdot \lvert 1\rvert \;=\; \lvert 1\cdot 1\rvert \quad\{\text{multiplicativity}\} \;=\; \lvert 1\rvert \quad\{1 \cdot 1 = 1\} \;=\; \lvert 1\rvert \cdot 1 \quad\{x = x \cdot 1\}.

Reading the far ends, abs 1 * abs 1 == abs 1 * 1; and abs 1 is not zero, because abs a == 0 holds only when a == 0, whereas 1 =/= 0 in any field. Cancelling the common factor abs 1 on the left leaves abs 1 == 1. Here is the proof exactly as it appears in Theory/Order.v:

Lemma abs_one : abs (1 : F) == 1.
Proof.
  apply (mul_cancel_l (abs 1)).
  - intros Hz. exact (field_nontrivial (proj1 (abs_zero_iff 1) Hz)).
  - calc (abs (1 : F) * abs (1 : F))
      == (abs ((1 : F) * 1))  { abs_mul }
      == (abs (1 : F))        { ring_mul_1r }
      == (abs (1 : F) * 1)    { ring_mul_1r }.
Qed.

Walking it:

  • apply (mul_cancel_l (abs 1)) invokes left cancellation, mul_cancel_l : forall a b c, a =/= 0 -> a * b == a * c -> b == c, with a := abs 1. This turns the goal abs 1 == 1 into two obligations: that abs 1 is nonzero, and that abs 1 * abs 1 == abs 1 * 1.
  • The first bullet discharges abs 1 =/= 0: assume abs 1 == 0 (Hz); then proj1 (abs_zero_iff 1) Hz gives 1 == 0, and field_nontrivial — the field law 1 =/= 0 — turns that into the contradiction the goal wants.
  • The second bullet is the blackboard chain, machine-checked. The calc block reads abs 1 * abs 1 == abs (1 * 1) == abs 1 == abs 1 * 1, and each { ... } names the single law that justifies its step: abs_mul folds the two factors into one absolute value, then ring_mul_1r (x * 1 == x) is used once to simplify 1 * 1 to 1 and once, in reverse, to reintroduce the * 1 on the right.

The payoff: zero is at most one

Now the theorem the title promises. In English: 0 <= 1, and this is a derived fact, not an axiom. On the blackboard it is two short steps:

0≤|1|{absolute values are nonnegative}=1{|1|=1,just proved}.0 \;\le\; \lvert 1\rvert \quad\{\text{absolute values are nonnegative}\} \;=\; 1 \quad\{\lvert 1\rvert = 1,\ \text{just proved}\}.

The first is a genuine inequality — abs_nonneg says the absolute value of anything is at least 0, so in particular 0 <= abs 1. The second is the equation we just proved, abs 1 == 1. Chaining an inequality with an equation gives 0 <= 1. Here it is verbatim from Theory/Order.v:

Lemma zero_le_one : (0 : F) <= 1.
Proof.
  calc (0 : F)
    == (abs (1 : F)) { (abs_nonneg (1 : F)) }
    == (1 : F)       { abs_one }.
Qed.

The one subtlety worth pausing on is that this calc chain is not a chain of equalities, even though every separator is written ==. In this library the == token is the fixed separator of a calculational chain; the relation actually being chained is the one in the goal, and here the goal is 0 <= 1, so the chain is over <=. It works because <= is registered as reflexive and transitive: calc breaks the goal at the intermediate object abs 1 into 0 <= abs 1 and abs 1 <= 1, and discharges each step by trying its justification as a rewrite or an application until one closes.

  • The first step, 0 <= abs 1, is closed by applying abs_nonneg 1 directly — it is exactly that inequality.
  • The second step, abs 1 <= 1, is closed by rewriting with abs_one (abs 1 == 1), which turns the goal into 1 <= 1, and that holds by reflexivity of <=.

So an inequality and an equation combine, through transitivity, into the plain fact 0 <= 1 — derived from the laws, computing nothing.

A field where this all runs: the rationals

None of this is abstract for its own sake. The rational numbers are the first type shown to satisfy OrderedField, in Impl/Qof.v, with Coq’s standard Qle as the order and Qabs as the absolute value:

#[global] Instance Q_Le  : HasLe Q := Qle.
#[global] Instance Q_Abs : HasAbs Q := Qabs.

Each law of the class is discharged by the corresponding standard fact about Q, and the instance is assembled as one record:

#[global] Instance Q_OrderedField : OrderedField Q := {|
  of_le         := Q_Le;
  of_abs        := Q_Abs;
  le_refl       := Qle_refl;
  le_trans      := Qle_trans;
  le_antisym    := Qle_antisym;
  le_total      := Qle_total;
  le_stable     := Qle_stable;
  le_proper     := Qle_comp;
  add_le_mono_l := Qplus_le_mono_l;
  mul_nonneg    := Qmult_le_0_compat;
  abs_proper    := Qabs_wd;
  abs_nonneg    := Qabs_nonneg;
  abs_zero_iff  := Qabs_zero_iff;
  abs_mul       := Qabs_Qmult;
  abs_neg       := Qabs_opp;
  abs_triangle  := Qabs_triangle;
  abs_nonneg_id := Qabs_pos
|}.

Notice le_stable is filled by Qle_stable: Q’s order is decidable (through Qle_bool), and the file’s comment records this as a check on the design — stating stability rather than decidability costs nothing where decidability happens to hold, and buys the freedom to instantiate the same interface at number systems where it does not.

With the instance in scope, the generic order theory computes on Q. The theory defines half a := a * inv two; instantiated at the rational 2, it really divides:

Example q_half : (half (2#1) == 1#1)%Q.
Proof. reflexivity. Qed.

The two sides reduce to the same rational, so reflexivity closes it. That the whole instance assembles — that Coq’s instance search can produce an OrderedField Q on demand — is itself checked in the file, as a one-line resolution test:

Example q_of_check : OrderedField Q := _.

The underscore asks instance search to fill in the record, and it does. And with the instance in scope, every generic theorem holds of Q with no further work: the one this post proved is among them, so 0≤10 \le 1 holds on the rationals as zero_le_one instantiated at Q, no new proof written. Whatever <= and abs mean on the rationals, they were forced to mean it by the laws — and 0 <= 1 came out the far end.
