Vector spaces without coordinates

A field is a set of scalars you can add, multiply, and — except for zero — divide. The rational numbers ℚ\mathbb{Q} and the real numbers ℝ\mathbb{R} are the standard examples: addition and multiplication each have an identity (00 and 11), negation undoes addition everywhere, reciprocals undo multiplication everywhere except at 00, and multiplication distributes over addition. Everything below sits on top of one fixed field FF, whose elements we will call scalars.

One notational point first. Throughout, == is not Coq’s built-in equality of terms but a setoid equality: a chosen equivalence relation that every operation is required to respect. For ℚ\mathbb{Q} that relation makes 1/2 and 2/4 the same element; for the abstract structures here it is whatever equality each carrier comes with.

A vector space is an abelian group acted on by a field

The vectors themselves form an abelian group: they add, addition is associative and commutative, there is a zero vector, and every vector has a negative. On top of that group, the field acts by scaling — a scalar aa and a vector uu produce a vector a*:ua *: u — subject to four laws: scaling distributes over vector sums and over scalar sums, two scalings compose by multiplying the scalars, and the scalar 11 scales trivially.

In the code those four laws are a small mixin, ScaleLaws, kept separate from the group so that other structures can reuse it over a group they already own:

Class ScaleLaws (F V : Type) `{Field F} `{AbGroup V} `{HasScale F V} := {
  #[global] sc_proper :: Proper (equ ==> equ ==> equ) (@scale F V _);
  sc_scale_add   : forall (a : F) (u v : V), a *: (u + v) == a *: u + a *: v;
  sc_add_scale   : forall (a b : F) (u : V), (a + b) *: u == a *: u + b *: u;
  sc_scale_scale : forall (a b : F) (u : V), (a * b) *: u == a *: (b *: u);
  sc_scale_1     : forall u : V, 1 *: u == u
}.

A vector space then bundles the abelian group, the scaling operation, and this mixin:

Class VectorSpace (F V : Type) `{Field F} := {
  #[global] vs_ab    :: AbGroup V;
  #[global] vs_scale :: HasScale F V;
  #[global] vs_laws  :: ScaleLaws F V
}.

No basis, no coordinates, no list of numbers — a vector space is these operations and these laws, nothing more. Everything provable about vector spaces in general is provable from exactly this.

Scaling commutes: scale_swap

Here is a small consequence of the laws, proved for every vector space at once. Scaling by aa and then by bb gives the same vector as scaling by bb and then by aa:

a*:(b*:u)=(ab)*:u=(ba)*:u=b*:(a*:u).a *: (b *: u) \;=\; (a b) *: u \;=\; (b a) *: u \;=\; b *: (a *: u).

The two outer equalities collapse and re-expand a nested scaling into a single scalar factor; the middle one is just the field’s commutativity of multiplication. In Coq:

Theorem scale_swap : forall (a b : F) (u : V),
  a *: (b *: u) == b *: (a *: u).
Proof.
  intros a b u.
  calc (a *: (b *: u))
    == ((a * b) *: u)  { (sc_scale_scale a b u) }
    == ((b * a) *: u)  { (comm a b) }
    == (b *: (a *: u)) { (sc_scale_scale b a u) }.
Qed.

Step by step:

  • intros a b u fixes two scalars and a vector.
  • calc opens an equational chain. Each line == y { law } claims the current expression equals y, justified by the named law, and Coq checks that the chain as a whole proves the goal.
  • sc_scale_scale a b u is the axiom (ab)*:u=a*:(b*:u)(a b) *: u = a *: (b *: u), used right-to-left to fuse the nested scaling into the single factor aba b.
  • comm a b is the field’s law ab=baa b = b a.
  • sc_scale_scale b a u, this time left-to-right, re-splits (ba)*:u(b a) *: u back into b*:(a*:u)b *: (a *: u).

The one fact doing the real work is comm: scalars commute because the field’s multiplication does. Hold on to that — a page from now we will meet a multiplication that does not commute, and the contrast is the whole point.

Reuse: scaling preserves the zero vector

Not every proof needs its own induction. Fix a scalar aa and look at the map u↦a*:uu \mapsto a *: u. The law sc_scale_add says this map sends a sum to the sum of the images — that is exactly what it means to be a homomorphism of the additive group. And back in the groups post we proved, once, that any map preserving + must send 00 to 00 (hom_zero). So scaling a zero vector gives zero, with no new work:

Theorem scale_zero : forall a : F, a *: (0 : V) == 0.
Proof.
  intros a.
  exact (hom_zero (scale a) (scale_l_proper a) (sc_scale_add a)).
Qed.

The proof is a single exact: hand hom_zero the map (scale a), a proof that it respects ==, and its additivity law, and the group-level result applies verbatim. This is the payoff of proving things about structures rather than about particular types: a lemma about groups is already a lemma about the additive part of every vector space.

A space of transformations, one level up

Now the twist that gives this post its title. Among all functions between two vector spaces UU and VV, the interesting ones are the linear maps: those that respect ==, send sums to sums, and commute with scaling. In the code a linear map is a record carrying its function together with those three guarantees:

Record LinearMap (F U V : Type) {FF : Field F}
    {VU : VectorSpace F U} {VV : VectorSpace F V} := {
  lm_fun :> U -> V;
  lm_proper : Proper (equ ==> equ) lm_fun;
  lm_add : forall u v : U, lm_fun (u + v) == lm_fun u + lm_fun v;
  lm_scale : forall (a : F) (u : U), lm_fun (a *: u) == a *: lm_fun u
}.

Write Hom U V for the collection of these maps. Here is the fact: the linear maps between UU and VV are themselves a vector space. You can add two maps (u↦fu+guu \mapsto f\,u + g\,u), negate a map, and scale a map by a scalar (u↦a*:fuu \mapsto a *: f\,u) — all pointwise — and each result is again linear, and the vector-space laws hold at every point because they hold in VV.

Take the scaling of a map. To register it as a linear map we owe a proof that a*:fa *: f commutes with scaling — and that proof is scale_swap, the theorem above, applied at each point:

Lemma hom_smul_scale : forall (b : F) (u : U),
  a *: f (b *: u) == b *: (a *: f u).
Proof.
  intros b u.
  rewrite (lm_scale f b u).
  apply scale_swap.
Qed.

rewrite (lm_scale f b u) pushes the inner scaling through ff; what remains, a*:(b*:fu)=b*:(a*:fu)a *: (b *: f\,u) = b *: (a *: f\,u), is scale_swap. Assemble the four such laws and the vector-space structure of Hom U V is complete:

#[global] Instance Hom_VectorSpace : VectorSpace F Hom :=
  @Build_VectorSpace F Hom FF Hom_AbGroup Hom_Scale Hom_ScaleLaws.

A space of transformations has its own vector-space structure, one level up — built out of the space it maps into, by lifting each law through “evaluate at a point.”

The endomorphism ring: composition does not commute

Specialize to the maps from a space to itself, EndV=HomVV\mathrm{End}\,V = \mathrm{Hom}\,V\,V. These have everything above — they add and scale as a vector space — but they carry one operation more: you can compose two of them. Composition takes the role of multiplication, with the identity map as the unit 11:

(* lm_comp g f applies g after f, so f * g is f ∘ g. *)
#[global] Instance End_Mult : HasMult EndV :=
  fun f g : EndV => lm_comp f g.

With addition from the vector-space structure and composition as the product, EndV\mathrm{End}\,V satisfies the ring laws — composition is associative, the identity is a two-sided unit, and composition distributes over addition on both sides:

#[global] Instance End_Ring : Ring EndV := {|
  ring_ab         := @Hom_AbGroup F V V FF VV VV;
  ring_one        := End_One;
  ring_mul        := End_Mult;
  ring_mul_assoc  := end_mul_assoc;
  ring_mul_1l     := end_mul_1l;
  ring_mul_1r     := end_mul_1r;
  ring_distl      := end_distl;
  ring_distr      := end_distr;
  ring_mul_proper := end_mul_proper
|}.

This is the first noncommutative ring in the development, and the reason is the one flagged earlier. Scalar multiplication commuted because the field’s multiplication did — scale_swap had comm at its centre. Composition has no such law: f*gf * g means “ff after gg”, and f(gu)f(g\,u) is not in general g(fu)g(f\,u). Applying one transformation and then another is simply not the same as applying them in the other order — rotating then reflecting differs from reflecting then rotating — so f*g≠g*ff * g \neq g * f, and the ring is honestly noncommutative. Nothing is broken; the axioms a ring demands (associativity, units, distributivity) never included commutativity of the product, precisely so that structures like this one qualify.

And because EndV\mathrm{End}\,V is registered as a ring, every theorem proved once for all rings applies to it with no further work. Here is one, quoted from the file as a check that the structure is real:

Example end_mul_0_l (F V : Type) (FF : Field F) (VV : VectorSpace F V) :
  forall f : LinearMap F V V, 0 * f == 0.
Proof.
  intros f.
  exact (mul_0_l f).
Qed.

mul_0_l — “zero times anything is zero” — is a generic ring lemma. For EndV\mathrm{End}\,V it says the zero map composed after any ff is again the zero map, and its proof here is one exact: the general theorem, instantiated. Prove the ring laws once, inherit the ring theory everywhere.
