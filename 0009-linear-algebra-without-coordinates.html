riktig #9 — Linear algebra without coordinates

Ninth post in the series documenting the riktig project. This one is a little different: it teaches a piece of a linear algebra course, with each proof shown twice — once as you’d write it on a blackboard, once as it lives in the repository — so you can check, line by line, that the machine proof is the human proof.

What we are doing, and why. We are building software whose correctness is proved in Coq, aiming for modifiability: change something, re-establish a few local facts, trust the rest. In post #8 we bought the algebra that analysis needs. Now we start the linear algebra course proper — and we do it without coordinates. No matrices in this post. A matrix, when it finally appears, will be a representation of a linear map, never the definition of one.

The cast: linear maps

A linear map between vector spaces is a function that respects the structure. In the repository (Interface/LinearMap.v) it is a record: a function lm_fun, respect for equality (lm_proper), and two laws

f(u+v)=f(u)+f(v)f(au)=af(u).f(u + v) = f(u) + f(v) \qquad f(a\,u) = a\,f(u).

Theorem 1: a linear map sends zero to zero

Every course proves this first, and the proof is a small gem because it doesn’t use what you’d expect — no scaling by 0, just additivity and cancellation.

Blackboard. f(0)=f(0+0)=f(0)+f(0)f(0) = f(0+0) = f(0) + f(0). Now cancel one f(0)f(0) from both sides of f(0)+f(0)=f(0)+0f(0) + f(0) = f(0) + 0… wait, that’s the trick: rewrite the goal f(0)=0f(0) = 0 as f(0)+f(0)=f(0)+0f(0) + f(0) = f(0) + 0 and cancel on the left. ∎

Repository (Theory/Group.v — note it’s proved for any additive map between groups; vector spaces inherit it):

Theorem hom_zero : f 0 == 0.
Proof.
  apply (cancel_l (f 0)).
  rewrite <- fadd, idl, idr.
  reflexivity.
Qed.

To check the Coq proof against the blackboard, you need the types of the pieces:

name statement
cancel_l forall a b c, a + b == a + c -> b == c
fadd forall a b, f (a + b) == f a + f b (the map’s additivity)
idl, idr 0 + a == a, a + 0 == a

Read it backwards, the way apply works: to show f 0 == 0, by cancel_l it suffices to show f 0 + f 0 == f 0 + 0. Rewriting right-to-left with fadd turns the left side into f (0 + 0); idl collapses the inner 0 + 0; idr collapses the right side; both sides are now f 0. That is the blackboard proof, with the bookkeeping made honest.

The twist: the maps themselves form a vector space

Here is where the course usually gets interesting. Define, for linear maps f,g:U→Vf, g : U \to V,

(f+g)(u)=f(u)+g(u),(af)(u)=af(u).(f + g)(u) = f(u) + g(u), \qquad (a\,f)(u) = a\,f(u).

Claim: with these operations, Hom(U,V)\mathrm{Hom}(U,V) — the set of all linear maps U→VU \to V — is itself a vector space.

There are two things to prove, and they are different in kind:

Closure (the interesting part): f+gf + g must again be linear. Blackboard, for additivity:

(f+g)(u+v)=f(u+v)+g(u+v)=(f(u)+f(v))+(g(u)+g(v))=!(f(u)+g(u))+(f(v)+g(v))(f+g)(u+v) = f(u+v) + g(u+v) = \big(f(u)+f(v)\big) + \big(g(u)+g(v)\big) \stackrel{!}{=} \big(f(u)+g(u)\big) + \big(f(v)+g(v)\big)

The marked step is the interchange law — swapping the middle two summands — which holds in any abelian group. In Theory/HomVS.v:

Lemma hom_plus_add : forall u v : U,
  f (u + v) + g (u + v) == (f u + g u) + (f v + g v).
Proof.
  intros u v.
  rewrite (lm_add f u v), (lm_add g u v).
  apply add_interchange.
Qed.

with add_interchange : (a + b) + (c + d) == (a + c) + (b + d) doing exactly the marked step. Closure of afa\,f under the scale law is the one place a commutativity of scalars sneaks in: af(bu)=abf(u)=baf(u)=b(af(u))a\,f(b\,u) = ab\,f(u) = ba\,f(u) = b\,(a\,f(u)) — in the repo this is the lemma scale_swap, and it is why Hom over a noncommutative ring would be a subtler story.

Laws (the boring part, and that’s the point): associativity, identities, inverses for + on maps all hold because they hold at every point u. Each proof in the repository is three lines of “apply the pointwise law”. We built this pattern once, as the pointwise construction of post #6; Hom just rides it through lm_fun.

Dual space: measurements are vectors too

A special case with a famous name (Theory/Hom.v):

V*=Hom(V,F)V^* \;=\; \mathrm{Hom}(V, F)

the dual space — linear “measurements” of vectors. Since scalars form a (one-dimensional) vector space over themselves, V*V^* is a vector space by the theorem above, with nothing new to prove: in the repository the instance resolves automatically:

Example dual_vs_check : VectorSpace F (Dual F V) := _.

That underscore is Coq assembling the whole proof by instance search.

And now the most elegant map in this post. Every vector vv is a measurement of measurements: feed it a functional, it returns the functional’s value on it:

ι:V→V**,ι(v)(φ)=φ(v).\iota : V \to V^{**}, \qquad \iota(v)(\varphi) = \varphi(v).

Note what is not in that definition: no basis, no coordinates, no choices. Its linearity is, satisfyingly, each φ\varphi’s own linearity read backwards — ι(u+v)(φ)=φ(u+v)=φ(u)+φ(v)=(ι(u)+ι(v))(φ)\iota(u+v)(\varphi) = \varphi(u+v) = \varphi(u)+\varphi(v) = \big(\iota(u)+\iota(v)\big)(\varphi) — and in the repository the middle equality is literally lm_add φ, each functional testifying for the map. (That ι\iota is an isomorphism when VV is finite-dimensional is a later theorem; it needs bases, which is exactly why it’s later.)

Why this ordering pays

Post #6 ended with “the representation is a degree of freedom”. This post built the space where representations of operations will live: in wave L2 (in progress as this posts), choosing bases for UU and VV gives an isomorphism

Hom(U,V)≅Fm×n\mathrm{Hom}(U,V) \;\cong\; F^{m \times n}

and the punchline theorem [[g∘f]]=[[g]][[f]][\![g \circ f]\!] = [\![g]\!]\,[\![f]\!]: matrix multiplication is function composition, transported along that isomorphism. Every matrix identity you’ve ever memorized will be a one-line rewrite along a representation change. The blackboard and the kernel agree; bring your own coordinates, or don’t.
