Commutative structure: abelian groups and rings

A monoid is a set with an associative combining operation + and a do-nothing element 0 (so (a + b) + c == a + (b + c), 0 + a == a, a + 0 == a). A group adds undoing: a negation - a with (- a) + a == 0 and a + (- a) == 0, so every combination can be reversed. Nothing so far says the order of combination matters — and in a group it need not: composing two rotations one way generally differs from the other. This post is about the structures where order doesn’t matter, and about what a second operation buys once it does.

Throughout, == is the structure’s own notion of equality: an equivalence relation the carrier comes equipped with, not necessarily Coq’s built-in =. For the integers it is ordinary equality; in the next post it will be something else. And a =/= b is shorthand for ~ (a == b).

Each proof below is written as a chain. The line

calc x
  == y  { law1 }
  == z  { law2 }.

reads as x equals y by law1, which equals z by law2. Every brace names the single fact that justifies its step, and Coq checks the chain end to end, so the displayed calculation is the proof, not a comment alongside one.

The commutativity law

An abelian group is a group with one law added: combining is commutative, a + b == b + a. That is the whole definition.

Class AbGroup G := {
  #[global] ab_group :: Group G;
  ab_comm  : forall a b : G, a + b == b + a
}.

The first field says an abelian group is a group (with all its operations and laws); the second adds the one new axiom. Everything proved for groups holds here unchanged, and ab_comm lets us rearrange sums freely.

A reshuffling kit

With commutativity in hand, sums of several terms can be permuted into any order. Two small lemmas do the local moves. Their statements:

Theorem add_swap_r : forall a b c : G, (a + b) + c == (a + c) + b.
Theorem add_swap_l : forall a b c : G, a + (b + c) == b + (a + c).

add_swap_r swaps the last two summands of a left-nested sum; add_swap_l swaps the first two of a right-nested one. Here is the left one in full. In words: reassociating to expose the front pair, swapping it, and reassociating back moves b ahead of a.

a+(b+c)=(a+b)+c=(b+a)+c=b+(a+c)a + (b + c) \;=\; (a + b) + c \;=\; (b + a) + c \;=\; b + (a + c)

Theorem add_swap_l : forall a b c : G, a + (b + c) == b + (a + c).
Proof.
  intros a b c.
  calc (a + (b + c))
    == ((a + b) + c)  { assoc }
    == ((b + a) + c)  { (comm a b) }
    == (b + (a + c))  { assoc }.
Qed.

Step by step: assoc (associativity) reparenthesizes a + (b + c) as (a + b) + c; comm a b (commutativity, applied to the pair a + b) turns it into (b + a) + c; assoc reparenthesizes once more to land on b + (a + c).

The lemma worth its own name is the interchange law: two sums added together can be regrouped by pairing first-with-first and second-with-second.

(a+b)+(c+d)=a+(b+(c+d))=a+(c+(b+d))=(a+c)+(b+d)(a + b) + (c + d) \;=\; a + (b + (c + d)) \;=\; a + (c + (b + d)) \;=\; (a + c) + (b + d)

Theorem add_interchange : forall a b c d : G,
  (a + b) + (c + d) == (a + c) + (b + d).
Proof.
  intros a b c d.
  calc ((a + b) + (c + d))
    == (a + (b + (c + d)))  { assoc }
    == (a + (c + (b + d)))  { (add_swap_l b c d) }
    == ((a + c) + (b + d))  { assoc }.
Qed.

The middle step is the only interesting one. assoc first pulls the leading a out front, exposing the tail b + (c + d). Then add_swap_l b c d — the lemma just proved — swaps b past c inside that tail, rewriting b + (c + d) into c + (b + d). A final assoc regroups a + (c + (b + d)) as (a + c) + (b + d). Four terms went in as (a b)(c d) and came out as (a c)(b d), with commutativity used exactly once, buried in add_swap_l.

Rings: a second operation

A ring keeps the abelian group under + and adds a second operation * with its own identity 1. Multiplication is associative and unital, and — the law that ties the two operations together — it distributes over addition: a * (b + c) == a * b + a * c, and the same on the right. Multiplication is not assumed commutative.

Class Ring R := {
  #[global] ring_ab         :: AbGroup R;
  #[global] ring_one        :: One R;
  #[global] ring_mul        :: HasMult R;
  ring_mul_assoc  : forall a b c : R, (a * b) * c == a * (b * c);
  ring_mul_1l     : forall a : R, 1 * a == a;
  ring_mul_1r     : forall a : R, a * 1 == a;
  ring_distl      : forall a b c : R, a * (b + c) == a * b + a * c;
  ring_distr      : forall a b c : R, (a + b) * c == a * c + b * c;
  #[global] ring_mul_proper :: Proper (equ ==> equ ==> equ) (@mult R _)
}.

The last field, ring_mul_proper, records that * respects == in both arguments: replacing a factor by an equal one gives an equal product. That is automatic when == is plain equality, but it is a real obligation when == is a coarser equivalence — which is the whole subject of the next post.

One name, two operations

Look back at the ring: it now has two associative operations on the same carrier (+ and *) and two identities (0 and 1). A proof about * will want to say “reassociate” and “use the identity” just as often as a proof about +. Watch how one short ring theorem does it. The fact — negating by multiplying by -1 — reads

(−1)⋅a=−(1⋅a)=−a(-1)\cdot a \;=\; -(1\cdot a) \;=\; -a

Theorem mul_neg1_l : forall a : R, (- (1)) * a == - a.
Proof.
  intros a.
  calc ((- (1)) * a)
    == (- (1 * a)) { (mul_neg_l 1 a) }
    == (- a)       { idl }.
Qed.

The first step uses mul_neg_l — the fact (- x) * y == - (x * y), that a negation can be pulled out of a product, proved earlier in the same file — to rewrite (-1) * a as - (1 * a). The second step names idl, the left-identity law. But idl is the very name a monoid uses for 0 + a == a. How does the same name discharge 1 * a == a?

The answer is in Props.v, where the single-law classes live. Each is parameterized not just by a carrier but by the operation it talks about:

Class Idl T (e : EqRel T) (op : BinOp T) unit :=
  idl : forall a, op unit a == a.

Idl T e op unit is the statement op unit a == a for a given operation op and unit. A monoid registers its additive identity as one instance of this class; a ring registers its multiplicative identity as a second, separate instance on the same carrier:

#[global] Instance Ring_MulAssoc R (H : Ring R) : Assoc R equ mult := ring_mul_assoc.
#[global] Instance Ring_MulIdl R (H : Ring R) : Idl R equ mult 1 := ring_mul_1l.
#[global] Instance Ring_MulIdr R (H : Ring R) : Idr R equ mult 1 := ring_mul_1r.

So on a ring carrier there are two Idl instances: Idl R equ add 0 (from the additive monoid) and Idl R equ mult 1 (from Ring_MulIdl). When a proof writes rewrite idl, Coq first reads the goal to see which operation is present — in mul_neg1_l the term is - (1 * a), which contains mult, not add — and only then searches for an Idl instance whose operation matches. It finds Ring_MulIdl. A goal mentioning 0 + a would have matched the additive instance instead. One word, idl, resolved by the shape of the goal; likewise assoc on a product means multiplicative associativity, assoc on a sum means additive.

One instance is pointedly absent. A ring registers Assoc, Idl, and Idr for *, but not Commutes — multiplication in a general ring need not commute (square matrices are the standard counterexample), so rewrite comm on a product finds no matching instance and simply fails. Restoring that instance is exactly what makes a ring a field, which is where the next post begins.

The integers are a ring

None of the above is worth anything until a real type satisfies it. The integers Z do: with == as ordinary equality, addition and multiplication from the standard library, and every law discharged by a standard-library lemma, Z is registered as a ring. The generic theory then applies to it directly. For instance, mul_0_l — the ring theorem that 0 * a == 0, proved once from distributivity for every ring — instantiates at Z:

Example z_mul_0_l : (0 * 7)%Z == 0.
Proof. exact (mul_0_l 7%Z). Qed.

No induction on integers, no case split on signs: the fact 0 * 7 == 0 for Z is the general ring lemma with R fixed to Z and a fixed to 7. The abstraction earns its keep the moment a second ring arrives and inherits the same body of theorems unchanged.
