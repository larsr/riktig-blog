Inserting a channel between two composed programs

Two programs are wired together: the first computes a value, the second consumes it. Between them you would like to slip a channel — a step that serializes the value to some wire format and deserializes it back again — without changing what the pair computes. Intuitively this is obviously fine, provided the serialize/deserialize pair is a genuine round trip: whatever you encode, you get back unchanged when you decode. This post makes that intuition into a theorem, proves it once for any monad, and then runs it on a concrete codec built from a standard pairing bijection.

Monads and Kleisli composition

A monad is a way to sequence computations that might fail (or have some other effect). It comes with two operations: ret x produces the value x with no effect, and bind m f — written m >>= f — runs m, and if it produced a value, hands that value to f to get the next computation. Throughout, == is a setoid equality: a chosen equivalence relation, one per result type, that every operation must respect. For the option monad it is simple — Some a == Some b holds exactly when a = b.

An ordinary function composes with ∘: given f : A -> B and g : B -> C, the composite g ∘ f feeds f’s output into g. When the functions are effectful — of the shape A -> M B and B -> M C, each returning a value inside the monad M — plain ∘ no longer type-checks: f’s output is an M B, but g wants a B. Kleisli composition is the fix. It runs f, then binds its value into g, threading the monad through:

Notation "f >=> g" := (fun x => bind (f x) g) (at level 55, right associativity).

So (f >=> g) x is f x >>= g: run f on x, and pipe the result into g. This is ∘ for functions that live in a monad, and it is what lets us write a pipeline of effectful steps as one arrow.

A Kleisli isomorphism: a round trip that lands on ret

The channel we want to insert is a matched encode/decode pair. Encoding turns a B into some wire representation B', decoding turns it back; both are effectful, so encoding is a Kleisli arrow B -> M B' and decoding is B' -> M B. For the pair to be a lossless round trip, going one way and back must return the original value with no effect — that is, must equal ret. Bundling the two arrows with the two round-trip laws gives a record:

Record KleisliIso (M : Type -> Type) `{Monad M} (A B : Type) := {
  ki_to : A -> M B;
  ki_from : B -> M A;
  ki_fromto : forall a : A, bind (ki_to a) ki_from == ret a;
  ki_tofrom : forall b : B, bind (ki_from b) ki_to == ret b
}.

ki_to is the forward arrow and ki_from the backward one. The law ki_fromto says: encode any a, then bind the result through ki_from to decode it, and you are back to ret a — the original value, no effect left over. ki_tofrom is the same round trip started from the other side. These two bind ... == ret ... equations are the entire content of “lossless.” (The round trips land on ret, not on the raw identity function, because in a monad the identity for bind is ret, not fun x => x.)

The theorem: the channel is transparent

Now the claim. Take two programs prog1 : A -> M B and prog2 : B -> M C, composed as prog1 >=> prog2. Given any codec KleisliIso M B B', splicing its encode-then-decode into the seam between them leaves the composite unchanged. Writing p1p_1, p2p_2 for the two programs and ee, dd for the codec’s encode (ki_to) and decode (ki_from) arrows:

p1>=>p2=p1>=>e>=>d>=>p2.p_1 \mathbin{>\!=\!>} p_2 \;=\; p_1 \mathbin{>\!=\!>} e \mathbin{>\!=\!>} d \mathbin{>\!=\!>} p_2.

The whole proof reduces to one small fact about a single value, and then a two-line theorem that lifts it under prog1.

The heart: one value through the codec and back

Fix a value b : B. Encoding it, then binding through decode, then running prog2, is the same as running prog2 b directly. Writing >>=\mathbin{>\!\!>\!\!=} for bind and η\eta for ret, the argument is a three-step chain:

e(b)>>=(λb′.d(b′)>>=p2)=(e(b)>>=d)>>=p2{associativity, right to left}=η(b)>>=p2{the round trip: e(b)>>=d=η(b)}=p2(b){η is bind’s left unit.} \begin{aligned} e(b) \mathbin{>\!\!>\!\!=} \big(\lambda b'.\ d(b') \mathbin{>\!\!>\!\!=} p_2\big) &= \big(e(b) \mathbin{>\!\!>\!\!=} d\big) \mathbin{>\!\!>\!\!=} p_2 && \{\text{associativity, right to left}\}\\ &= \eta(b) \mathbin{>\!\!>\!\!=} p_2 && \{\text{the round trip: } e(b) \mathbin{>\!\!>\!\!=} d = \eta(b)\}\\ &= p_2(b) && \{\eta \text{ is bind's left unit.}\} \end{aligned}

The first step reassociates the two binds so that the encode and decode sit next to each other. The second step is where losslessness is spent: (ki_to b) >>= ki_from is exactly the round trip ki_fromto collapses to ret b. The third step uses the monad law that ret b >>= prog2 is just prog2 b — feeding a pure value straight in. Here is the chain in Coq:

Lemma ki_insert_pointwise : forall b : B,
  bind (ki_to codec b) (fun b' => bind (ki_from codec b') prog2) == prog2 b.
Proof.
  intro b.
  calc (bind (ki_to codec b) (fun b' => bind (ki_from codec b') prog2))
    == (bind (bind (ki_to codec b) (ki_from codec)) prog2)
    == (bind (ret b) prog2)
    == (prog2 b).
  - symmetry. apply bind_assoc.
  - apply bind_proper.
    + exact (ki_fromto codec b).
    + intro x. reflexivity.
  - apply bind_ret_l.
Qed.

The calc block states the three-line chain; each bullet below discharges one step:

  • symmetry. apply bind_assoc is the reassociation. The monad’s associativity law bind_assoc reads (m >>= f) >>= g == m >>= (fun x => f x >>= g); the chain uses it right-to-left, so symmetry flips it before applying.
  • apply bind_proper handles the middle step. bind_proper says bind respects == in both arguments: replace the first argument by an equal one and the continuation by a pointwise-equal one, and the result is unchanged. Here the first argument bind (ki_to codec b) (ki_from codec) is replaced by ret b — that is exact (ki_fromto codec b), the codec’s own round-trip law — while the continuation prog2 is unchanged, so its side is intro x. reflexivity.
  • apply bind_ret_l is the last step: bind_ret_l is the law bind (ret x) f == f x, turning bind (ret b) prog2 into prog2 b.

Lifting it under prog1

The theorem itself now barely has to work. Both sides start with the same prog1; they differ only in the continuation, and ki_insert_pointwise says those continuations agree on every value b. So one appeal to bind_proper — reflexivity on prog1, the pointwise lemma on the continuation — finishes it:

Theorem channel_insertion : forall a : A,
  (prog1 >=> prog2) a == (prog1 >=> ki_to codec >=> ki_from codec >=> prog2) a.
Proof.
  intro a.
  apply bind_proper.
  - reflexivity.
  - intro b. symmetry. exact (ki_insert_pointwise b).
Qed.

Unfolding the >=> notation, the left side is prog1 a >>= prog2 and the right side is prog1 a >>= (fun b => ki_to codec b >>= ...). bind_proper splits the goal into the two arguments of the outer bind: reflexivity on prog1 a (literally the same on both sides), and the continuations, which ki_insert_pointwise matched at every b — used via symmetry because the theorem states the plain composite on the left and the channelled one on the right. That is the whole proof: the codec’s round trip is ret, and ret is what bind does nothing with.

A concrete channel: the Cantor pairing codec

To see the theorem at work, build a real codec. Cantor’s pairing function is a standard-library bijection between nat * nat and nat: to_nat packs a pair of naturals into a single natural, of_nat unpacks it, and the standard library already proves the two round trips cancel_of_to and cancel_to_of. Wrapping each direction in Some gives a KleisliIso for the option monad — the two round-trip laws are exactly those cancellation facts:

Lemma nat_pair_fromto : forall p : nat * nat,
  bind (Some (to_nat p)) (fun n => Some (of_nat n)) == Some p.
Proof.
  intro p.
  unfold bind, Option_HasBind, equ, Option_EqRel.
  rewrite cancel_of_to.
  reflexivity.
Qed.

Lemma nat_pair_tofrom : forall n : nat,
  bind (Some (of_nat n)) (fun p : nat * nat => Some (to_nat p)) == Some n.
Proof.
  intro n.
  unfold bind, Option_HasBind, equ, Option_EqRel.
  rewrite cancel_to_of.
  reflexivity.
Qed.

Each proof unfolds the option bind and its equality, then rewrites by the matching cancellation law; the two sides become identical. Packaging the arrows and laws into the record gives the channel:

Definition nat_pair_codec : KleisliIso option (nat * nat) nat :=
  {| ki_to := fun p : nat * nat => Some (to_nat p);
     ki_from := fun n : nat => Some (of_nat n);
     ki_fromto := nat_pair_fromto;
     ki_tofrom := nat_pair_tofrom |}.

ki_to sends a pair to its single-nat “wire format,” ki_from reads it back. Now two toy programs to sit on either side of it — one producing a pair, one consuming it:

Definition split_n (n : nat) : option (nat * nat) := Some (n, n).
Definition sum_pair (p : nat * nat) : option nat := Some (Nat.add (fst p) (snd p)).

split_n duplicates its input into a pair; sum_pair adds the two components back together. The pipeline split_n >=> sum_pair takes n to n + n. Applying the theorem to exactly this pipeline and codec is a one-liner — it type-checks precisely because everything lines up:

Example channel_insertion_nat_pair_check :=
  channel_insertion split_n sum_pair nat_pair_codec.

channel_insertion_nat_pair_check is a proof that split_n >=> sum_pair and split_n >=> ki_to nat_pair_codec >=> ki_from nat_pair_codec >=> sum_pair agree on every input. The serialization channel is invisible to the answer — which is the whole point, and now a checked fact rather than a hope.
