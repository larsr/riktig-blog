The alternating-bit protocol: safety, unconditionally

A communication channel, in the model this post works over, is not a pipe that either works or doesn’t — it is a relation between the list of messages a sender handed in and the list a receiver got out. A perfect channel is the relation “received equals sent.” A lossy one is “received is what you get by dropping some of the sent items.” Others duplicate items, or permute them. Each fault class is just a set of sent/received pairs the channel is allowed to produce, and a worse channel is a bigger set — more behaviours it might inflict on you.

This post is about one protocol running over one of the nastier relations, and one theorem: no matter how badly the channel misbehaves, the receiver never hands its application a message that is wrong. It proves that unconditionally — with no assumption whatever about how often the channel cooperates. Whether messages actually arrive is a genuinely separate question with a genuinely different answer, and it is deliberately not in this post.

The problem: telling a retransmission from a new message

Suppose the channel can lose packets and can also duplicate them. A sender that wants a message to get through has an obvious move: send it several times, and hope at least one copy survives. But now the receiver has a new problem. It sees a copy of message m. Later it sees another copy of m. Is that second copy a retransmission of the same logical message — which it should ignore, having already delivered m — or is it a genuinely new message that just happens to equal m, which it should deliver again? Nothing in the message itself answers this.

The alternating-bit protocol answers it with a single extra bit. The sender tags each logical message with one bit, and flips that bit every time it moves on to the next message. So the wire carries (0, m₀), then (1, m₁), then (0, m₂), and so on, alternating. All the retransmissions of one message carry the same bit; the next message carries the opposite bit. The receiver keeps a single bit of its own — the bit it is currently expecting. When a packet arrives whose bit matches, the receiver delivers the message and flips its expected bit. When a packet arrives whose bit does not match, it is a stale retransmission of a message already delivered, and the receiver silently drops it. One bit is enough because at any moment only two logical messages are ever in play: the one just delivered and the one coming next, and they always carry opposite bits.

The channel: loss and duplication, in any interleaving

Here is the exact relation the sender’s wire runs through, from Channel/Trace.v. It is an inductively defined relation between a sent list and a received list:

Inductive lossy_dup : list M -> list M -> Prop :=
  | lossy_dup_nil : lossy_dup [] []
  | lossy_dup_keep : forall x s r,
      lossy_dup s r -> lossy_dup (x :: s) (x :: r)
  | lossy_dup_drop : forall x s r,
      lossy_dup s r -> lossy_dup (x :: s) r
  | lossy_dup_stutter : forall x s r,
      lossy_dup s (x :: r) -> lossy_dup s (x :: x :: r).

Read the four constructors as the four things the channel may do walking along the sent list: match empty with empty; keep a sent item, delivering it once; drop a sent item, delivering nothing for it; or stutter, re-emitting an item it has already delivered, which is how duplication enters. There is no constructor that reorders, and none that invents an item out of nothing — every delivered item traces back to a sent one. That last fact is what makes the protocol provable, and we will lean on it directly.

The sender: a burst per message, alternating the bit

A wire packet is a message with its one-bit tag, from Channel/ABP.v:

Definition bit := bool.

Record Packet := { pkt_bit : bit; pkt_msg : M }.

The sender emits, for each application message, a burst of k identical copies — its retransmission budget — and flips the bit once per logical message, not once per copy:

Fixpoint sender_burst (b : bit) (m : M) (k : nat) : list Packet :=
  match k with
  | O => []
  | S k' => {| pkt_bit := b; pkt_msg := m |} :: sender_burst b m k'
  end.

Fixpoint sender_stream (b : bit) (ms : list M) (k : nat) : list Packet :=
  match ms with
  | [] => []
  | m :: ms' => sender_burst b m k ++ sender_stream (negb b) ms' k
  end.

sender_burst b m k is k copies of the same packet {| pkt_bit := b; pkt_msg := m |}. sender_stream lays the bursts end to end, negating the bit (negb) at each step down the message list. So every packet belonging to one message is literally the same record, and the bit alternates exactly at burst boundaries — the two structural facts everything below rests on.

The receiver: accept on a match, flip, and drop the rest

The receiver folds over the packets that reach it, carrying its expected bit. On a match it delivers the message and flips; otherwise it discards and keeps waiting:

Fixpoint deliver (expect : bit) (ps : list Packet) : list M :=
  match ps with
  | [] => []
  | p :: ps' =>
      if Bool.eqb (pkt_bit p) expect
      then pkt_msg p :: deliver (negb expect) ps'
      else deliver expect ps'
  end.

A companion function records only the final expected bit after processing a list of packets, with no delivery — the state at the split point, which we will need to reason about a stream one burst at a time:

Fixpoint next_bit (expect : bit) (ps : list Packet) : bit :=
  match ps with
  | [] => expect
  | p :: ps' =>
      if Bool.eqb (pkt_bit p) expect
      then next_bit (negb expect) ps'
      else next_bit expect ps'
  end.

What safety says: a subsequence, nothing more

“The receiver never delivers anything wrong” is made precise by the subsequence relation, from Theory/Subseq.v. subseq r s means r is s with some elements dropped — same elements, same order, possibly fewer:

Inductive subseq : list A -> list A -> Prop :=
  | subseq_nil : subseq [] []
  | subseq_keep : forall x r s, subseq r s -> subseq (x :: r) (x :: s)
  | subseq_drop : forall x r s, subseq r s -> subseq r (x :: s).

If the delivered list is a subsequence of the sent list, then every message delivered was really sent, no message is delivered twice, and the order is preserved. That single relation rules out corruption, duplication, and reordering all at once. The safety theorem is exactly: delivered is a subsequence of sent.

The crux: one burst delivers at most one message

The whole proof turns on one lemma about a single burst. Whatever lossy_dup does to a burst for message m tagged with bit b — losing copies, duplicating copies, in any interleaving — the receiver either delivers nothing from it, or delivers exactly [m] once, and the two cases are told apart by what happens to the expected bit.

The blackboard reason is short. Every packet the channel produces from this burst is the same record {| pkt_bit := b; pkt_msg := m |}, because the burst was homogeneous and lossy_dup never invents a packet (that is the keep/drop/stutter reading above). So the receiver is folding over a list in which every packet carries bit b. If its expected bit is not b, no packet matches and it delivers []. If its expected bit is b, the first packet matches, delivers m, and flips the expectation to negb b — after which no remaining packet in the burst (all still bit b) can match, so nothing more is delivered. Writing De(w)D_e(w) for what the receiver delivers from ww starting at expectation ee, and Ne(w)N_e(w) for its final expectation:

lossy_dup(burstb(m,k),w)⇒(De(w)=[]∧Ne(w)=e)∨(De(w)=[m]∧Ne(w)=¬e).\mathrm{lossy\_dup}\bigl(\mathrm{burst}_b(m,k),\, w\bigr) \;\Longrightarrow\; \bigl(D_e(w) = [\,] \wedge N_e(w) = e\bigr) \;\vee\; \bigl(D_e(w) = [m] \wedge N_e(w) = \neg e\bigr).

Here is the lemma and its proof from Channel/ABP.v:

Lemma burst_delivery : forall (b : bit) (m : M) (k : nat) (w : list Packet) (expect : bit),
  lossy_dup (sender_burst b m k) w ->
  (deliver expect w = [] /\ next_bit expect w = expect) \/
  (deliver expect w = [m] /\ next_bit expect w = negb expect).
Proof.
  intros b m k w expect H.
  assert (Hall : forall p, In p w -> p = {| pkt_bit := b; pkt_msg := m |}).
  { apply (lossy_dup_preserves_all_eq {| pkt_bit := b; pkt_msg := m |}
             (sender_burst b m k) w H).
    intros p Hp. apply (sender_burst_all_eq b m k p Hp). }
  destruct (bit_dec (pkt_bit ({| pkt_bit := b; pkt_msg := m |})) expect) as [Heq | Hneq].
  - destruct (deliver_homog_eq {| pkt_bit := b; pkt_msg := m |} w expect Hall Heq)
      as [[Hd Hn] | [Hd Hn]].
    + left. split.
      * exact Hd.
      * exact Hn.
    + right. split.
      * exact Hd.
      * exact Hn.
  - left. apply (deliver_homog_neq {| pkt_bit := b; pkt_msg := m |} w expect Hall Hneq).
Qed.

Walking it:

  • The Hall assertion is the homogeneity fact: every packet p in the received burst w equals the record {| pkt_bit := b; pkt_msg := m |}. It is discharged by lossy_dup_preserves_all_eq — the channel keeps every received item equal to something sent — fed the fact sender_burst_all_eq, that every sent packet in the burst is that record.
  • destruct (bit_dec ...) splits on whether the burst’s bit matches the receiver’s expect.
  • In the matching case (Heq), deliver_homog_eq says a homogeneous list whose bit matches delivers either [] (if the channel left the burst empty) or exactly [pkt_msg x0] = [m] and flips the bit; both sub-cases are packaged into the left/right disjuncts.
  • In the non-matching case (Hneq), deliver_homog_neq says nothing is delivered and the expectation is unmoved — the left disjunct.

Notice the budget k never constrains anything here. It holds for k = 0 (an empty burst, delivering nothing) just as for a large k. The size of a burst affects whether a message survives, never whether the wrong thing is delivered.

The safety theorem, proved and walked

Now the induction. It is stated for an arbitrary receiver expectation expect, not tied to the sender’s starting bit, because that generality is exactly what the induction step needs: a burst that is entirely lost leaves the receiver’s expectation out of phase with the next burst, and the proof must handle that. The identity that lets us split the receiver across a burst boundary is

De(w1++w2)=De(w1)++DNe(w1)(w2),D_e(w_1 \mathbin{+\!+} w_2) \;=\; D_e(w_1) \mathbin{+\!+} D_{N_e(w_1)}(w_2),

in the DD/NN notation above — deliver the first part, then deliver the second part starting from wherever the first part left the expectation —

which is the deliver_app lemma, and it is why next_bit exists. The induction runs on the message list ms:

Lemma abp_deliver_subseq : forall (k : nat) (ms : list M)
    (b_stream expect : bit) (wire : list Packet),
  lossy_dup (sender_stream b_stream ms k) wire -> subseq (deliver expect wire) ms.
Proof.
  intros k ms.
  induction ms as [| m ms' IH]; intros b_stream expect wire H.
  - simpl in H.
    assert (Hnil : wire = []).
    { apply (lossy_dup_nil_l [] wire H). reflexivity. }
    rewrite Hnil. simpl. apply subseq_nil.
  - simpl in H.
    destruct (lossy_dup_app_split _ _ H
                (sender_burst b_stream m k) (sender_stream (negb b_stream) ms' k)
                eq_refl) as [w1 [w2 [Hwire [H1 H2]]]].
    rewrite Hwire.
    rewrite (deliver_app w1 w2 expect).
    destruct (burst_delivery b_stream m k w1 expect H1) as [[Hd1 Hn1] | [Hd1 Hn1]].
    + rewrite Hd1. rewrite Hn1. simpl.
      apply subseq_drop.
      apply (IH (negb b_stream) expect w2 H2).
    + rewrite Hd1. rewrite Hn1. simpl.
      apply subseq_keep.
      apply (IH (negb b_stream) (negb expect) w2 H2).
Qed.

Walking it:

  • Base case, ms = []. The sender emitted nothing, so the sent stream is []. lossy_dup_nil_l says a channel with an empty input produces empty output, so wire = []. Then deliver expect [] = [], and subseq_nil closes subseq [] [].
  • Inductive case, ms = m :: ms'. The sent stream is the burst for m followed by the stream for ms'. lossy_dup_app_split cuts the received wire at the matching point into w1 ++ w2, with w1 the channel’s rendering of m’s burst (H1) and w2 the rendering of the rest (H2).
  • rewrite (deliver_app w1 w2 expect) applies the split identity above: what the receiver delivers over the whole wire is what it delivers over w1, then what it delivers over w2 starting from the bit next_bit expect w1.
  • destruct (burst_delivery ...) invokes the crux lemma on w1, giving the two cases.
  • Burst lost (Hd1 : deliver expect w1 = [], Hn1 : next_bit expect w1 = expect): the delivered list is just deliver expect w2. Rewriting leaves the goal subseq (deliver expect w2) (m :: ms'); subseq_drop reduces it to subseq (deliver expect w2) ms', closed by the induction hypothesis on ms' — with the same expect, because a lost burst did not move the expectation.
  • Burst delivered (Hd1 : deliver expect w1 = [m], Hn1 : next_bit expect w1 = negb expect): the delivered list is m :: deliver (negb expect) w2. Rewriting leaves subseq (m :: deliver (negb expect) w2) (m :: ms'); subseq_keep strips the shared head m and reduces it to subseq (deliver (negb expect) w2) ms', closed by the induction hypothesis — this time with the flipped negb expect, because the delivery moved the expectation.

The theorem actually wanted is the instance where the receiver starts by expecting the sender’s starting bit. It is a one-line corollary:

Theorem abp_safety : forall (k : nat) (b0 : bit) (ms : list M) (wire : list Packet),
  lossy_dup (sender_stream b0 ms k) wire -> subseq (deliver b0 wire) ms.
Proof.
  intros k b0 ms wire H.
  apply (abp_deliver_subseq k ms b0 b0 wire H).
Qed.

In words: for any budget k, any starting bit b0, any list of application messages ms, and any wire trace the lossy-and-duplicating channel could produce from the sender’s output, whatever the receiver delivers is a subsequence of ms — writing ≼\preceq for the subsequence relation,

lossy_dup(sent(b0,ms,k),w)⇒Db0(w)≼ms.\mathrm{lossy\_dup}\bigl(\mathrm{sent}(b_0, ms, k),\; w\bigr) \;\Longrightarrow\; D_{b_0}(w) \preceq ms.

No corruption, no duplication, no reordering reaches the application.

What was, and was not, assumed

Look back at the hypotheses of abp_safety. There is exactly one: that wire is some trace the channel lossy_dup allows. There is no assumption that the channel ever cooperates, no lower bound on the budget k, no bound on how much it loses or duplicates. A channel that drops every single packet forever satisfies the hypothesis perfectly — it makes wire = [], the receiver delivers [], and [] is a subsequence of any ms. Vacuously safe, and completely useless.

That is not a flaw in the theorem; it is precisely the boundary of what safety means. Safety promises that nothing wrong comes out. It says nothing about whether anything comes out at all. Ruling out the drops-everything channel needs a separate hypothesis — a promise that the channel is, in some precise sense, fair — and a separate theorem about arrival. That is the subject of the next post, kept apart from this one on purpose, because mixing the two is exactly how the meaning of each gets blurred.
