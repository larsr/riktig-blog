riktig #8 — The shopping list for analysis

Eighth post in the series documenting the riktig project.

What we are doing, and why. We are building software whose correctness is proved, layer by layer, in Coq; the goal is modifiability — a change only obliges you to re-establish a few local facts. Every post starts from zero. In post #1 we built algebraic structures as interfaces with laws; in post #6 the structures met concrete data and representation isomorphisms. Where this is all heading is analysis: limits, derivatives, integrals — verified numerics. Today we buy the items on the shopping list that analysis will ask for — order, absolute value, norms, bases, the real numbers as a specification — and then spend the second half on this wave’s one substantial theorem: complex numbers over any formally real field form a field.

The list, item by item

Order, and an honest absolute value (Interface/Order.v / Theory/Order.v). Analysis begins with “smaller than”: <= arrives as opt-in vocabulary, and OrderedField states a field whose order is reflexive, transitive, antisymmetric up to ==, total, respects +, and keeps products of nonnegatives nonnegative. The design choice worth pausing on is abs: defining |a||a| as “if a≥0a \ge 0 then aa else −a-a” needs to decide the order — something real-number programs cannot generally do — so abs is an operation with laws, not a construction. The headline three:

0≤|a|,|ab|=|a||b|,|a+b|≤|a|+|b|0 \le |a|, \qquad |ab| = |a|\,|b|, \qquad |a+b| \le |a| + |b|

(an abridged list — the interface states seven abs laws in all, the others being properness, abs a == 0 <-> a == 0, abs (- a) == abs a, and 0 <= a -> abs a == a). Proved from the laws alone, in Theory/Order.v: |1|=1|1| = 1, and from that a small gem — 0≤10 \le 1 is a theorem, because one is an absolute value and absolute values are nonnegative.

Norms: length, axiomatically (Interface/Norm.v / Theory/Norm.v). A norm is a ruler for a vector space: ‖u‖ in the ordered scalar field, with the four classical laws — nonnegative, definite, homogeneous (∥a⋅u∥=|a|∥u∥\|a \cdot u\| = |a|\,\|u\| — this is why we needed abs), and the triangle inequality. Proved: ∥0∥=0\|0\| = 0 and ∥−u∥=∥u∥\|-u\| = \|u\|, each a one-line rewrite chain. When limits arrive, “within ε\varepsilon of” will be measured by exactly this ‖·‖.

A basis is a coordinate isomorphism (Interface/Basis.v). Post #6 sold representations-as-isomorphisms; a basis is the same idea wearing a math hat — a way of assigning coordinates: basis vectors b : Fin n -> V plus a linear coordinate map coords : V -> (Fin n -> F) with two round-trip laws,

∑i(coordsv)i⋅bi==vandcoords(∑ici⋅bi)==c.\textstyle\sum_i (\mathrm{coords}\,v)_i \cdot b_i == v \qquad\text{and}\qquad \mathrm{coords}\left(\textstyle\sum_i c_i \cdot b_i\right) == c.

Spanning and linear independence, packed into round trips — “V has a basis” literally is “V ≅ Fin n -> F”. Proved: the standard basis eie_i really is a basis of Fin n -> F, via the delta-collapse ∑ici⋅ei(j)==cj\sum_i c_i \cdot e_i(j) == c_j, by induction. (Infinite bases wait for the analysis track; ListVS from post #6 is the warm-up.)

The reals: a specification, not a construction (Interface/Reals.v). The stance that shapes everything downstream: we do not build ℝ out of Cauchy sequences or cuts. We state what any implementation must provide — ordered-field laws plus completeness: every nonempty, bounded-above predicate has a least upper bound. Programs get written against this interface and then refined to things a computer can hold: rationals for approximation, intervals guaranteed to contain the true value. (The analysis roadmap’s long-term destination for this: state classical mechanics — a Lagrangian L=T−VL = T - V and its equations of motion — against the real-number interface, then run it by refining ℝ to computable approximations, with the proofs carried along. Specification now, numerics later.) (Q famously fails completeness — the supremum of “squares to less than 2” is missing — which is precisely why the interface earns its keep. Nothing is “proved:” here on purpose; a specification’s theorems belong to its implementations.)

The centerpiece: complex numbers, and the law that makes division work

Now the construction the list was leading to (Impl/Complex.v): a genuinely new field from an old one. The carrier is a record of two components, (a,b)(a, b) read as a+bia + bi; equality and addition are componentwise (post #6’s product pattern, again), and multiplication is the Gaussian product, exactly as written in the file:

#[global] Instance C_Mult : HasMult (C F) :=
  fun z w => mkC (re z * re w + - (im z * im w))
                 (re z * im w + im z * re w).

The ring is routine, by design. Every ring law reduces to two component equations over F; the file states one helper lemma per component — pure F-algebra, proved by post-#1 rewrite chains through distributivity and the abelian-group interchange laws — and each C-level law is then a two-line split into the helpers. So far, any field F would do.

Division is where the construction must pay. The candidate inverse is z−1=z‾/(a2+b2)z^{-1} = \bar z / (a^2 + b^2):

#[global] Instance C_Inv : HasInv (C F) :=
  fun z => mkC (re z * inv (norm2 z)) (- im z * inv (norm2 z)).

and the field law inv z * z == 1 needs norm2 z =a2+b2= a^2 + b^2 to be invertible whenever z≠0z \neq 0. Over which fields is that true? Not all of them — over ℂ itself, 12+i2=01^2 + i^2 = 0 with neither summand zero. The exact assumption needed has a classical name (Interface/FormallyReal.v):

Class FormallyReal (F : Type) `{Field F} :=
  fr_sos : forall a b : F, a * a + b * b == 0 -> a == 0.

A sum of two squares vanishes only if each summand does (the symmetric half, fr_sos_r, is derived in Theory/Field.v). That is precisely what norm2_nonzero consumes, and with it the wave’s theorem assembles:

#[global] Instance C_Field : Field (C F) := ...

Over any formally real field F, the complex numbers C F form a field — proved down to the last Proper.

Two design notes. First, the hypothesis is exactly the debt: not “F is ordered”, not “equality is decidable” — one implication about sums of squares, the single fact complex inversion needs. The imaginary unit exists because we paid for it with an axiom about the scalars; nothing is free, but everything is itemized. Second, the file deliberately does not prove FormallyReal Q: that fact belongs to the order machinery (a sum of squares is nonnegative, and zero only at zero), so it lives with Q’s order instance — and once that lands, Field (C Q) assembles by instance resolution with no new proofs in Complex.v. The file ends with resolution checks pinning both halves: Ring (C F) needs only Field F; Field (C F) needs (and only needs) FormallyReal F.

The shopping list is bought. Next stop on the analysis roadmap: sequences, limits, and the first ε\varepsilon-δ\delta proofs — with ‖·‖ as the ruler and Reals as the promise.

Also on deck, whenever we choose to switch tracks: communication channels (lossy, reordering, duplicating — and the protocols that fix them), and the refinement program sketched above — mechanics written against the Reals interface, executed through computable approximations.
