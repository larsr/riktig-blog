riktig #8 — The shopping list for analysis

Eighth post in the series documenting the riktig project.

What we are doing, and why. We are building software whose correctness is proved, layer by layer, in Coq; the goal is modifiability — a change only obliges you to re-establish a few local facts. Every post starts from zero. In post #1 we built algebraic structures as interfaces with laws; in post #6 the structures met concrete data and representation isomorphisms. Where this is all heading is analysis: limits, derivatives, integrals — verified numerics. Today we buy everything on the shopping list that analysis will ask for: order, absolute value, norms, bases, the real numbers as a specification, and complex numbers as a construction.

Order first, and an honest absolute value

Analysis begins with “smaller than”. Interface/Order.v adds <= as opt-in vocabulary and states an OrderedField: a field whose order is reflexive, transitive, antisymmetric up to ==, total, respects +, and keeps products of nonnegatives nonnegative.

The interesting choice is abs. Defining |a||a| as “if a≥0a \ge 0 then aa else −a-a” needs to decide the order — something real-number programs cannot generally do. So abs is an operation with laws, not a construction:

0≤|a|,|ab|=|a||b|,|a+b|≤|a|+|b|0 \le |a|, \qquad |ab| = |a|\,|b|, \qquad |a+b| \le |a| + |b|

No decidability assumed, and yet the laws already prove things — Theory/Order.v derives |1|=1|1| = 1 (because |1||1| solves x=x⋅xx = x\cdot x and isn’t zero), and from that a small gem: 0≤10 \le 1 is a theorem — one is an absolute value, and absolute values are nonnegative.

Norms: length, axiomatically

A norm is a ruler for a vector space (Interface/Norm.v): ‖u‖ in the ordered scalar field, with the four classical laws — nonnegative, definite, homogeneous (∥a⋅u∥=|a|∥u∥\|a \cdot u\| = |a|\,\|u\| — this is why we needed abs), and the triangle inequality. Theory/Norm.v starts collecting consequences: ∥0∥=0\|0\| = 0, ∥−u∥=∥u∥\|-u\| = \|u\|, each a one-line rewrite chain through the vector-space theory of post #6. When limits arrive, “within ε\varepsilon of” will be measured by exactly this ‖·‖.

A basis is a coordinate isomorphism

Post #6 sold representations-as-isomorphisms; bases are the same idea wearing a math hat. What is a basis, really? A way of assigning coordinates. So Interface/Basis.v defines a finite basis of V as: basis vectors b : Fin n -> V together with a linear coordinate map coords : V -> (Fin n -> F) such that

∑i(coordsv)i⋅bi==vandcoords(∑ici⋅bi)==c.\textstyle\sum_i (\mathrm{coords}\,v)_i \cdot b_i == v \qquad\text{and}\qquad \mathrm{coords}\left(\textstyle\sum_i c_i \cdot b_i\right) == c.

Spanning and linear independence, packed into two round trips — “V has a basis” literally is “V ≅ its canonical representation Fin n -> F”. And the sanity check is proved, not assumed: the standard basis eie_i really is a basis of Fin n -> F, via the delta-collapse ∑ici⋅ei(j)==cj\sum_i c_i \cdot e_i(j) == c_j, by induction. (Infinite bases — finitely-supported coordinates over an arbitrary index — wait for the analysis track, where ListVS from post #6 is already the warm-up.)

The reals: a specification, not a construction

Here is the stance that shapes everything downstream (Interface/Reals.v): we do not build ℝ out of Cauchy sequences or cuts. We state what any implementation must provide — ordered-field laws plus completeness: every nonempty, bounded-above predicate has a least upper bound. Programs will be written against this interface and then refined to things a computer can hold: rationals for approximation, intervals [a, b] guaranteed to contain the true value. That is the Lagrangian plan from the roadmap — compute against the spec, run on a representation, carry the soundness proof across. (Q famously fails completeness — the supremum of “squares to less than 2” is missing — which is precisely why the interface earns its keep.)

Complex numbers, and the law that makes division work

Finally, a genuinely new field from an old one (Impl/Complex.v): pairs (a,b)(a, b) read as a+bia + bi, with the Gaussian product (a,b)(c,d)=(ac−bd,ad+bc)(a,b)(c,d) = (ac - bd,\ ad + bc). Addition is componentwise (post #6’s product space, again), and every ring law reduces to two component equations proved by rewriting.

The crux is division: z−1=z‾/(a2+b2)z^{-1} = \bar z / (a^2 + b^2) needs a2+b2≠0a^2 + b^2 \neq 0 whenever z≠0z \neq 0. What makes that true? One assumption, with a classical name:

Class FormallyReal F `{Field F} :=
  fr_sos : forall a b : F, a * a + b * b == 0 -> a == 0.

Sums of squares vanish only trivially. Over any formally real field — ℝ included, by the order axioms — C F is a Field, proved down to the last Proper. The imaginary unit exists precisely because we paid for it with an axiom about the reals; nothing is free, but everything is itemized.

The shopping list is bought. Next stop on the analysis roadmap: sequences, limits, and the first ε\varepsilon-δ\delta proofs — with ‖·‖ as the ruler and Reals as the promise.

Also on deck, whenever we choose to switch tracks: communication channels (lossy, reordering, duplicating — and the protocols that fix them), and the Lagrangian-by-refinement program.
