Normed spaces, and what a limit is

An ordered field is a field — the rationals, say — carrying two extra operations: a comparison <= and an absolute value abs, each fixed by laws rather than by any formula (<= is a total order compatible with addition and multiplication; abs is nonnegative, multiplicative, and obeys the triangle inequality abs (a + b) <= abs a + abs b). From those laws one derives the ordinary facts of ordering — that 0 <= 1, that halving a positive quantity leaves it positive and strictly smaller. This post puts a length on the vectors of a vector space, measured in such a field, and uses it to make precise what it means for a sequence of vectors to converge.

A norm: length measured in the scalar field

A vector space over a field F is an abelian group V — its elements add and negate — together with a scaling operation a *: u that multiplies a vector u by a scalar a from F. A norm assigns to each vector a scalar length, written ‖ u ‖, subject to four laws. Here is the vocabulary and the structure, from Interface/Norm.v:

Class HasNorm F V := vnorm : V -> F.
(* Level 20 keeps the closed ‖…‖ out of application-argument position:
   with identical opening and closing tokens, a level-0 notation would
   let the inner application parser swallow the closing ‖. *)
Notation "‖ u ‖" := (vnorm u) (at level 20, u at level 200).
Class NormedSpace (F V : Type) `{OrderedField F} {VS : VectorSpace F V} := {
  #[global] ns_norm     :: HasNorm F V;
  #[global] norm_proper :: Proper (equ ==> equ) (@vnorm F V _);
  norm_nonneg   : forall u : V, 0 <= ‖ u ‖;
  norm_def      : forall u : V, ‖ u ‖ == 0 -> u == 0;
  norm_scale    : forall (a : F) (u : V), ‖ a *: u ‖ == abs a * ‖ u ‖;
  norm_triangle : forall u v : V, ‖ u + v ‖ <= ‖ u ‖ + ‖ v ‖
}.

The length lands in the scalar field F, which is why F must be an ordered field: a length has to be comparable and nonnegative. The four laws read: every length is nonnegative (norm_nonneg); only the zero vector has length zero (norm_def — the “def” is for definiteness); scaling a vector by a scales its length by abs a (norm_scale); and the length of a sum is at most the sum of the lengths (norm_triangle). The norm_proper field says the norm respects the vector space’s equality — equal vectors have equal length.

The length of a negated vector is unchanged

A first consequence, and a chance to see the ordered-field law abs_one (that abs 1 == 1) doing real work one layer up. In English: ‖ - u ‖ == ‖ u ‖ — negating a vector does not change its length.

Negation is scaling by -1, so on the blackboard the four norm-and-abs facts fall in a row:

‖−u‖=‖(−1)⋅u‖{−u=(−1)⋅u}=|−1|⋅‖u‖{scaling law} \lVert -u\rVert \;=\; \lVert (-1)\cdot u\rVert \quad\{-u = (-1)\cdot u\} \;=\; \lvert -1\rvert\cdot\lVert u\rVert \quad\{\text{scaling law}\} =|1|⋅‖u‖{|−a|=|a|}=1⋅‖u‖{|1|=1}=‖u‖{1⋅x=x}. \;=\; \lvert 1\rvert\cdot\lVert u\rVert \quad\{\lvert -a\rvert = \lvert a\rvert\} \;=\; 1\cdot\lVert u\rVert \quad\{\lvert 1\rvert = 1\} \;=\; \lVert u\rVert \quad\{1 \cdot x = x\}.

Here it is verbatim from Theory/Norm.v:

Theorem norm_neg : forall u : V, ‖ - u ‖ == ‖ u ‖.
Proof.
  intros u.
  calc (‖ - u ‖)
    == (‖ (- (1 : F)) *: u ‖)     { neg1_scale }
    == (abs (- (1 : F)) * ‖ u ‖)  { norm_scale }
    == (abs (1 : F) * ‖ u ‖)      { abs_neg }
    == ((1 : F) * ‖ u ‖)          { abs_one }
    == (‖ u ‖)                    { ring_mul_1l }.
Qed.

The chain is over ==, the vector space’s equality, and each step names one law. neg1_scale rewrites - u as (-1) *: u; norm_scale pulls the scalar out as abs (-1) * ‖ u ‖; abs_neg (abs (- a) == abs a) turns abs (-1) into abs 1; abs_one turns that into 1; and ring_mul_1l (1 * x == x) drops the factor. Five laws, no case split on whether u “points which way” — negation is just a scaling, and the norm laws handle it.

What a limit is, in words before symbols

Now the reason a norm is worth having: it lets us say when a sequence x 0, x 1, x 2, ... of vectors converges to a vector L. The intuition is that the terms get and stay arbitrarily close to L. To make “arbitrarily close” precise without any appeal to infinity, turn it around: name a tolerance first. Convergence to L means that for every positive tolerance eps, however small, there is a cut-off index N beyond which every term is within eps of L — the distance ‖ x n + - L ‖ drops below eps and stays below it. Choosing a smaller eps may force a later cut-off, but a cut-off always exists. That is the whole ε–N idea. Here is the definition, from Theory/Limits.v:

Definition seq_lim (x : nat -> V) (L : V) : Prop :=
  forall eps : F, 0 < eps ->
    exists N : nat, forall n : nat, (N <= n)%nat -> ‖ x n + - L ‖ < eps.

Read literally: for all eps, if 0 < eps, then there exists an index N such that for all n with N <= n, the distance ‖ x n + - L ‖ from the n-th term to L is strictly below eps. The distance is measured with the norm; x n + - L is “the n-th term minus L.”

The payoff: a sequence has at most one limit

The theorem that makes seq_lim a good definition: a sequence cannot converge to two different points. In English: if x tends to L1 and also to L2, then L1 == L2.

The argument is the classic ε/2. Suppose both limits hold, and fix any eps > 0. Feed eps/2 to each hypothesis: past some N1 every term is within eps/2 of L1, and past some N2 within eps/2 of L2. Pick an index n at least as large as both — max N1 N2 serves. At that one index the triangle inequality routes the distance from L1 to L2 through x n:

‖L1−L2‖≤‖L1−xn‖+‖xn−L2‖{triangle inequality}<ε2+ε2{the two limit bounds at n}=ε. \lVert L_1 - L_2\rVert \;\le\; \lVert L_1 - x_n\rVert + \lVert x_n - L_2\rVert \quad\{\text{triangle inequality}\} \;<\; \tfrac{\varepsilon}{2} + \tfrac{\varepsilon}{2} \quad\{\text{the two limit bounds at } n\} \;=\; \varepsilon.

So the fixed distance ‖ L1 + - L2 ‖ is below every positive eps. A quantity that is nonnegative (a norm always is) and below every positive bound can only be zero; and a vector of length zero is the zero vector. Hence L1 - L2 == 0, that is, L1 == L2. Here is the proof verbatim from Theory/Limits.v:

Theorem lim_unique : forall (x : nat -> V) (L1 L2 : V),
  seq_lim x L1 -> seq_lim x L2 -> L1 == L2.
Proof.
  intros x L1 L2 Hx1 Hx2.
  apply sub_eq_zero.
  apply norm_def.
  apply nonneg_below_all_pos_is_zero.
  - apply norm_nonneg.
  - intros eps Heps.
    destruct (Hx1 (half eps) (half_pos eps Heps)) as [N1 HN1].
    destruct (Hx2 (half eps) (half_pos eps Heps)) as [N2 HN2].
    apply lt_le.
    apply (le_lt_trans (‖ L1 + - L2 ‖)
      (‖ L1 + - x (Nat.max N1 N2) ‖ + ‖ x (Nat.max N1 N2) + - L2 ‖) eps).
    + apply norm_sub_triangle.
    + apply (lt_le_trans _ (half eps + half eps) eps).
      * apply add_lt_mono.
        -- rewrite (norm_sub_swap L1 (x (Nat.max N1 N2))).
           apply HN1.
           apply Nat.le_max_l.
        -- apply HN2.
           apply Nat.le_max_r.
      * apply eq_le. apply half_sum.
Qed.

Walking it, top to bottom:

  • apply sub_eq_zero reduces the goal L1 == L2 to L1 + - L2 == 0 — the group fact that two elements are equal exactly when their difference is zero.
  • apply norm_def reduces that to ‖ L1 + - L2 ‖ == 0: by definiteness, showing the difference is zero is enough to show its length is zero.
  • apply nonneg_below_all_pos_is_zero is the “below every positive bound” lemma. It splits the goal ‖ L1 + - L2 ‖ == 0 into two: the length is nonnegative, and it lies below every positive eps.
  • The first bullet, apply norm_nonneg, is exactly nonnegativity of the norm.
  • The second bullet opens with intros eps Heps, fixing a positive eps, and must show ‖ L1 + - L2 ‖ <= eps.
  • destruct (Hx1 (half eps) ...) feeds eps/2 — positive by half_pos — to the first limit, naming its cut-off N1 and the bound HN1; the next line does the same for the second limit, giving N2 and HN2.
  • apply lt_le weakens the goal from <= to a strict <: it suffices to get ‖ L1 + - L2 ‖ < eps, which is what the ε/2 split delivers.
  • apply (le_lt_trans ...) is the triangle step. It routes ‖ L1 + - L2 ‖ through the term x (max N1 N2), bounding it by ‖ L1 + - x_n ‖ + ‖ x_n + - L2 ‖; the bound itself is norm_sub_triangle, the difference form of the triangle inequality.
  • apply (lt_le_trans _ (half eps + half eps) eps) says that sum is below eps/2 + eps/2, which is at most eps. The add_lt_mono step bounds each summand by eps/2 separately: for the first, norm_sub_swap swaps ‖ L1 + - x_n ‖ to ‖ x_n + - L1 ‖ so HN1 applies (at index max N1 N2, which is >= N1 by Nat.le_max_l); the second is HN2 directly (>= N2 by Nat.le_max_r).
  • apply eq_le. apply half_sum closes the last step: half_sum is the fact eps/2 + eps/2 == eps, and eq_le turns that equation into the <= the chain needs.

That is the ε/2 argument in full: two half-bounds, summed at a shared index, squeeze the distance between the two candidate limits below every positive number, and a nonnegative such quantity is zero.

A space where this all runs: matrices

The theorems apply to any normed space. Matrices give a concrete one, in Impl/MatrixNorm.v, with the entrywise L1 norm — the sum of the absolute values of all the entries:

Definition mat_norm (A : Matrix m n F) : F :=
  bigopF (fun i : Fin.t m => bigopF (fun j : Fin.t n => abs (A i j))).

This needs only abs, which every ordered field already carries — no square root, so no L2 norm, is required. The four norm laws are checked for mat_norm, and the file assembles the NormedSpace instance; resolution then confirms it holds for matrices over any ordered field:

Example matrix_ns_check (F : Type) (HFF : Field F) (HF : OrderedField F)
  (m n : nat) : NormedSpace F (Matrix m n F) := _.

The := _ leaves the instance for Coq to find, and it does. With that in scope, seq_lim and lim_unique are immediately available for sequences of matrices: a convergent sequence of m-by-n matrices over the rationals has exactly one limit matrix, by the ε/2 proof above, with nothing further to prove.
