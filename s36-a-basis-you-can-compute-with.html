A basis is a coordinate isomorphism

A vector space over a field FF is a set of vectors you can add and scale by scalars from FF, with the scaling and addition obeying the usual laws: scaling distributes over sums, the scalar 11 leaves a vector alone, and so on. Crucially, that definition mentions no coordinates and no list of numbers — a vector is an abstract thing you can add and scale, nothing more. This post is about the bridge back to numbers: a basis, the structure that lets you write every vector as a finite tuple of scalars and read it back again without loss.

A basis is a coordinate isomorphism

The usual textbook definition of a basis has two clauses — the basis vectors span the space (every vector is some combination of them) and they are linearly independent (that combination is unique). Both clauses are really one statement about a single map. Fix nn basis vectors b0,…,bn−1b_0, \dots, b_{n-1}. They define a coordinate map sending a tuple of scalars cc to the linear combination ∑ici⋅bi\sum_i c_i \cdot b_i, and its would-be inverse sending a vector to its coordinates. Spanning says the first map is onto; independence says it is one-to-one. Together they say: the coordinate map is an isomorphism between the abstract space VV and the concrete space of tuples, which for a finite basis is Fin.t n -> F — functions from the finite index set {0,…,n−1}\{0, \dots, n-1\} to scalars.

So the library defines a basis as exactly that: the data of the basis vectors, a linear coordinate map, and proofs that the two directions undo each other. lincomb b c is the linear combination ∑ici⋅bi\sum_i c_i \cdot b_i; the record demands that reconstructing a vector from its coordinates gives the vector back, and that reading the coordinates of a reconstruction gives the coordinates back:

Record Basis (n : nat) (F V : Type) {FF : Field F}
    {VV : VectorSpace F V} := {
  basis_vec : Fin.t n -> V;
  basis_coords : LinearMap F V (Fin.t n -> F);
  basis_recon : forall v : V,
    lincomb basis_vec (basis_coords v) == v;
  basis_coords_lincomb : forall c : Fin.t n -> F,
    basis_coords (lincomb basis_vec c) == c
}.

Two round trips, basis_recon and basis_coords_lincomb, are spanning and independence packaged as a single mutual inverse — no separate machinery for “a linearly independent list.” The linear combination it refers to is defined as a finite sum over the index set:

Definition lincomb {n : nat} (b : Fin.t n -> V) (c : Fin.t n -> F) : V :=
  bigopF (fun i => c i *: b i).

Here bigopF f sums f over all nn indices of Fin.t n, and *: is scaling a vector by a scalar. To exhibit an actual basis, we need vectors b and a coordinate map for which those two round trips hold.

The standard basis vector

The cleanest example lives in the coordinate space itself, $V = $ Fin.t n -> F — the vectors are the tuples. Its standard basis is the one every linear-algebra course draws first: eie_i is the tuple that is 11 in position ii and 00 everywhere else. In code:

Definition e {n : nat} (i : Fin.t n) : Fin.t n -> F :=
  fun j => if dec (i = j) then 1 else 0.

The if dec (i = j) then 1 else 0 is the decidable-equality idiom: dec (i = j) is not a boolean but a proof-carrying decision that either hands back evidence i = j or evidence i <> j, and the if picks the branch while making that evidence available to whatever needs it (this is the subject of the earlier post on decidable propositions). So e i is 11 on the diagonal i=ji = j and 00 off it — which is precisely the Kronecker delta δij\delta_{ij} written as a computable function.

Two one-line facts read straight off that definition: e i j is 11 when the indices agree and 00 when they differ.

Lemma e_at_eq : forall (n : nat) (i j : Fin.t n), i = j -> e i j == 1.
Proof.
  intros n i j Hij.
  unfold e; destruct (dec (i = j)) as [_ | Hne].
  - reflexivity.
  - destruct (Hne Hij).
Qed.

Lemma e_at_neq : forall (n : nat) (i j : Fin.t n), i <> j -> e i j == 0.
Proof.
  intros n i j Hij.
  unfold e; destruct (dec (i = j)) as [Heq | _].
  - destruct (Hij Heq).
  - reflexivity.
Qed.

Each unfolds e, splits on the decision, and in the impossible branch derives a contradiction from the two conflicting proofs (Hne Hij and Hij Heq are each a proof of False, which destruct turns into anything). One term of the coordinate sum, ci⋅ei(j)c_i \cdot e_i(j), therefore collapses the same way — it is cic_i on the diagonal and 00 off it, because scaling in FF is just multiplication:

Lemma mul_e_eq : forall (n : nat) (a : F) (i j : Fin.t n),
  i = j -> a * e i j == a.
Proof.
  intros n a i j Hij.
  rewrite (e_at_eq n i j Hij).
  exact (ring_mul_1r a).
Qed.

Lemma mul_e_neq : forall (n : nat) (a : F) (i j : Fin.t n),
  i <> j -> a * e i j == 0.
Proof.
  intros n a i j Hij.
  rewrite (e_at_neq n i j Hij).
  exact (mul_0_r a).
Qed.

On the diagonal, a * 1 = a (ring_mul_1r); off it, a * 0 = 0 (mul_0_r). These are the two facts the main argument feeds on.

The delta collapse

Here is the theorem that makes e a genuine basis. Take any coordinate tuple cc and any position jj. The full sum ∑ici⋅ei(j)\sum_i c_i \cdot e_i(j) has one term for every index ii, but ei(j)=δije_i(j) = \delta_{ij} is zero unless i=ji = j, so every term vanishes except the single one at i=ji = j, where it is cj⋅1=cjc_j \cdot 1 = c_j:

∑ici⋅ei(j)=∑ici⋅δij=cj. \sum_{i} c_i \cdot e_i(j) \;=\; \sum_i c_i \cdot \delta_{ij} \;=\; c_j.

That is the whole content of “coordinates read back what a combination put in.” In Coq:

Lemma lincomb_std_delta : forall (n : nat) (c : Fin.t n -> F)
    (j : Fin.t n),
  bigopF (fun i => c i * e i j) == c j.
Proof.
  intros n; induction n as [| n IH]; intros c j.
  - exact (Fin.case0
             (fun j0 => bigopF (fun i => c i * e i j0) == c j0) j).
  - apply (Fin.caseS' j
             (fun j0 => bigopF (fun i => c i * e i j0) == c j0)).
    + transitivity (c Fin.F1 * @e F FF (S n) Fin.F1 Fin.F1
          + bigopF (fun i => c (Fin.FS i) * e (Fin.FS i) Fin.F1)).
      * reflexivity.
      * rewrite (mul_e_eq (S n) (c Fin.F1) Fin.F1 Fin.F1 eq_refl),
          (bigopF_ext n
             (fun i => c (Fin.FS i) * e (Fin.FS i) Fin.F1)
             (fun _ : Fin.t n => (0 : F))
             (fun i => mul_e_neq (S n) (c (Fin.FS i)) (Fin.FS i)
                         Fin.F1 (fin_FS_neq_F1 n i))),
          (bigopF_zero n).
        exact (mon_idr (c Fin.F1)).
    + intros j'.
      transitivity (c Fin.F1 * @e F FF (S n) Fin.F1 (Fin.FS j')
          + bigopF (fun i => c (Fin.FS i) * e (Fin.FS i) (Fin.FS j'))).
      * reflexivity.
      * rewrite (mul_e_neq (S n) (c Fin.F1) Fin.F1 (Fin.FS j')
           (fin_F1_neq_FS n j')),
          (bigopF_ext n
             (fun i => c (Fin.FS i) * e (Fin.FS i) (Fin.FS j'))
             (fun i => c (Fin.FS i) * e i j')
             (fun i => mul_e_FS n (c (Fin.FS i)) i j')),
          (IH (fun i => c (Fin.FS i)) j').
        exact (mon_idl (c (Fin.FS j'))).
Qed.

The proof is induction on the dimension nn, because bigopF is defined by recursion on nn: an empty sum is 00, and a sum over n+1n+1 indices splits as first term plus the sum over the rest, ∑i<n+1fi=fF1+∑i<nfFSi\sum_{i<n+1} f_i = f_{F1} + \sum_{i<n} f_{FS\,i}. Walking it:

  • induction n sets up the base case n=0n = 0 and the step, with IH the statement at dimension nn.
  • The base case has no positions at all — Fin.t 0 is empty — so the index j cannot exist. Fin.case0 discharges the goal by observing there is no such j to prove anything about.
  • In the step, Fin.caseS' j splits on where j sits: either the first position Fin.F1, or Fin.FS j' for some earlier position j'. These are the two cases j=0j = 0 and j≠0j \neq 0 of the blackboard argument.
  • Case $j = $ F1. The transitivity ... reflexivity peels the sum’s first term off the rest: it becomes cF1⋅eF1(F1)+∑icFSi⋅eFSi(F1)c_{F1}\cdot e_{F1}(F1) + \sum_i c_{FS\,i} \cdot e_{FS\,i}(F1), which is what bigopF already equals by definition. Now the first term is on the diagonal, so mul_e_eq turns cF1⋅eF1(F1)c_{F1}\cdot e_{F1}(F1) into cF1c_{F1}. Every remaining term is off-diagonal — FS i is never F1 (that is fin_FS_neq_F1) — so mul_e_neq sends each to 00, bigopF_ext rewrites the whole tail into a sum of zeros, and bigopF_zero collapses it to 00. What is left is cF1+0c_{F1} + 0, and mon_idr (the right-identity law: adding 00 changes nothing) finishes it at cF1c_{F1}.
  • Case $j = $ FS j'. Peel off the first term again. This time it is off-diagonal — F1 is never FS j' (fin_F1_neq_FS) — so mul_e_neq sends cF1⋅eF1(FSj′)c_{F1}\cdot e_{F1}(FS\,j') to 00. In the tail, every term compares FS i against FS j', and stripping the common FS leaves e i j' (that is mul_e_FS); bigopF_ext rewrites the tail into ∑icFSi⋅ei(j′)\sum_i c_{FS\,i}\cdot e_i(j'), which is exactly the induction hypothesis IH applied to the shifted coordinates i↦cFSii \mapsto c_{FS\,i} at j′j' — it equals cFSj′c_{FS\,j'}. What is left is 0+cFSj′0 + c_{FS\,j'}, and mon_idl (the left-identity law) finishes it.

Both cases end where the blackboard did: the whole sum equals the single surviving coordinate.

From the collapse to a basis

The collapse is stated on a bare sum. Lifting it to lincomb gives the round-trip law itself — for every coordinate tuple cc,

∑ici⋅ei=c,\sum_i c_i \cdot e_i \;=\; c,

and the lift is one short step: evaluating a linear combination lincomb e c at a position j means evaluating the sum at j, and evaluation commutes with the sum (bigopF_app), turning it into ∑i(ci⋅ei)(j)=∑ici⋅ei(j)\sum_i (c_i \cdot e_i)(j) = \sum_i c_i \cdot e_i(j) — the collapse again.

Lemma lincomb_std : forall (n : nat) (c : Fin.t n -> F),
  lincomb e c == c.
Proof.
  intros n c j.
  transitivity (bigopF (fun i => (c i *: e i) j)).
  - exact (bigopF_app n (fun i => c i *: e i) j).
  - exact (lincomb_std_delta n c j).
Qed.

lincomb e c == c says: build a vector from a tuple of coordinates using the standard basis, and you recover the very tuple you started with. That single equation is both round trips the Basis record asks for — for this space the coordinate map is the identity, so reconstruction and coordinate-reading are the same statement read in the two directions. The basis assembles with lincomb_std supplied for both fields:

Definition FinVec_Basis (n : nat) : Basis n F (Fin.t n -> F) :=
  @Build_Basis n F (Fin.t n -> F) FF
    (@PW_VectorSpace (Fin.t n) F F FF (@Field_VectorSpace F FF))
    (@e F FF n)
    (@lm_id F (Fin.t n -> F) FF
       (@PW_VectorSpace (Fin.t n) F F FF (@Field_VectorSpace F FF)))
    (fun v => lincomb_std n v)
    (fun c => lincomb_std n c).

The coordinate map is lm_id, the identity linear map; the basis vectors are e; and the two obligations are both discharged by lincomb_std. The delta collapse is the one real computation in the file, and it is what earns e the name “basis”: coordinates become a genuine, checkable round trip rather than a definition asserted by hand.
