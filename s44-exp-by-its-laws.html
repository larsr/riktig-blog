exp, characterized by its laws

A field is a set with the four ordinary arithmetic operations — +, *, negation, and inverses of everything except zero — obeying the usual laws; the rationals are the standard example. An ordered field adds a total order <= compatible with that arithmetic: you may add the same thing to both sides of an inequality, and a product of two nonnegative elements stays nonnegative. Everything below lives over an arbitrary ordered field F.

The previous post specified the real numbers by the properties they must satisfy, with no construction attached. The same move works one level down, for a single function: instead of writing a formula for the exponential — a power series, or a limit of (1+x/n)n(1 + x/n)^n — we can write down the handful of laws exp must obey and derive everything else from those. This post does exactly that.

The four laws

Here is the specification, verbatim from Interface/Exp.v. It takes an ordered field F and asks for a function exp_fn on it, together with four laws (and the fact that exp_fn respects the equality ==):

Class Exp (F : Type) `{OrderedField F} := {
  exp_fn        : F -> F;
  #[global] exp_fn_proper :: Proper (equ ==> equ) exp_fn;

  (* exp is a homomorphism from the additive group to the
     multiplicative monoid. *)
  exp_zero : exp_fn 0 == 1;
  exp_add  : forall a b : F, exp_fn (a + b) == exp_fn a * exp_fn b;

  (* exp lands in the nonnegative reals. Strict positivity (exp_fn a
     =/= 0, hence 0 < exp_fn a since exp_pos already gives 0 <=
     exp_fn a) is NOT stated as a further law: it is a cheap corollary
     of exp_add/exp_zero (exp a * exp (-a) == exp 0 == 1 forces exp a
     to be invertible, hence nonzero) proved once and for all at
     Theory level — see Theory/Exp.v's exp_nonzero and
     exp_strict_pos. Stating it as a law here would be redundant. *)
  exp_pos  : forall a : F, 0 <= exp_fn a;

  (* Strict monotonicity: the property `log` (a later, separate
     development) needs to invert exp onto the positive reals. *)
  exp_mono : forall a b : F, a < b -> exp_fn a < exp_fn b
}.

The four laws read as plain statements about the function:

  • exp_zero: exp_fn 0 == 1 — the exponential of zero is one.
  • exp_add: exp_fn (a + b) == exp_fn a * exp_fn b — it turns sums into products. Together with exp_zero this is exactly what it means for exp_fn to be a homomorphism from (F, +, 0) to (F, *, 1).
  • exp_pos: 0 <= exp_fn a — its values are never negative.
  • exp_mono: a < b -> exp_fn a < exp_fn b — it is strictly increasing.

No formula appears. exp_fn is an unknown function about which we assume only these four things. The claim of this post is that a surprising amount follows from just them.

From nonnegative to strictly positive

exp_pos says exp_fn a is never negative, but it leaves the door open to exp_fn a == 0. The laws already rule that out. The reason is one line of algebra: exp_fn a has a multiplicative partner, namely exp_fn (- a), whose product with it is 1, and nothing whose product with something is 1 can be zero. In symbols, with each step justified by the law it uses:

exp⁡(a)⋅exp⁡(−a)=exp⁡(a+(−a)){sums-to-products law, read backwards}=exp⁡(0){a+(−a)=0, the additive inverse law}=1{the value-at-zero law}\begin{aligned} \exp(a)\cdot\exp(-a) &= \exp(a + (-a)) && \{\text{sums-to-products law, read backwards}\}\\ &= \exp(0) && \{a + (-a) = 0\text{, the additive inverse law}\}\\ &= 1 && \{\text{the value-at-zero law}\} \end{aligned}

That chain is the first lemma in Theory/Exp.v, exp_add_neg:

Lemma exp_add_neg : forall a : F, exp_fn a * exp_fn (- a) == 1.
Proof.
  intros a.
  rewrite <- exp_add, invr, exp_zero.
  reflexivity.
Qed.

Read the rewrites right to left along the blackboard chain. rewrite <- exp_add folds the product exp_fn a * exp_fn (- a) back into exp_fn (a + - a) — the exp_add law used backwards. invr is the field law a + - a == 0, rewriting the argument to exp_fn 0. exp_zero turns that into 1, and reflexivity closes 1 == 1.

Nonvanishing follows immediately: if exp_fn a were 0, then the product above would be 0, but it equals 1, and a field’s 0 and 1 are distinct. On the blackboard, supposing exp⁡(a)=0\exp(a) = 0:

1=exp⁡(a)⋅exp⁡(−a){the chain just proved}=0⋅exp⁡(−a){the supposition}=0{0⋅x=0}\begin{aligned} 1 &= \exp(a)\cdot\exp(-a) && \{\text{the chain just proved}\}\\ &= 0\cdot\exp(-a) && \{\text{the supposition}\}\\ &= 0 && \{0 \cdot x = 0\} \end{aligned}

contradicting 1≠01 \neq 0. That is exp_nonzero:

Lemma exp_nonzero : forall a : F, exp_fn a =/= 0.
Proof.
  intros a Heq.
  apply field_nontrivial.
  rewrite <- (exp_add_neg a), Heq, mul_0_l.
  reflexivity.
Qed.

Here =/= is “not equal”, so the goal unfolds to an assumption Heq : exp_fn a == 0 leading to a contradiction. field_nontrivial is the field’s law that 1 =/= 0; to reach a contradiction it suffices to prove 1 == 0. Rewriting by exp_add_neg turns the target 1 into exp_fn a * exp_fn (- a), Heq replaces exp_fn a by 0, and mul_0_l (0 * x == 0) collapses the product to 0, giving 1 == 0.

Now strict positivity is just “nonnegative and nonzero”. In this development < is defined as <= together with =/=, so 0<exp⁡(a)0 < \exp(a) means exactly 0≤exp⁡(a)0 \le \exp(a) together with 0≠exp⁡(a)0 \neq \exp(a) — the first is the nonnegativity law, the second is the lemma just proved. Proving 0 < exp_fn a therefore splits into precisely those two obligations, both already in hand. This is exp_strict_pos:

Lemma exp_strict_pos : forall a : F, 0 < exp_fn a.
Proof.
  intros a.
  split.
  - apply exp_pos.
  - intros Heq.
    apply (exp_nonzero a).
    symmetry.
    exact Heq.
Qed.

split breaks 0 < exp_fn a into its two parts. The first, 0 <= exp_fn a, is exp_pos verbatim. The second is the apartness 0 =/= exp_fn a; assuming Heq : 0 == exp_fn a, we invoke exp_nonzero a (which wants exp_fn a == 0) and supply Heq turned around by symmetry. Strict positivity of exp, from four laws and no formula.

The exact inverse

The same lemma file records the exact multiplicative inverse, not just nonzero-ness: the exponential of -a is the inverse of the exponential of a. On the blackboard this is a cancellation argument — exp⁡(a)\exp(a) times each of the two candidates gives 11, and a nonzero factor can be cancelled:

exp⁡(a)⋅exp⁡(−a)=1=exp⁡(a)⋅exp⁡(a)−1,exp⁡(a)≠0⇒exp⁡(−a)=exp⁡(a)−1.\exp(a)\cdot\exp(-a) \;=\; 1 \;=\; \exp(a)\cdot\exp(a)^{-1}, \qquad \exp(a) \neq 0 \;\Longrightarrow\; \exp(-a) = \exp(a)^{-1}.

This is exp_neg:

Lemma exp_neg : forall a : F, exp_fn (- a) == inv (exp_fn a).
Proof.
  intros a.
  apply (mul_cancel_l (exp_fn a) (exp_fn (- a)) (inv (exp_fn a))
    (exp_nonzero a)).
  rewrite (exp_add_neg a), (field_invr (exp_fn a) (exp_nonzero a)).
  reflexivity.
Qed.

mul_cancel_l is the cancellation step: since exp_fn a =/= 0 (by exp_nonzero), it reduces the goal to exp_fn a * exp_fn (- a) == exp_fn a * inv (exp_fn a). The left side is 1 by exp_add_neg, and the right side is 1 by field_invr, the field’s law that a nonzero element times its inverse is 1.

Injective, from monotone

The one law not yet used is exp_mono, strict monotonicity, and it buys injectivity: if two inputs have the same exponential, they were the same input. The idea is that a strictly increasing function cannot send two different points to one value — if a and b differed, the smaller would have the strictly smaller exponential, contradicting equality. On the blackboard: suppose exp⁡(a)=exp⁡(b)\exp(a) = \exp(b). If b<ab < a held, monotonicity would give exp⁡(b)<exp⁡(a)\exp(b) < \exp(a), impossible when the two values are equal; so a≤ba \le b, and by the same argument with the roles swapped, b≤ab \le a. Antisymmetry of the order then closes it:

exp⁡(a)=exp⁡(b)⇒a≤b and b≤a⇒a=b.\exp(a) = \exp(b) \;\Longrightarrow\; a \le b \;\text{ and }\; b \le a \;\Longrightarrow\; a = b.

The file proves each direction of <= and then closes by antisymmetry. Here is the finishing lemma, exp_injective:

Lemma exp_injective : forall a b : F, exp_fn a == exp_fn b -> a == b.
Proof.
  intros a b Heq.
  apply le_antisym.
  - exact (exp_le_of_eq a b Heq).
  - apply (exp_le_of_eq b a).
    symmetry.
    exact Heq.
Qed.

le_antisym is the order law a <= b -> b <= a -> a == b, so it splits a == b into the two inequalities a <= b and b <= a. Each is an instance of the helper exp_le_of_eq : exp_fn a == exp_fn b -> a <= b: the first direction feeds it Heq directly, the second feeds it Heq turned around by symmetry to get b <= a. The helper itself is where exp_mono does its work — assuming a and b were ordered the wrong way would make their exponentials strictly ordered, contradicting exp_fn a == exp_fn b.

The point

Every lemma above — exp_add_neg, exp_nonzero, exp_neg, exp_strict_pos, exp_injective — was proved with a Context {F : Type} `{Exp F} at the top of the section and nothing else. Not one of them looked at a formula for exp, because there is no formula to look at: Interface/Exp.v supplies only the four laws. So each of these facts holds for any function on any ordered field that satisfies exp_zero, exp_add, exp_pos, and exp_mono — the power series over the reals, the limit of (1+x/n)n(1 + x/n)^n, or any other construction someone eventually writes down and proves obeys the four laws. Characterizing exp by its laws is what makes all of its theory independent of which construction you pick.
