riktig #24 — multiplication is differentiable

Twenty-fourth post documenting the riktig project. A confirmation, not a discovery: a fact the analysis roadmap assumed was cheap turns out to need one genuine piece of new mathematics, cleanly separated from the bookkeeping around it.

What we are doing, and why. Verified software in Coq, built for modifiability. Every transformer layer that multiplies an input by a weight matrix is, mathematically, a bilinear map applied to a pair — the input, and the weights, treated as two arguments to the same multiplication. docs/ANALYSIS-ROADMAP.md’s Stage 6½ flagged this as something that “should already work for free,” with the caveat “worth a concrete worked example to confirm rather than assume.” Today’s post is that confirmation.

The claim

If BB is a bilinear map — linear in each argument separately, like matrix multiplication or scalar-times-vector — then BB, viewed as a function of both arguments together (a function on the product space U×VU \times V), is differentiable everywhere, with the familiar product-rule derivative:

DB(u0,v0)(du,dv)=B(du,v0)+B(u0,dv)D B(u_0, v_0)(du, dv) = B(du, v_0) + B(u_0, dv)

Every calculus student has seen this for ordinary multiplication (f(x,y)=xyf(x,y) = xy, Df=ydx+xdyDf = y\,dx + x\,dy). The content of today’s theorem is that it holds for any bilinear map into any vector space, proved once, generically:

Theorem bilinear_is_deriv : bm_bounded B ->
  is_deriv (fun p : U * V => B (fst p) (snd p)) (u0, v0) bilinear_deriv_lm.

Matmul is the instance that matters for transformers — multiply an activation by a weight matrix, and you’re applying mat_mul as a bilinear map. Get the general theorem once, and every “multiply input by weights” layer inherits its derivative for free, no separate proof needed per layer.

What had to be built just to state it

Two gaps, found rather than assumed, before any differentiation could happen at all:

No norm on a product space existed. is_deriv needs its domain to carry a NormedSpace. The domain here is U×VU \times V — a product — and nothing in the tree said how to measure the size of a pair. The fix, Theory/ProdVS.v’s new Prod_NormedSpace, uses the sum norm ∥(u,v)∥:=∥u∥+∥v∥\|(u,v)\| := \|u\| + \|v\|, chosen deliberately over the more common max norm because it makes every downstream inequality a one-line add_interchange rewrite instead of a case split on which coordinate is larger.

No boundedness notion existed for a BilinearMap. LinearMap already had one (lm_bounded, living in Theory/Continuity.v as a hypothesis some theorems assume, not a field baked into the record — Interface/LinearMap.v stays norm-free by design). The bilinear analogue, bm_bounded B := exists C, 0 <= C /\ forall u v, ‖B u v‖ <= C * ‖u‖ * ‖v‖ — the nonnegativity conjunct is part of the definition, so later estimates can multiply by C without a sign side-condition — follows the identical pattern: a hypothesis bilinear_is_deriv takes, not a new field forced onto every bilinear map whether or not anyone needs an estimate.

The one genuine piece of analysis

Expand B(u0+du,v0+dv)B(u_0+du,\,v_0+dv) by bilinearity and four terms fall out. Three of them assemble into exactly the claimed value plus the claimed linear approximation; the fourth — the cross term B(du,dv)B(du, dv) — is what’s left over, and it doesn’t cancel:

Lemma bilinear_remainder_eq : forall (du : U) (dv : V),
  B (u0 + du) (v0 + dv) + - (B u0 v0 + (B du v0 + B u0 dv)) == B du dv.

This is the one place bilinearity is a genuinely harder hypothesis than plain linearity. Post #12’s chain rule (linear maps compose, remainders add) never had to fight this — a sum of two small things is small, full stop, zero-cost bookkeeping. But B(du,dv)B(du,dv) is bounded by C⋅∥du∥⋅∥dv∥C \cdot \|du\| \cdot \|dv\| — the product of two small things, which is quadratic in the combined size ∥h∥=∥du∥+∥dv∥\|h\| = \|du\|+\|dv\|, not linear. A quadratic bound isn’t automatically o(∥h∥)o(\|h\|); it takes a genuine squeeze to make it one, in the same spirit as post #13’s Cauchy-Schwarz epsilon-padding: choose δ:=ε/(1+C)\delta := \varepsilon / (1+C), and for ∥h∥<δ\|h\| < \delta the bound C∥h∥2C\|h\|^2 collapses to ε∥h∥\varepsilon\|h\| exactly. Algebra gets you three of the four terms for free; analysis earns the fourth.

What’s confirmed, and what’s still owed

The theorem is unconditional given bm_bounded — no missing cases, no deferred sorry. What’s not done yet, honestly: nothing in the tree currently packages mat_mul itself as a BilinearMap witness, so matmul’s differentiability — the whole point — is a corollary one step away rather than a theorem already stated. That’s a small, mechanical gap (build the witness, show it’s bounded — true for any finite-dimensional bilinear map, itself a fact not yet formalized) and the natural next step once Stage B moves to matmul specifically. Today bought the hard part: the general theorem, proved once, that every future weight-multiplication layer will just inherit.

Next: either matmul’s own BilinearMap witness, closing this loop, or back to the determinant — a fourth attempt has the row-swap gap narrowed to a single classical fact (sign flips under transposition), closer than it’s been all roadmap.
