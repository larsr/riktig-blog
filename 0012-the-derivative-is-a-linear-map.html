riktig #12 — The derivative is a linear map

Twelfth post documenting the riktig project; the linear-algebra thread (posts #9, #10) and the analysis thread (#11) meet. Proofs shown blackboard-style and as they live in the repository.

What we are doing, and why. Verified software in Coq, built for modifiability. Post #9 built the space Hom(U,V) of linear maps; post #10 showed matrices merely represent them; post #11 defined limits with ε and a norm. Today those threads braid: the derivative of a function at a point is a linear map — an element of the Hom-space — and the chain rule is composition. When we later write the Jacobian, it will be post #10’s mat_of applied to today’s object, and the chain rule for Jacobians will already be proved: it is mat_of_comp.

What “differentiable” says

Forget slopes. The derivative answers: which linear map best approximates f near a? In Theory/Derivative.v:

Definition is_deriv (f : U -> V) (a : U) (L : LinearMap F U V) : Prop :=
  lm_bounded L /\
  forall eps, 0 < eps -> exists delta, 0 < delta /\
    forall h, ‖ h ‖ < delta -> ‖ f (a + h) + - (f a + L h) ‖ <= eps * ‖ h ‖.

Three deliberate choices, each with a story:

No division. The textbook says ∥f(a+h)−f(a)−Lh∥/∥h∥→0\|f(a+h) - f(a) - Lh\| / \|h\| \to 0, which forces the annoying side condition h≠0h \neq 0. The equivalent form ∥r(h)∥≤ε∥h∥\|r(h)\| \le \varepsilon \|h\| is division-free and at h=0h = 0 is simply true. Fewer side conditions, shorter rewrites.

LL is a LinearMap — a citizen of post #9’s Hom-space. So “derivative of a sum” can literally say Lf + Lg using Hom’s addition, and does: deriv_add : is_deriv f a Lf -> is_deriv g a Lg -> is_deriv (fun u => f u + g u) a (Lf + Lg).

LL is bounded (∃C,∥Lu∥≤C∥u∥\exists C,\ \|Lu\| \le C\|u\|). In finite dimension every linear map is bounded, so courses never mention it — but the chain rule uses it, silently. Watch it appear below.

Uniqueness — and an honest constructive wrinkle

Blackboard. If L1,L2L_1, L_2 both approximate ff at aa, then for small hh, ∥(L1−L2)h∥≤2ε∥h∥\|(L_1 - L_2)h\| \le 2\varepsilon\|h\|. Take h=tuh = t\,u with t>0t > 0 tiny: t∥(L1−L2)u∥≤2εt∥u∥t\|(L_1-L_2)u\| \le 2\varepsilon\, t\|u\|, cancel tt, and ∥(L1−L2)u∥\|(L_1-L_2)u\| is below 2ε∥u∥2\varepsilon\|u\| for every ε — hence zero, hence L1u=L2uL_1 u = L_2 u for every uu. ∎

The machine proof follows exactly this shape, and the “take tt tiny” step is a lemma with a name and a life of its own — small_scale_exists : 0 < d -> 0 <= c -> exists t, 0 < t /\ t * c < d in Theory/Order.v — no Archimedean axiom needed, just halving. The final squeeze is post #11’s nonneg_below_all_pos_is_zero, and the cancellation of tt is mul_le_cancel_l. Every wave leaves the order-arithmetic toolbox a little richer.

The chain rule

Blackboard. Let k(h)=f(a+h)−f(a)k(h) = f(a+h) - f(a), the increment. Then

g(f(a+h))−g(f(a))−Lg(Lfh)=[g(f(a)+k)−g(f(a))−Lgk]⏟g’s remainder at k+Lg[k−Lfh]⏟Lg of f’s remainder.g(f(a+h)) - g(f(a)) - L_g(L_f h) = \underbrace{\big[g(f(a)+k) - g(f(a)) - L_g k\big]}_{\text{g's remainder at } k} + \underbrace{L_g\big[k - L_f h\big]}_{L_g \text{ of f's remainder}}.

Bound the two pieces: the second is ≤Cg⋅ε1∥h∥\le C_g \cdot \varepsilon_1\|h\| — there is the boundedness of LgL_g, doing the work your lecturer waved through. The first is ≤ε2∥k∥\le \varepsilon_2 \|k\|, and the increment itself satisfies ∥k∥≤(Cf+ε1)∥h∥\|k\| \le (C_f + \varepsilon_1)\|h\| — ff is locally Lipschitz-ish because it is differentiable, boundedness of LfL_f again. Add up, choose the ε’s to share the budget. ∎

Theorem deriv_comp : forall (f : U -> V) (g : V -> W) a Lf Lg,
  Proper (equ ==> equ) g ->
  is_deriv f a Lf -> is_deriv g (f a) Lg ->
  is_deriv (fun u => g (f u)) a (lm_comp Lg Lf).

The repository proof is the blackboard split, factored: the algebraic identity is comp_remainder_split (whose engine is a new AbGroup telescope, sub_mid_cancel : (p - (q + x)) + (x - y) == p - (q + y)), the increment estimate is deriv_increment_bound, and the ε-budget is the half toolbox. One honest deviation from the classical statement, worn openly: g must respect equality (Proper) — setoid discipline demands it, every actual function has it, and the header documents why.

And the derivative of the composition is lm_comp Lg Lf — the composition in Hom. Apply post #10’s mat_of to both sides of that statement and mat_of_comp turns it into: the Jacobian of a composition is the product of the Jacobians. No new proof will be needed. That sentence is the whole architecture in miniature.
