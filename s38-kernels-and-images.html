Kernels, images, and what a linear map forgets

A linear map f:U→Vf : U \to V between two vector spaces is a function that respects the structure: it sends a sum to the sum of the images, f(u+v)=f(u)+f(v)f(u + v) = f(u) + f(v), and it commutes with scaling, f(a⋅u)=a⋅f(u)f(a \cdot u) = a \cdot f(u). From those two facts alone a linear map sends the zero vector to the zero vector, because zero is 0⋅u0 \cdot u for any uu.

Two subsets tell you almost everything about such a map. The kernel is everything the map collapses to zero — what it forgets. The image is everything the map actually reaches — what it can produce. This post shows both, defined directly, and proves that each is a subspace: a subset closed under the vector-space operations, hence a vector space in its own right with the inherited operations. It ends with rank-nullity — the theorem relating their sizes — which is here stated precisely but not yet proved, for a reason given at the end.

The kernel: what the map sends to zero

The kernel of f is the predicate “f sends this vector to zero” (throughout the code, == is a setoid equality — a chosen equivalence relation every operation respects — not Coq’s built-in equality of terms):

  Definition ker_pred (u : U) : Prop := f u == 0.

To call this a subspace we owe three closure facts: the zero vector is in it, it is closed under addition, and it is closed under scaling.

Zero is immediate — f 0 == 0 is the zero-preservation noted above, recorded here as lm_zero f:

  Lemma ker_zero : ker_pred 0.
  Proof. exact (lm_zero f). Qed.

Addition is the first real step. If f forgets u and forgets v, it forgets u + v, because on a sum f splits and the two zeros collapse:

f(u+v)=f(u)+f(v)=0+0=0.f(u + v) \;=\; f(u) + f(v) \;=\; 0 + 0 \;=\; 0.

  Lemma ker_add : forall u v : U, ker_pred u -> ker_pred v -> ker_pred (u + v).
  Proof.
    intros u v Hu Hv.
    unfold ker_pred in Hu, Hv |- *.
    calc (f (u + v))
      == (f u + f v)          { (lm_add f u v) }
      == ((0 : V) + f v)      { Hu }
      == ((0 : V) + (0 : V))  { Hv }
      == (0 : V)              { mon_idl }.
  Qed.

intros u v Hu Hv names the two vectors and the hypotheses f u == 0, f v == 0. unfold ker_pred turns the goal from “u + v is in the kernel” into the plain equation f (u + v) == 0. The calc block is the blackboard chain transcribed step for step, each step justified by the law in braces: lm_add f u v replaces f (u + v) by f u + f v; then Hu rewrites f u to 0, and Hv rewrites f v to 0, leaving 0 + 0. Finally mon_idl — the law 0 + x == x — collapses 0 + 0 to 0.

Scaling is the same shape, one step shorter. If f forgets u, it forgets every scalar multiple a *: u:

f(a⋅u)=a⋅f(u)=a⋅0=0.f(a \cdot u) \;=\; a \cdot f(u) \;=\; a \cdot 0 \;=\; 0.

  Lemma ker_scale : forall (a : F) (u : U), ker_pred u -> ker_pred (a *: u).
  Proof.
    intros a u Hu.
    unfold ker_pred in Hu |- *.
    calc (f (a *: u))
      == (a *: f u)     { (lm_scale f a u) }
      == (a *: (0 : V)) { Hu }
      == (0 : V)        { scale_zero }.
  Qed.

lm_scale f a u pulls the scalar out through f, giving a *: f u; Hu rewrites f u to 0; and scale_zero — the fact that scaling the zero vector gives zero — finishes. Bundle the three closure lemmas and the kernel is registered as a subspace:

  #[global] Instance Ker_SubspacePred : SubspacePred F U ker_pred := {|
    sp_zero  := ker_zero;
    sp_add   := ker_add;
    sp_scale := ker_scale
  |}.

The image: what the map reaches

The image is the dual idea — a vector v is in it when some u maps onto it:

  Definition im_pred (v : V) : Prop := exists u : U, f u == v.

The closure proofs now carry a witness. Membership of v1+v2v_1 + v_2 needs a uu with f(u)=v1+v2f(u) = v_1 + v_2; if u1u_1 reaches v1v_1 and u2u_2 reaches v2v_2, then u1+u2u_1 + u_2 reaches the sum:

f(u1+u2)=f(u1)+f(u2)=v1+v2.f(u_1 + u_2) \;=\; f(u_1) + f(u_2) \;=\; v_1 + v_2.

  Lemma im_add : forall v1 v2 : V, im_pred v1 -> im_pred v2 -> im_pred (v1 + v2).
  Proof.
    intros v1 v2 Hv1 Hv2.
    destruct Hv1 as [u1 Hu1].
    destruct Hv2 as [u2 Hu2].
    exists (u1 + u2).
    calc (f (u1 + u2))
      == (f u1 + f u2)  { (lm_add f u1 u2) }
      == (v1 + f u2)    { Hu1 }
      == (v1 + v2)      { Hu2 }.
  Qed.

The two destructs unpack the existentials into concrete preimages u1, u2 with f u1 == v1 and f u2 == v2; exists (u1 + u2) proposes their sum as the preimage of v1 + v2; and the calc chain — lm_add to split f over the sum, then Hu1 and Hu2 — lands on v1 + v2 exactly as the blackboard line does. Scaling is the analogous one-witness argument (exists (a *: u)), and again the three facts register the image as a subspace:

  #[global] Instance Im_SubspacePred : SubspacePred F V im_pred := {|
    sp_zero  := im_zero;
    sp_add   := im_add;
    sp_scale := im_scale
  |}.

Because both are subspaces, Ker f and Im f are themselves vector spaces — Coq builds those structures by instance search, with no further input, which the file records as a check:

Example ker_vs_check (F U V : Type) (FF : Field F)
  (VU : VectorSpace F U) (VV : VectorSpace F V) (f : LinearMap F U V) :
  VectorSpace F (Ker f) := _.

The := _ asks Coq to find the vector-space structure on Ker f; it succeeds by combining the subspace instance above with the generic fact that any subspace is a vector space.

Rank-nullity: stated, not proved

There is a famous theorem waiting here. The kernel measures how much f forgets; the image measures how much it reaches; and their dimensions are not independent — they must add up to the dimension of the domain. Every dimension the domain spends being forgotten is one it does not spend reaching something new:

dim⁡U=dim⁡(ker⁡f)+dim⁡(im⁡f).\dim U \;=\; \dim(\ker f) + \dim(\operatorname{im} f).

The codebase states exactly this, phrased so it type-checks as a proposition today: given a basis of size n for U, a basis of size k for Ker f, and a basis of size r for Im f, the claim is n = k + r.

Definition rank_nullity_statement : Prop :=
  forall (F U V : Type) (FF : Field F)
    (VU : VectorSpace F U) (VV : VectorSpace F V)
    (f : LinearMap F U V) (n : nat) (Bn : Basis n F U)
    (k : nat) (Bk : Basis k F (Ker f)) (r : nat) (Br : Basis r F (Im f)),
    n = Nat.add k r.

This is a Definition of a Prop, not a Theorem — it names the statement without asserting it. It is not proved in this codebase, and the file is candid about why. To even reach the statement without being handed those bases, one first needs the fact that every subspace of a finite-dimensional space has a basis at all, of some bounded size:

Definition subspace_has_basis_statement : Prop :=
  forall (F V : Type) (FF : Field F) (VV : VectorSpace F V) (n : nat)
    (Bn : Basis n F V) (P : V -> Prop) (SP : SubspacePred F V P),
    exists k : nat, k <= n /\ exists Bk : Basis k F (Sub V P), True.

That missing fact is the Steinitz exchange lemma — extend a basis of a subspace to a basis of the whole space — which is genuine dimension theory not yet built here. So the file does exactly this much: the kernel and image are constructed and proved to be subspaces, the theorem relating their dimensions is written down precisely, and the lemma it would rest on is named rather than quietly assumed. Stating a goal you have not reached, and saying plainly that you have not reached it, is part of doing this carefully.
